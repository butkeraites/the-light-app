## Objetivo

Fazer o bootstrap da estrutura base do repositório `the-light-app` e criar os
documentos de processo, **sem código de produto ainda**: nenhum crate Rust
compilável de lógica (isso é a F0.2) e nenhum app Expo funcional (isso é a F0.5).
A entrega é puramente estrutura de diretórios + docs + a **decisão registrada**
de como o `the-light` será consumido.

## Contexto / arquivos

- Árvore-alvo: seção 1 do `IMPLEMENTATION_PLAN.md` e seção 7 do
  `VISION_AND_ARCHITECTURE.md`.
- Versões do ambiente já confirmadas e auditadas na F0.0 — fonte para a tabela do
  `DECISIONS.md`: `loop/archive/F0.0.result.md`. Resumo: rustc/cargo **1.96.0**,
  rustup 1.29.0, Node **v25.8.1** (npm 11.11.0 / pnpm 11.3.0), **Expo SDK 56**
  (`expo@56.0.12`), gerador de bindings `uniffi-bindgen-react-native` (CLI `ubrn`,
  em transição p/ `uniffi-bindgen-javascript`) — **não instalado** ainda. Targets
  Rust: só `aarch64-apple-darwin` instalado; `wasm32-unknown-unknown` + iOS +
  Android pendentes. `cargo-ndk` ausente; Xcode completo ausente (só Command Line
  Tools); Android SDK/NDK não configurados.
- Sobre a dependência `the-light`: a F0.0 mostrou que **não há crate Rust ainda**
  e a regra é **não modificar/forkar `the-light`**. Portanto a F0.1 entrega
  apenas a **decisão de consumo** (git dependency pinada por commit vs submódulo)
  documentada; a **resolução real** (`cargo metadata`) só será exercida na F0.2,
  quando o crate `core/` nascer. Não bloqueie a F0.1 se o `the-light` não estiver
  resolvível agora.
- `scripts/` já existe (contém `run-loop.sh`) — não recriar.

Arquivos a criar: a árvore de diretórios da seção 1, `DECISIONS.md`,
`PROGRESS.md`, `.gitignore` (todos na raiz do repo).

## Restrições

- **Não tocar nem forkar o repositório `the-light`.** A F0.1 só registra a
  *decisão* de como consumi-lo; nenhuma alteração no core. Mudança no core, se um
  dia necessária, é PR + ADR — nunca aqui.
- **Não bloquear na resolução do `the-light`.** Se o repo não estiver resolvível
  agora, documentar como **pendência para a F0.2** e seguir — não escrever
  `blocked`, não improvisar um fork/cópia.
- **Escopo estrito:** não criar crate Rust compilável de produto (F0.2) nem
  projeto Expo funcional (F0.5). Apenas estrutura de diretórios + docs.
- **Licenciamento:** não embarcar nenhum texto bíblico ou versão nesta tarefa;
  `assets/data/` fica vazio (só `.gitkeep`).
- **Offline-first / BYOK:** nada na estrutura ou nos docs introduz dependência de
  rede ou de conta.
- **Segredos nunca em git:** o `.gitignore` deve cobrir `secrets.toml`/`.env`/
  `*.key`; nenhuma chave commitada.
