## Objetivo

Destubar o **caminho web de IA** (`ask` ancorado no browser) usando as partes
**PURAS** do `ai` do core (feature `ai-pure`, agora ligada na linha WEB de
`core/Cargo.toml`) para **montagem de prompt/RAG + citação** no **wasm** (uma fonte
da verdade em Rust, **zero drift** nativo↔web), com o **transporte por `fetch`**
(delegado ao TS/browser, com a chave). O `cited_text` vem **sempre do store web**
(subset `reading-sample.sqlite`, F1.13) e a `interpretation` vem do `fetch` ao
provedor — **nunca** texto bíblico gerado pelo LLM. Fixar a **política de chave web**
(**session-only / in-memory**, ADR-0025). Prova **headless node** com **MOCK de
`fetch`** (sem rede/chave reais).

### ACHADOS na fonte do core (c8ecb2f, só-leitura — DECISÃO DE DESENHO)
Checkout `~/.cargo/git/checkouts/the-light-9eb8809a6d68281a/c8ecb2f/crates/the-light-core/src/ai/`.

**PÚBLICAS e disponíveis sob `ai-pure` (compilam no wasm E no nativo — chamáveis da
fronteira `the-light-app-core`):**
- `ai::numbered_verses<'a>(impl IntoIterator<Item=(u16,&'a str)>) -> String` (study.rs:113)
  — **cited_text** a partir das tuplas `(número, texto)` do **store web** (MESMA rota do
  nativo, cujo `numbered_passage` chama `numbered_verses`).
- `ai::numbered_passage(&Passage) -> String` (study.rs:126).
- `ai::ask_context(label, numbered, related) -> String` (study.rs:139) — bloco RAG.
- `ai::ask(&dyn LlmProvider, question, context, lang) -> Result<String>` (study.rs:611).
- `ai::default_model(provider) -> &'static str` (providers.rs:40) — `"gemini" => "gemini-2.5-flash"`.
- `ai::estimate_cost_usd(model, in, out) -> Option<f64>` (providers.rs:94).
- `ai::LlmProvider` (trait, mod.rs:347), `ai::MockLlmProvider`, `ai::{Result, AiError}`
  (mod.rs:315/272), `prompts::ask_system_prompt(lang) -> String` (prompts.rs:262),
  `citation::rewrite_anchors(text, &HashSet<String>) -> String` (citation.rs:230).
- `reference::{parse_reference, format_reference, chapters_in_book}` — **puros**, sempre.

**COMPILADAS sob `ai-pure` MAS NÃO-`pub` (módulo-privadas → NÃO chamáveis da fronteira):**
- `gemini_body`/`gemini_extract` (providers.rs:391/427) e todos os
  `anthropic_body`/`anthropic_extract`/`openai_body`/`openai_extract`/`ollama_extract`
  são **`fn` privados** (não `pub fn`). O helper `ask_user_prompt` (usado por `ask`)
  também é **privado**.
- `ai::build_provider` é `#[cfg(feature="embedded")]` → **ausente no wasm**.

**Consequência de desenho (registrar em ADR-0025):** o **corpo do request do provedor**
(JSON `generateContent` do Gemini) e a **extração da resposta crua** (`candidates[].
content.parts[].text`) **NÃO podem** usar `gemini_body`/`gemini_extract` (privados) →
são feitos em **TS** no `fetch` (transporte = infra, exatamente o que ADR-0023 **D2**
atribui ao `fetch`). Isso é **transporte, não anti-alucinação** → **sem drift de
anti-alucinação**: as invariantes anti-alucinação (cited_text do store, **montagem de
prompt** e **citação**) ficam **em Rust `ai-pure`** (zero drift nativo↔web). Se o
Executor concluir que body/extract **precisam** ser Rust (drift de transporte
multi-provedor) → isso exige **novo PR ao core** para tornar `*_body`/`*_extract`
`pub` ⇒ **`blocked`+HALT com o achado** (fora do escopo desta task).

## Passos sugeridos

### 1) Fronteira web (Rust, `core/src/lib.rs`, app-side — NÃO toca the-light)
Adicionar **1–2 funções `#[uniffi::export]`** que usam **só a superfície `pub` do
`ai-pure`** (disponível nos dois alvos ⇒ **corpo cfg-free**, o que maximiza o zero
drift; não precisa de `cfg` porque não tocam store/`rusqlite`/`reqwest`):

- **Record novo** `AiWebRequest { reference: Reference, cited_text: String, system:
  String, user: String, provider: String, model: String }` (tipos puros → todos os alvos).
- **`ai_web_prepare(reference: String, question: String, provider_name: String,
  model: Option<String>, lang: String, verses: Vec<AiVerseInput>) -> Result<AiWebRequest,
  CoreError>`** onde `AiVerseInput { number: u16, text: String }` (o **texto vem do
  store web** — anti-alucinação; a fronteira **não** lê DB no wasm):
  1. `reference::parse_reference(&reference)` (canônica PT/EN);
  2. `lang.parse::<Lang>().unwrap_or(Lang::Pt)`;
  3. `cited_text = ai::numbered_verses(verses.iter().map(|v| (v.number, v.text.as_str())))`
     — **verbatim do store**, numerado pelo Rust (MESMA fn do nativo);
  4. `label = reference::format_reference(&ref, lang)`; `context = ai::ask_context(&label,
     &cited_text, &[])`;
  5. `model = model.filter(|m| !m.trim().is_empty()).unwrap_or_else(|| ai::default_model(&provider_name).to_string())`;
  6. **system+user com ZERO drift** via um `CaptureProvider` local que **implementa
     `ai::LlmProvider`** (o `ask_user_prompt` é privado, então captura-se pela rota
     pública `ask`): `complete(&self, system, user)` grava `system`/`user` (ex.
     `Cell`/`RefCell`) e retorna `Ok(String::new())`; então
     `let _ = ai::ask(&cap, &question, &context, lang);` popula os prompts EXATOS que o
     nativo enviaria; extrair `(system, user)`. (`name()`/`model()` devolvem
     `provider_name`/`model`.)
  7. devolver `AiWebRequest { reference: ref.into(), cited_text, system, user,
     provider: provider_name, model }`.
- **`ai_web_finalize(reference: String, cited_text: String, provider: String, model:
  String, interpretation: String) -> Result<AiAnswer, CoreError>`**: re-parseia a
  `reference` (canônica) e aplica `citation::rewrite_anchors(&interpretation,
  &HashSet::new())` (limpeza de âncoras de citação inválidas — **a etapa de citação
  anti-alucinação em Rust**, mesma impl do nativo; no `ask` simples o set é vazio →
  no-op seguro). Monta o `AiAnswer { reference, cited_text, interpretation: limpa,
  provider, model }` (cited_text do store **separado** da interpretation do LLM).
- **Regenerar bindings web:** `./scripts/gen-bindings-web.sh` (as funções `aiWebPrepare`/
  `aiWebFinalize` + o Record `AiWebRequest`/`AiVerseInput` presentes em
  `app/web/generated/`). Regenerar também os **nativos** se necessário (sem regressão).
- **Nativo:** as funções novas compilam iguais no nativo (ai-pure ⊂ embedded) — permite
  um **teste de host Rust** opcional que prova `cited_text`/`system`/`user` idênticos
  ao caminho de `ask_anchored` (parity check). `ask_anchored[_stream]` **nativos
  intactos**.

### 2) Glue web `app/web/reading.web.ts` — destubar `askAnchored`
`askAnchored(dbPath, translation, reference, question, provider, key, model, lang)`:
1. `parseReference(reference)` (wasm) → book/chapter/verse;
2. abrir `openReadingDbWeb()`, `queryChapter(handle, translation, book, chapter)` e
   **filtrar** o(s) versículo(s) da referência → `verses: Vec<{number,text}>` (reusa o
   SELECT existente — **sem novo SQL**; texto do **store**);
3. `aiWebPrepare(reference, question, provider, model, lang, verses)` (wasm) →
   `{ citedText, system, user, provider, model }`;
4. **transporte TS (`fetch`)**: montar o corpo do provedor a partir de `{system, user,
   model}`. **MVP = Gemini** (validado na F2.6): `POST https://generativelanguage.
   googleapis.com/v1beta/models/{model}:generateContent`, header **`x-goog-api-key: <key>`**
   (a chave **nunca** na URL/log), body `{ system_instruction:{parts:[{text:system}]},
   contents:[{role:"user",parts:[{text:user}]}] }`;
5. **extração TS**: da resposta crua, `candidates[0].content.parts[].text` concatenado →
   `interpretation` (espelha `gemini_extract`, que é privado — ver ACHADOS);
6. `aiWebFinalize(reference, citedText, provider, model, interpretation)` (wasm) →
   `AiAnswer`; retornar.
- **`askAnchoredStream` web:** **não-streaming primeiro** (documentar) — chamar o
  caminho de `askAnchored` e emitir a `interpretation` inteira **1×** via `onToken`
  (streaming web via `ReadableStream`/SSE fica registrado como follow-up). Priorizar o
  **não-streaming funcional**.
- **Provider `"mock"` no web:** aceitar como caminho determinístico (o teste usa **fetch
  MOCK**, não o provedor real); a chave pode ser um valor dummy (nunca logado).

### 3) Política de chave WEB (ADR-0025) — `app/lib/keystore.web.ts` → cofre in-memory de sessão
- Trocar o stub por um **cofre in-memory de SESSÃO**: um `Map<string,string>` de módulo
  (vive só na aba/sessão; **perdido no reload** → re-inserir a cada visita). `setKey`
  (valida provedor via `isSupportedProvider`, `trim`, rejeita vazia), `getKey` (valor da
  sessão ou `null`), `deleteKey` (idempotente), `listProviders` (só **NOMES** com chave).
  Interface `Keystore`/`createKeystore` **idêntica** ao nativo.
- **NUNCA** `localStorage`/`sessionStorage`/`IndexedDB`/git/log. **Justificativa
  (D3/ADR-0023):** login-de-conta OAuth foi rejeitado e persistir segredo em
  `localStorage` é inseguro → session-only in-memory é a opção web mais segura (sem
  superfície persistente de exfiltração). Offline-first: sem chave/sessão, o resto do
  app segue 100% offline; a IA web é **opt-in** e só faz rede no `fetch`.

### 4) UI `ReaderAskPanel` no web
- O componente é compartilhado; no web, `askAnchoredStream`/`getKey`/`listProviders`
  resolvem para os `.web.ts`. Garantir que o painel **funciona no web** pelo novo
  caminho (citado/interpretação já separados da F2.5).
- **Fluxo mínimo de chave na sessão:** um campo/ação para **colar a chave 1× por
  sessão** (chama `setKey` do keystore web in-memory) **sem persistir** — sem re-inserir
  dentro da sessão, re-inserir a cada visita. (Pode ser um input no próprio painel,
  visível no web para provedor real.)

### 5) ADR-0025
Registrar em `DECISIONS.md`: **política de chave web = session-only/in-memory** +
**padrão prepare/fetch/extract** (prompt/citação em Rust `ai-pure`; body/extract em TS
porque `*_body`/`*_extract` são privados) + o achado (build/extract Rust exigiria PR ao
core). Depende de ADR-0024 (ai-pure), ADR-0023 (D2/D3), ADR-0011 (infra TS no web).

## Restrições (regras rígidas relevantes)

- **NÃO tocar o `the-light`** (só consumir o `ai-pure` **pinado** em `c8ecb2f`);
  **NÃO alterar `core/Cargo.toml`** (re-pin já feito). A fronteira web nova vive
  **só** em `core/src/lib.rs`.
- **Anti-alucinação com ZERO drift:** montagem de prompt + citação vêm do **MESMO
  Rust `ai-pure`** (`numbered_verses`/`ask_context`/`ask` via `CaptureProvider` +
  `rewrite_anchors`) no web e no nativo. `cited_text` **sempre do store**; o LLM só
  interpreta.
- **Chave WEB session-only/in-memory**, **NUNCA** `localStorage`/`sessionStorage`/
  `IndexedDB`/git/log. **Rede só no `fetch` opt-in** com a chave (a **única** rede em
  runtime); sem chave, o app segue offline.
- **Subset A1** (`reading-sample.sqlite`, ~4,4 MB) — o texto do versículo no web vem
  daqui (F1.13), como as demais paridades web.
- **Prova SEM rede/chave reais:** `fetch` MOCK no teste headless. **NUNCA** chamar
  Gemini/Claude/GPT real no CI/loop.
- **Se a fronteira web exigir algo do core NÃO disponível como `pub` em `ai-pure`**
  (ex.: `gemini_body`/`gemini_extract` para body/extract em Rust) → **`blocked`+HALT
  com o achado** (seria **outro PR ao core** para torná-los `pub`; fora do escopo).
- **Qualidade:** Rust `fmt`/`clippy -D warnings`/`test`; TS `tsc --noEmit` + `expo
  export web`. Verde antes do commit. Em conflito com qualquer regra: **HALT**.
