## Objetivo

Criar o **projeto Expo mínimo** (React Native + TypeScript, **Expo SDK 56**, com
**`expo-router`**) dentro do diretório **`app/`**, contendo **UMA tela** (rota) que
tenha um **campo de input** (`TextInput`) e uma **área de resultado** (um `Text`
identificável por `testID="result"`) exibindo apenas um **placeholder estático**.

Esta tarefa entrega o **esqueleto da UI e a prova de que o app compila de forma
não-interativa**. Ela **NÃO** liga nada ao core/bindings — digitar no input **não**
chama `parse_reference` ainda. A ligação real ao core é de outras tarefas:
**F0.6 (web/WASM)**, **F0.7 (iOS)**, **F0.8 (Android)**. Manter esse escopo é
parte do critério de aceite (guarda de escopo).

## Contexto / arquivos

**Onde o app vive — `app/` com `package.json` PRÓPRIO (crítico):**
- O projeto Expo é o diretório **`app/`** da árvore-alvo (seção 1 do
  `IMPLEMENTATION_PLAN.md`). Ele tem o **seu próprio `package.json`** (`app/package.json`)
  e o **seu próprio `package-lock.json`** (`app/package-lock.json`), **independentes**
  do `package.json` da RAIZ.
- O `package.json` da **RAIZ** existe desde a **F0.4** apenas para o devDependency
  do `ubrn` (`uniffi-bindgen-react-native`, ver ADR-0004) e **não** é o manifesto do
  app. Não há workspace nem linkagem entre os dois — são projetos npm separados em
  diretórios distintos.
- **Todos** os comandos de Expo/TypeScript desta tarefa rodam **de dentro de `app/`**
  (`cd app && …`). Não rode `expo`/`tsc` na raiz.

**Estado atual de `app/` (criado vazio na F0.1, commit `595c70c`):**
- Já existem os diretórios da árvore com placeholders `.gitkeep`:
  `app/app/` (rotas do `expo-router`), `app/components/`, `app/lib/`,
  `app/web/`, `app/modules/the-light-core/`. Você vai **popular** `app/` com o
  projeto Expo real. Os `.gitkeep` podem ser mantidos ou removidos onde já houver
  arquivos reais — indiferente para o aceite.
- **Atenção à colisão de nomes (esperada, não é erro):** o projeto Expo é `app/` e,
  **dentro** dele, as rotas do `expo-router` ficam em `app/app/` (o `app/` interno).
  A tela inicial é, portanto, **`app/app/index.tsx`** (com um **`app/app/_layout.tsx`**
  para o `expo-router`). Isso bate exatamente com a árvore do plano
  (`app/ ├── app/ # rotas (expo-router)`).

**Ambiente (F0.0 / DECISIONS.md — tabela de ambiente):**
- **Expo SDK 56** (`expo@56.0.12` resolvido via npm). Node **v25.8.1**, npm **11.11.0**.
- **Xcode completo AUSENTE** (só Command Line Tools) e **Android SDK/NDK AUSENTES**
  → portanto os **RUNs nativos** (`expo run:ios` / `expo run:android`) **NÃO** fazem
  parte desta tarefa: estão **adiados para F0.7/F0.8**, que tratam justamente da
  ligação nativa e dependem desses toolchains. **Não** exija toolchain nativo aqui.
- Por que **não** usar `expo start --web`: é um **servidor de dev interativo** que
  **não encerra**, impróprio para verificação headless/automatizada. A porta dura
  desta tarefa é o **export estático não-interativo** (abaixo).

**O que é GERADO/IGNORADO vs. VERSIONADO (leia com atenção — Executor e Reviewer):**
- O `.gitignore` da F0.1 ignora globalmente `node_modules/`, `.expo/` e `dist/`.
  **Verificado:** `app/node_modules/`, `app/.expo/` e `app/dist/` são **ignorados**
  por esse `.gitignore` (não anclados → casam em qualquer subdiretório). Portanto o
  **bundle web** (`app/dist/`) e as **deps** (`app/node_modules/`, `app/.expo/`)
  **NÃO** são versionados — e a verificação de "bundle compila" checa **exit 0 + a
  existência de `app/dist/` no FILESYSTEM**, **não** no git.
- **O que É versionado/committado por esta tarefa:** `app/package.json`,
  `app/package-lock.json`, o config do app (`app/app.json` ou `app/app.config.*`),
  `app/tsconfig.json`, o código das rotas/telas (`app/app/_layout.tsx`,
  `app/app/index.tsx`), e — se você optar por isolar artefatos do Expo — um
  `app/.gitignore` próprio. Se o `expo-router` gerar um `app/expo-env.d.ts`
  (referência de tipos), **garanta que o working tree fique limpo**: ou
  **committe** esse arquivo (é determinístico e sem segredos), ou **ignore-o** via
  `app/.gitignore`. Não deixe nada gerado solto fora desses dois tratamentos.

## Passos sugeridos

1. **Scaffold do app Expo em `app/` (SDK 56, expo-router).** Como `app/` já contém
   placeholders, prefira um dos caminhos:
   - **(a) Template + merge:** scaffold com o template `expo-router` num diretório
     temporário (ex.: `npx create-expo-app@latest tmp-app --template`) e **mova** os
     arquivos para `app/`, ajustando a versão do `expo` para a **linha SDK 56**; ou
   - **(b) Manual mínimo:** crie à mão `app/package.json` (com `"main":
     "expo-router/entry"`, `expo` SDK 56, `expo-router`, `react`, `react-native`),
     `app/app.json` (com `"web": { "bundler": "metro" }` e o plugin do `expo-router`),
     `app/tsconfig.json` (estendendo `expo/tsconfig.base`), e as rotas.
   Em ambos, **fixe o `expo` na linha SDK 56** (ex.: `"expo": "~56.0.0"` ou o exato
   `56.0.12`) e use `npx expo install` para alinhar as versões das libs ao SDK.
2. **Instalar deps do app** a partir de `app/`: `npm install`. Garanta também o
   necessário para o **export web** com `expo-router`:
   `npx expo install react-dom react-native-web @expo/metro-runtime` (e
   `typescript`/`@types/react` como devDeps, para o `tsc`). Rede em **dev/build é
   permitida** (offline-first é regra de **runtime do produto**, não do
   desenvolvimento). Isso cria `app/node_modules/` (ignorado) e
   `app/package-lock.json` (versionado).
3. **Criar a UMA tela.** `app/app/_layout.tsx` (Stack do `expo-router`) e
   `app/app/index.tsx` com:
   - um **`<TextInput>`** controlado por `useState` (o valor digitado fica no estado
     local — **sem** chamar o core);
   - uma **área de resultado**: um `<Text testID="result">` exibindo um
     **placeholder estático** (ex.: "Resultado aparecerá aqui" / eco do texto
     digitado). **NÃO** importe nem chame bindings/core (`@ubjs/core`,
     `the_light_app_core`, `parse_reference`, `app/modules/the-light-core`, etc.).
4. **Type-check** a partir de `app/`: `npx tsc --noEmit` deve sair **limpo**. (Se o
   `expo-router` exigir `app/expo-env.d.ts`, ele é gerado pelos comandos do Expo —
   o passo de export abaixo o regenera; trate-o no `.gitignore`/commit conforme o
   Contexto.)
5. **Provar o bundle web de forma NÃO-INTERATIVA** a partir de `app/`:
   `npx expo export --platform web`. Esse comando **produz `app/dist/` e ENCERRA**
   (sai 0) — **não** abra `expo start` (interativo) e **não** rode
   `expo run:ios`/`expo run:android` (exigem Xcode/NDK, adiados p/ F0.7/F0.8).
   Confirme `exit 0` e a existência de **`app/dist/`** no filesystem.
6. **Garantir o working tree limpo** respeitando o `.gitignore`: `app/node_modules/`,
   `app/.expo/`, `app/dist/` são ignorados; versione só os arquivos do app listados
   no Contexto. Trate `app/expo-env.d.ts` (commit OU `app/.gitignore`).
7. **Commit com prefixo do ID**, ex.:
   `feat(F0.5): app Expo mínimo (expo-router) com tela input + resultado`.

## Critério de aceite

- [ ] **`app/package.json` existe** e declara `expo` na **linha SDK 56** (versão
  contém `56`) **e** `expo-router` (entre `dependencies`/`devDependencies`).
- [ ] Existe **ao menos uma rota/tela** (em `app/app/`, ex.: `app/app/index.tsx`)
  contendo um **`TextInput`** e uma **área de resultado** identificável por
  **`testID="result"`** com placeholder estático.
- [ ] **`npx tsc --noEmit`** rodado **de dentro de `app/`** sai **limpo** (exit 0).
- [ ] **`npx expo export --platform web`** rodado **de dentro de `app/`** sai **0** e
  cria **`app/dist/`** no **FILESYSTEM** (não precisa estar no git — é ignorado).
- [ ] **Guarda de escopo:** a tela/rota **NÃO** importa nem referencia o
  core/bindings (`the_light_app_core`, `@ubjs`, `the-light-core`,
  `modules/the-light-core`, `parse_reference`/`parseReference`) — isso é F0.6/7/8.
- [ ] **Working tree limpo** após o commit, **considerando o `.gitignore`**:
  `app/node_modules/`, `app/.expo/`, `app/dist/` (e, se ignorado, `app/expo-env.d.ts`)
  não contam; só os arquivos **VERSIONADOS** do app foram committados.
  `git status --porcelain` vazio.

## Comando(s) de verificação

```sh
set -e
ROOT="$(git rev-parse --show-toplevel)"
APP="$ROOT/app"
cd "$APP"

# 0) SETUP: instalar deps do app se ainda não houver node_modules (rede em dev OK).
if [ ! -d "$APP/node_modules" ]; then
  if [ -f "$APP/package-lock.json" ]; then npm ci; else npm install; fi
fi

# 1) app/package.json declara expo (SDK 56) + expo-router
node -e '
const p=require("./package.json");
const d=Object.assign({},p.dependencies,p.devDependencies);
if(!d.expo){console.error("FALTA dep expo");process.exit(1)}
if(!/(^|[^0-9])56([.\-]|$)/.test(String(d.expo))){console.error("expo nao e SDK 56: "+d.expo);process.exit(1)}
if(!d["expo-router"]){console.error("FALTA dep expo-router");process.exit(1)}
console.log("OK  expo "+d.expo+" + expo-router "+d["expo-router"]);
'

# 2) Ao menos uma rota/tela com TextInput + area de resultado (testID="result")
SCREEN="$(grep -rlE "TextInput" "$APP/app" --include="*.tsx" | head -1 || true)"
test -n "$SCREEN" || { echo "FALHA nenhuma tela com TextInput em app/app"; exit 1; }
grep -qE 'testID=["'"'"']result["'"'"']' "$SCREEN" \
  || { echo "FALHA a tela $SCREEN nao tem area de resultado (testID=\"result\")"; exit 1; }
echo "OK  tela com TextInput + area de resultado: ${SCREEN#$ROOT/}"

# 3) GUARDA DE ESCOPO: a tela NAO importa/usa o core/bindings ainda (isso e F0.6/7/8)
if grep -rnE "the_light_app_core|@ubjs|the-light-core|modules/the-light-core|parse_?[Rr]eference" \
     "$APP/app" "$APP/components" "$APP/lib" "$APP/web" \
     --include="*.ts" --include="*.tsx" 2>/dev/null; then
  echo "FALHA escopo violado: a UI referencia o core/bindings (isso e F0.6, nao F0.5)"; exit 1
fi
echo "OK  nenhuma referencia ao core/bindings (escopo F0.5 respeitado)"

# 4) BUNDLE WEB NAO-INTERATIVO: expo export gera app/dist/ e ENCERRA (exit 0).
#    NAO usar expo start (interativo) nem run:ios/run:android (Xcode/NDK ausentes).
rm -rf "$APP/dist"
npx expo export --platform web \
  && echo "OK  expo export --platform web saiu 0" \
  || { echo "FALHA expo export --platform web nao compilou"; exit 1; }
test -d "$APP/dist" \
  && echo "OK  app/dist/ criado no filesystem" \
  || { echo "FALHA app/dist/ nao foi criado"; exit 1; }

# 5) Type-check limpo (usa o tsconfig do app). Roda apos o export, que gera os
#    tipos do expo-router (ex.: expo-env.d.ts) caso necessarios.
npx tsc --noEmit \
  && echo "OK  tsc --noEmit limpo" \
  || { echo "FALHA tsc --noEmit reportou erros"; exit 1; }

# 6) Working tree limpo: node_modules/.expo/dist (e expo-env.d.ts, se ignorado) sao
#    ignorados por design; so os arquivos VERSIONADOS do app foram committados.
cd "$ROOT"
test -z "$(git status --porcelain)" \
  && echo "OK  working tree limpo" \
  || { echo "FALHA working tree sujo:"; git status --porcelain; exit 1; }

echo "F0.5 verificada."
```

## Restrições

- **NÃO ligar ao core/bindings nesta tarefa.** A UI deve ser puramente local
  (estado React), sem importar `@ubjs/core`, `the_light_app_core`,
  `app/modules/the-light-core` nem chamar `parse_reference`. A ligação real é
  **F0.6 (web)**, **F0.7 (iOS)**, **F0.8 (Android)**. Violar isso reprova o aceite.
- **NÃO exigir toolchain nativo.** `expo run:ios` e `expo run:android` **estão fora
  do escopo** (Xcode completo e Android SDK/NDK AUSENTES — F0.0/DECISIONS.md) e
  pertencem a F0.7/F0.8. A verificação usa **somente** `expo export --platform web`
  (não-interativo) + `tsc`.
- **NÃO usar `expo start`** como porta de verificação (servidor interativo que não
  encerra). A prova de compilação é o **export estático** que **sai 0**.
- **`app/` é um projeto npm independente da RAIZ.** Rode `expo`/`tsc`/`npm` **dentro
  de `app/`**; não misture com o `package.json`/`package-lock.json` da raiz (que são
  do `ubrn`, ADR-0004). Mantenha os dois lockfiles separados.
- **NÃO modificar / forkar / copiar o `the-light`** nem a fronteira `core/`/`bindings/`.
  Esta tarefa é só UI Expo; o core não é tocado.
- **Rede só em dev/build.** Instalar deps do Expo via npm e baixar o template com
  rede é **permitido** (offline-first é regra de **runtime do produto**). Nenhuma
  tela pode exigir rede em runtime; nenhuma chamada de rede deve ser introduzida.
- **Reporte `blocked` (com o ERRO EXATO), sem improvisar:** se o scaffolding do Expo
  / pin do SDK 56 **não instalar**, ou o `expo export --platform web` **não compilar**
  por motivo de **ferramenta/ambiente** (incompatibilidade do Node v25.8.1 com a
  toolchain do SDK 56, falha do Metro/web, dep ausente irresolúvel, etc.), reporte
  **`blocked`** com a saída literal do `npm`/`expo`/`tsc` — **NÃO** crie um app falso,
  não maquie o export, não comente a verificação para "passar".
- **Working tree limpo respeita o `.gitignore`:** `node_modules/`, `.expo/`, `dist/`
  (e `app/expo-env.d.ts`, se você optar por ignorá-lo) não são versionados; versione
  apenas o código/manifestos do app. **Segredos nunca em git/log** (nada de
  `.env`/tokens/chaves).
