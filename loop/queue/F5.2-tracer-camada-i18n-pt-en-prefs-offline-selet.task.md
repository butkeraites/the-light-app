## Restrições

PRIMEIRA a semear (entry). Offline-first: detecção e persistência são 100% locais (sem rede). NÃO introduzir dependência nova (i18next/AsyncStorage/expo-localization) — molde dependency-free do theme.ts (ADR-0015) + backend injetável do keystore.ts; se o executor julgar uma dep inevitável → HALT/ADR, não improviso. Anti-alucinação: i18n NUNCA traduz texto de versículo (do store); só cromo de UI. O KV de prefs criado aqui é REUTILIZADO por theme-persist e é o alicerce de todo o workstream. Registrar a arquitetura de i18n (catálogo tipado dependency-free + prefs offline + detecção de locale) no próximo ADR livre (confirmar em DECISIONS.md; provavelmente ADR-0036 — pode colidir com workstreams irmãos, escolher o próximo realmente livre).

- **Offline-first:** capacidade essencial nunca exige rede/conta; recurso novo liga OFF/local por padrão.
- **BYOK/segredos:** chaves/tokens nunca em git/log; secure-store nativo / session-only web.
- **Anti-alucinação:** texto de versículo sempre do store local (Rust); i18n NUNCA traduz versículo; atribuições CC-BY (OpenBible/STEP) VISÍVEIS e verbatim.
- **the-light só via PR+ADR:** consumir `the-light-core` pinado (`2fc2dab`); nenhuma mudança no core nesta tarefa (senão vira PR+ADR gate à parte).
- **Qualidade:** Rust fmt/clippy -D warnings/test; TS `tsc --noEmit`; grafo wasm puro; prova determinística/headless (sem chave/rede real no loop).
