## Objetivo

Turn the measured metric from on-disk bytes into over-the-wire transfer bytes: precompress the large static assets (frontier wasm, wa-sqlite wasm, subset DBs) with gzip and brotli during export, extend measure-web-bundle.sh to report compressed transfer sizes, and document the immutable-hashed caching/serving strategy (assets are content-hashed, so far-future cache headers are safe).

> **Herda F5.6 (wasm release) + F5.3 (métrica; a baseline já grava raw+gzip).** Pré-comprime os
> assets grandes (`.wasm` 1,14MB, `reading-lite.sqlite` 4,53MB, `lexicon-sample.sqlite` 9,5MB) e
> define a estratégia de cache/serving p/ o alvo web, medindo o TRANSFER (over-the-wire). Zero-drift:
> o descomprimido é byte-idêntico. Realista sobre o que o `expo export`/serving estático suportam
> (se pré-comprimir gzip/br + servir com `Content-Encoding` não for suportável no static host do
> Expo sem servidor, registre honestamente o que é aplicável e meça o transfer size mesmo assim).

## Restrições

Depends on rp-2 so the compressed number reflects the release/opt wasm. Compression is a measurement + build-artifact concern; the loop cannot dictate the production web server, so document the assumption rather than asserting live headers. brotli may be absent — guard it like wasm-opt.

- **Offline-first:** os assets pré-comprimidos continuam LOCAIS (servidos do próprio app/asset dir); a estratégia de cache/serving NÃO introduz rede/servidor externo. O app funciona offline.
- **Zero-drift:** pré-compressão (gzip/br) NÃO muda o conteúdo descomprimido — o `.wasm`/`.sqlite` descomprimido é byte-idêntico. Self-tests idênticos.
- **the-light só via PR+ADR:** `the-light-core` pinado (`2fc2dab`); app-side (build/serving).
- **Qualidade:** `tsc --noEmit`; expo export web 0; self-tests sem regressão; medir e registrar o TRANSFER size (over-the-wire, gzip/br) além do raw; estender a métrica da F5.3 se aplicável.
