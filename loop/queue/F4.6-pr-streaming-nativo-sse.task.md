### Formatos de delta por provedor (a CONFIRMAR byte-a-byte na fonte antes de implementar; espelham o mapeamento web já aceito em ADR-0034)

- **Anthropic** (SSE): eventos `content_block_delta` → `data: {"delta":{"type":
  "text_delta","text":<delta>}}` → emitir `delta.text`. (`stop_reason=="refusal"`→erro.)
- **OpenAI** (SSE): `data: {"choices":[{"delta":{"content":<delta>}}]}` + sentinela
  `data: [DONE]`.
- **Gemini** (SSE, endpoint `:streamGenerateContent?alt=sse`): `data:
  {"candidates":[{"content":{"parts":[{"text":<delta>}]}}]}`.
- **Ollama** (**NDJSON**, não-SSE): 1 objeto JSON por linha, SEM prefixo `data:` —
  `{"message":{"content":<delta>},"done":<bool>}` → emitir `message.content` por linha.

## Restrições

- **BLOQUEANTE (gate:true) — toca o `the-light` (via PR + ADR).** O executor **NÃO**
  modifica o `the-light` sozinho: prepara a proposta/diff + ADR-0035 e o Driver

- **Não-quebrante:** o **default** de `complete_stream` (`mod.rs:378`) permanece
  (mock e provedores sem override); `complete` (não-streaming) intacto; `default =
  ["embedded"]` byte-a-byte; API pública nativa preservada. Escopo do PR = SÓ
  `ai/{mod.rs,providers.rs}` (+ testes do core).
- **Anti-alucinação ZERO-DRIFT:** o streaming muda **só o TRANSPORTE nativo**; a
  interpretação final (concat dos deltas) é IDÊNTICA ao `complete`; `cited_text`/
  citações continuam do store/`ai-pure`; a fronteira aplica a MESMA finalização. Os
  deltas são **só** texto da interpretação do modelo — **nunca** texto bíblico.
- **wasm PURO:** streaming real é `embedded`-only (`reqwest`) → o override fica sob
  `#[cfg(feature="embedded")]`; **nada** entra no grafo `ai-pure`/wasm; o web segue
  com o transporte TS (F4.1/F4.2). O trait/`complete_stream` default permanece puro.
- **BYOK:** a chave vai **só no header** do provedor (anthropic `x-api-key`; openai
  `authorization: Bearer`; gemini `x-goog-api-key`; ollama sem chave), **nunca** na
  URL/log/git. Prova **determinística por MOCK/parser puro** (sem rede/chave no CI);
  o LLM/streaming real é validação humana (F4.5, já aceita).
- **Offline-first:** streaming é opt-in e só melhora o transporte de uma chamada de
  IA que já é opt-in; nenhuma capacidade essencial passa a exigir rede.
