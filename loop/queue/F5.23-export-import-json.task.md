## Objetivo

Fundação da trilha de sync (ADR-0036): um **snapshot JSON máquina-legível** dos dados do
usuário (notas + marcações + progresso de plano), **round-trippável** — exportar e importar
(com merge) de volta. Todos os alvos. Base p/ o Google Drive (F5.24–25). NÃO sincroniza
sessões de IA (privacidade), banco bíblico nem chaves.

> **Herda F1.11 (`notesExport.buildNotesExport` = Markdown p/ Share) + F5.10 (`plans-fs`).** Esta
> tarefa adiciona um formato JSON round-trippável (o Markdown é só p/ leitura humana). Monta o
> snapshot dos RECORDS (`list_notes`/`list_highlights` + o progresso de plano) — NÃO reimplementa
> o formato do store. O IMPORT faz merge determinístico (união por referência p/ notas/marcações,
> LWW p/ progresso).

## Passos sugeridos
1. **Formato do snapshot** (`app/lib/userdataSnapshot.ts` novo, puro): `{ version, exportedAt?, notes: Note[], highlights: Highlight[], planProgress: {plan_id,start_date,completed}|null }` montado dos Records (sem reimplementar o store).
2. **Export:** fn que monta o snapshot JSON a partir de `list_notes`/`list_highlights` + progresso; a UI (F5.26) fará o Share/save. Determinístico (ordem estável).
3. **Import + merge determinístico:** parse + valida (versão, tipos); merge: notas/marcações = união por `reference` (colisão → o de `modifiedAt`/mais recente vence, ou o importado se sem timestamp — decidir e documentar); progresso = LWW (ou `max(completed)` se mesmo plano). Aplica via as fns de escrita do core (não reescreve o store à mão). Idempotente (importar o mesmo snapshot 2× = no-op).
4. **Prova headless** (molde `*-selftest`/`test:web:*`, sem rede): export → snapshot com N notas/M marcações/progresso; round-trip (export → limpar → import → estado idêntico); merge (dois snapshots divergentes → união determinística); import inválido rejeitado sem corromper.
