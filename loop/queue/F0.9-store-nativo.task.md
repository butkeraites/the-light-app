## Objetivo

Provar a **camada de store NO NATIVO** (iOS/Android): expor na fronteira UniFFI
uma função `get_passage` / `getPassage` que **lê uma passagem** (ex.: `"John 3:16"`)
de um **banco SQLite pequeno (subset KJV)** delegando ao **store do `the-light-core`**
(`rusqlite`, feature `embedded`, ativa só no nativo — `cfg(not(wasm32))`), e
retorna o **texto do versículo + a referência** num **tipo UniFFI Record**.

Regra inegociável (anti-alucinação): o texto do versículo vem **sempre do store
local** (`sample.sqlite`), verbatim de fonte de **domínio público (KJV)**. A
fronteira **NÃO** reimplementa SQL nem parsing — **delega** ao core (uma fonte da
verdade): `parse_reference` (já existe) + `the_light_core::store`/`source`.

A prova **obrigatória e determinística** é um **teste Rust de host** (`cargo test`)
que abre o `sample.sqlite` e assere `get_passage("John 3:16") == <texto KJV
verbatim>` com `book=43, chapter=3, verse=16`. O run nativo em sim/emulador
(estilo F0.7/F0.8) é verificação **adicional e opcional** (documentada se feita).

## Restrições

- **NÃO modificar nem forkar `the-light`** (`/Users/butkeraites/Documents/the-light`
  é leitura-apenas; consumo é git dep pinada `rev 8f66004`, ADR-0002). Qualquer
  necessidade de mudar o core ⇒ **`blocked`** com o erro EXATO (PR + ADR = decisão
  humana), **não** improviso.
- **NÃO reimplementar SQL nem parsing no app.** A passagem é lida **delegando** ao
  `the_light_core::store`/`source` (uma fonte da verdade). O schema do
  `sample.sqlite` vem das **migrações do core**, não de SQL de schema à mão.
- **Anti-alucinação:** o texto do versículo vem **do store local** (`sample.sqlite`),
  verbatim. **Nunca** o LLM/código gera texto bíblico.
- **Licenciamento:** somente **KJV / domínio público** no sample. NÃO embarcar
  texto protegido/não-livre. Registrar origem/licença em ADR-0010.
- **`cfg`-gating obrigatório:** a função `get_passage` e tudo que toca
  `store`/`source`/`rusqlite` deve ser `#[cfg(not(target_arch = "wasm32"))]` — o
  build web/wasm (F0.6b) **deve permanecer puro** (sem `rusqlite`). Store **web** é
  F0.10 (`wa-sqlite`+OPFS), **fora deste escopo**.
- **Offline-first:** nenhuma rede no runtime do produto. Rede só em dev/build
  (resolver deps/cargo). **Segredos nunca** em git/log.
- **Escopo:** F0.9 prova a **leitura via store do core** (teste Rust). O
  **embarque** do banco no app nativo (bundling iOS/Android) e o run nativo são
  fase posterior / verificação adicional opcional — não bloqueiam o aceite.
- **Bloqueio legítimo (`blocked`) com erro EXATO** se: o store do core não for
  consumível sem mudar o core/fronteira de forma proibida; o `embedded` não
  compilar no host/nativo; ou a geração honesta do sample exigir dados protegidos.
  Nesse caso, `done/F0.9.result.md` com `status: blocked`, a saída literal do
  comando que falhou, e a decisão pendente → ADR-0010.
