## Objetivo

Build the native Reading Plans experience: a new expo-router route that lists the 3 plans, starts one, shows the day list with TODAY highlighted, lets the user open a day's chapters in the existing reader, and mark days complete with a streak/progress indicator. Wires rp-1 (generation) + rp-2 (progress). Native only; web parity = rp-4.

> `app/web/reading.ts`). Também reusa a i18n da F5.2/F5.5 (`useI18n`/`t()`). Esta tarefa é a
> **UI NATIVA** (rota expo-router): lista de planos → iniciar → dia de hoje → marcar concluído.
> A UI navega p/ `/read/${book}/${chapter}` (leitura já existente). NÃO reimplementar geração/
> progresso; só orquestrar as fns da fronteira. Paridade web = F5.10 (gate à parte).

## Restrições

Purely app-side (UI + glue + self-test), no core change. Reuse the reader route for day navigation (no new reading UI). Streak = consecutive completed days (compute from completed vs today_index; keep it simple/deterministic). Plan-day chapters are WHOLE chapters → navigate by ref.book/ref.chapter. Self-test proves the vertical on device without any UI screenshot dependency (marker-based, molde F1.11).

- **Offline-first:** UI 100% local (planos gerados no core, progresso em fs nativo via F5.4); nada exige rede/conta.
- **Anti-alucinação:** refs/dias vêm de `reading_plan_day` (core); nomes de plano do CATALOG; a UI navega p/ `/read/...` que lê o versículo do store. NÃO inventar conteúdo.
- **i18n/a11y:** strings de cromo via `t()` (reusa F5.2/F5.5); elementos interativos com `accessibilityRole`/label; datas/números localizados sem tocar texto bíblico.
- **the-light só via PR+ADR:** `the-light-core` pinado (`2fc2dab`); nenhuma mudança no core. A UI é NATIVA (planos = native-first; paridade web = F5.10, gate à parte).
- **Qualidade:** `tsc --noEmit`; self-test nativo (molde `run-ios-selftest.sh`/TLA_*) provando lista→iniciar→dia→marcar; sem regressão.
