## Objetivo

`app/app/index.tsx` esconde os links de abrir leitor (`/read`), busca (`/search`) e planos
(`/plans`) atrás de `Platform.OS !== 'web'` (~linhas 189/203/218), então usuários web só
alcançam esses recursos — que hoje FUNCIONAM no web — digitando a URL. Os três têm paridade
web (F1.13/F1.14 leitura+busca DESTUBADAS; F5.10/ADR-0050 planos; `dist/read`, `dist/search`,
`dist/plans` são emitidos). Renderizar os links no web e corrigir os comentários de gating
obsoletos que precedem essas conclusões. 100% offline, sem tocar o `the-light`, sem segredo.

## Aceite

- [ ] Na build web a home renderiza links navegáveis para `/read`, `/search` e `/plans`, e cada
      um alcança a rota web funcional; comportamento nativo inalterado.
- [ ] Os comentários obsoletos em `app/app/index.tsx` (~:201-202 e :215-217) e a nota "esta tela
      não é alcançada no web" em `app/app/search/index.tsx:11-13` são removidos/corrigidos para
      refletir a paridade web concluída.
- [ ] `npx expo export --platform web` sucede e a saída web contém os três links (verificar por
      inspeção/grep do bundle exportado ou por um check headless no branch de plataforma web).
- [ ] `npx tsc --noEmit` 0; `npm run test:web:perf-budget` verde (ou re-baseline de moduleCount
      documentado se a mudança adicionar módulos eager); the-light intocado (`225b8c9`); sem segredo.

## Verificação

```sh
cd app && npx tsc --noEmit && npx expo export --platform web >/dev/null && echo EXPO_OK && npm run test:web:perf-budget
```
