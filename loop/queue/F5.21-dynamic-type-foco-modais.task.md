## Objetivo

Fechar os eixos de a11y que exigem runtime: respeitar dynamic type / escala de fonte (allowFontScaling, sem clipping em tamanhos grandes), ordem de foco lógica, e semântica de modal nos painéis de leitura (accessibilityViewIsModal + foco inicial + fechamento por leitor de tela), provados por um self-test no device que emite um marcador TLA_A11Y e por checagem manual documentada.

> **ÚLTIMA task não-gate da Fase 5** (herda F5.20 a11y interativos). Garante: **dynamic type**
> (a UI de leitura respeita a escala de fonte do sistema; não trava `allowFontScaling`), **ordem de
> foco** lógica, e **semântica de modais** nos painéis de leitura (`accessibilityViewIsModal`, foco
> preso, anúncio ao abrir/fechar). Prova no device (`TLA_A11Y` via `run-ios-selftest.sh`) se o
> simulador estiver disponível; senão, prova headless honesta + relatar qual foi usada.

## Restrições

Última da largura de a11y; parte é runtime (device) então usa o padrão TLA_ + checagem manual documentada (não requer chave/rede — apenas o simulador). Não é gate: é app-side determinístico + manual documentado, sem sign-off humano de marco. O Marco 4/§F4.2 como um todo é avaliado separadamente pelo Guia.

- **Offline-first:** local; nada exige rede.
- **Anti-alucinação:** dynamic type/foco/modais são só UI; não tocar texto bíblico (o versículo escala com o tema/fonte, mas o conteúdo vem do store).
- **the-light só via PR+ADR:** `the-light-core` pinado (`2fc2dab`); app-side.
- **Qualidade:** `tsc --noEmit`; garantir que a UI de leitura respeita a escala de fonte do sistema (dynamic type / `allowFontScaling`), ordem de foco lógica e semântica de modais (`accessibilityViewIsModal`/foco preso/anúncio); prova no device (self-test `TLA_A11Y` via `run-ios-selftest.sh`) SE o simulador estiver disponível, senão prova headless honesta + relatar; self-tests sem regressão.
