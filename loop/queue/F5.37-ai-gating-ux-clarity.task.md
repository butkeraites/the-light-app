## Pedido (do humano)

"Preciso que seja claro para o usuário se essas funcionalidades só vão ser habilitadas quando
tiver uma chave de LLM configurada, para que ele não fique pensando que o aplicativo
simplesmente não funciona."

## Princípio a comunicar

- **Sempre offline (SEM chave):** leitura, busca (FTS5), referências cruzadas, notas/marcações,
  planos de leitura, temas, backup/export. Nunca devem parecer "quebradas" nem pedir conta.
- **Precisam de provedor de IA (BYOK):** Perguntar (Ask), Estudo Profundo, Comparar, Conversa
  (session). Precisam de uma chave de Claude/GPT/Gemini **ou** Ollama local. Sem isso, devem
  mostrar um call-to-action CLARO ("Configure um provedor de IA em Ajustes"), não um erro genérico.

## Escopo

1. **Auditar** os 4 painéis de IA (`ReaderAskPanel`/`AskPanel`, `ReaderStudyPanel`,
   compare/multi-provider, session/chat) e o fluxo de chave (`app/lib/keystore.ts` + a tela de
   config de chave): o que o usuário vê HOJE quando NÃO há chave/provedor? (grep por onde a
   ausência de chave vira erro/estado vazio.)
2. **Estado "sem provedor de IA" claro e acionável:** quando nenhum provedor está configurado (nem
   chave BYOK nem Ollama), cada recurso de IA mostra um aviso amigável via `t()`:
   "Este recurso usa IA — configure um provedor (Claude, GPT, Gemini) ou Ollama local em Ajustes"
   com um **link/botão para a tela de configuração de chave**. NÃO um erro cru nem uma tela vazia.
3. **Distinção visível offline vs IA:** deixar claro (na About/F5.35 e/ou nos pontos de entrada de
   IA) que leitura/busca/xref/notas/planos funcionam 100% offline sem conta; só a IA precisa de
   provedor. Reusar/estender a About (F5.35) se couber, sem duplicar.
4. **Confirmar que os recursos OFFLINE nunca gatilham o aviso de IA** (não introduzir gating onde
   não deve haver). Leitura/busca não dependem de chave.
5. Todo cromo novo via `t()` com paridade pt/en. a11y (role/label) no link/botão de config. Sem hex.

## Fora de escopo

- Validar chamada REAL a um provedor (isso é BYOK do usuário; nada de chave no loop/repo).
- Mudar a arquitetura de IA/anti-alucinação (só a MENSAGERIA/estado vazio).

## Aceite

- [ ] Sem provedor configurado, cada painel de IA (Ask/Study/Compare/Session) mostra um CTA claro
      via `t()` ("configure um provedor de IA em Ajustes") com link para a config de chave — nunca
      um erro genérico/tela vazia que pareça "app quebrado".
- [ ] Recursos offline (leitura/busca/xref/notas/planos) NÃO exibem aviso de IA e não pedem chave.
- [ ] Distinção offline-vs-IA visível ao usuário (About/F5.35 e/ou pontos de IA).
- [ ] `t()` paridade pt/en (`test:i18n`+`test:i18n-coverage`), a11y (`test:a11y-scan`), contraste
      (`test:web:contrast`) verdes; `tsc --noEmit` 0; `expo export --platform web` 0.
- [ ] Anti-alucinação intacta (cited_text/interpretation seguem separados; nada de texto bíblico via `t()`);
      the-light `225b8c9`; sem segredo.

