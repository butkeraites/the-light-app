## Objetivo

Na tela Home o `<Text>` de resultado (`app/app/index.tsx:166-173`) define
`accessibilityLabel={t('a11y.result')}` ("Interpretation result"/"Resultado da
interpretação"), que **sobrescreve** os filhos — então o texto resolvido de
passagem/referência (`resultText`) NUNCA é anunciado por tecnologia assistiva. Remover o
label que sobrescreve (preferido) ou compor `resultText` nele, de modo que o CONTEÚDO real
seja exposto. Os guardas F5.20/F5.21 só checam PRESENÇA de label, então este resíduo passou.
100% offline, sem tocar o `the-light`, sem segredo.

## Aceite

- [ ] Um leitor de tela anuncia o texto resolvido de passagem/referência na Home, não só
      "Resultado da interpretação" — corrigido removendo o `accessibilityLabel` que sobrescreve
      o `Text` de resultado (preferido) ou compondo `resultText` no label anunciado.
- [ ] `npm run test:a11y-scan` verde; `npx tsc --noEmit` 0.
- [ ] Sem regressão de cromo PT: `npm run test:i18n-coverage` verde; the-light intocado (`225b8c9`);
      sem segredo/rede.

## Verificação

```sh
cd app && npx tsc --noEmit && npm run test:a11y-scan && npm run test:i18n-coverage
```
