## Objetivo

Tornar o override de tema (claro/escuro) PERSISTENTE entre reinícios usando o mesmo KV de prefs offline criado no tracer (nativo=arquivo; web=localStorage), fechando a lacuna 'persistência entre reinícios é futura' do ADR-0015, sem dependência nova.

> **Herda F5.2 (KV de prefs offline).** O modo de tema (claro/escuro/sistema) hoje NÃO
> persiste entre reinícios (lacuna registrada no ADR-0015). Esta tarefa persiste a escolha no
> MESMO KV da F5.2 (`app/lib/prefs.ts`) — REUSA a infra, NÃO cria um 2º KV (o `prefs.ts` foi
> desenhado p/ isto, per F5.2). Provider de tema hidrata do KV no boot e grava on-change.

## Passos sugeridos

1. Em app/lib/theme.ts: no ThemeProvider, ler o modo salvo (prefs, chave ex. 'theme.mode') no boot para inicializar o override; em toggle/setMode, persistir (ou limpar, p/ voltar a seguir o sistema). Manter useColorScheme como base quando não há override salvo.
2. Garantir que o valor persistido só assume 'light'|'dark'|ausente (sem outros estados); nunca loga.
3. Estender/duplicar a prova headless: um caso em app/web/__tests__/i18n.test.mjs (ou novo prefs.test.mjs) que faz round-trip do 'theme.mode' com backend em memória (set→get→re-hidrata).
4. Gates: tsc/eslint/expo export web verdes.

## Critério de aceite

- [ ] Alternar o tema e reabrir o app mantém o modo escolhido (persistido via prefs offline).
- [ ] Sem override salvo, o app segue o esquema do sistema (comportamento F1.4 preservado).
- [ ] Round-trip de 'theme.mode' provado por headless (backend injetável em memória); nenhuma dependência nova.
- [ ] tsc 0, eslint 0, expo export web exit 0; core intacto.

## Restrições

Reusa o KV de prefs do tracer (NÃO criar outro mecanismo de persistência). Edita a lógica do ThemeProvider; a-contrast edita as consts de paleta (LIGHT/DARK) — regiões distintas de theme.ts, mas se ambas mexerem no arquivo, sequenciar. Registrar como amendment ao ADR-0015 (persistência agora entregue).

- **Offline-first:** o tema persiste no KV offline da F5.2 (local); nada exige rede.
- **Anti-alucinação:** n/a (só UI de tema); não tocar texto bíblico.
- **the-light só via PR+ADR:** `the-light-core` pinado (`2fc2dab`); app-side.
- **Qualidade:** `tsc --noEmit`; self-tests sem regressão; prova de persistência (round-trip do modo de tema sobrevive reopen, molde da F5.2); sem flash de tema errado no boot (hidratação síncrona/adiada aceitável).
