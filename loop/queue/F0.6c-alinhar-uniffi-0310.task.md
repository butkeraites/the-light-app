## Objetivo

Corrigir a causa-raiz que reprovou a F0.6b (attempt 1): nossa fronteira
`the-light-app-core` fixa `uniffi = "0.31.2"`, mas o **runtime web do `ubrn`
0.31.0-3** (`uniffi-runtime-javascript`) fixa `uniffi_core = "=0.31.0"`. No grafo
do wasm-crate gerado pelo `ubrn build web`, os dois pins exatos (`=0.31.2` vs
`=0.31.0`) sobre o mesmo `uniffi_core` são irreconciliáveis → `cargo` não resolve.

**Esta tarefa alinha a `uniffi` da NOSSA fronteira para `=0.31.0`** (a versão que a
toolchain `ubrn` já usa), reverte o ponto correspondente do ADR-0003, e
**revalida** que F0.2/F0.3 (build+test) e F0.4 (geração JSI) continuam verdes sob
0.31.0 — destravando a F0.6b (retry).

É uma mudança **no `the-light-app`** (nosso `core/`), **não** no `the-light`:
patch-version dentro de `0.31.x` (mesmo *contract version*), de baixo risco — o
ADR-0004 já provou que o `ubrn` 0.31.0-3 (bindgen `=0.31.0`) lê a cdylib da
fronteira sem erro de contrato.

## Contexto / arquivos

- Reprovação documentada: `loop/archive/F0.6b.attempt1.result.md` (reviewer:
  `failed`, correção em escopo). Pins reais confirmados:
  - `core/Cargo.toml:19` → `uniffi = "0.31.2"` (→ `uniffi_core =0.31.2`).
  - `node_modules/uniffi-bindgen-react-native/crates/uniffi-runtime-javascript/Cargo.toml`
    → `uniffi_core = { version = "=0.31.0" }`; feature `wasm32` encaminha
    `uniffi_core/wasm-unstable-single-threaded` (resolve o `+Send` — não é o muro).
- Arquivos a tocar: `core/Cargo.toml` (pin do `uniffi`), `core/Cargo.lock`
  (regenerado), `DECISIONS.md` (atualizar ADR-0003 + escrever **ADR-0006**).
- `uniffi 0.31.0` provavelmente **não** está no cache local (só `0.31.2` estava) —
  o build vai **baixar** (rede em dev/build é permitida; offline-first é runtime).
- NÃO altere a forma da fronteira (modo library / `setup_scaffolding!` /
  `#[uniffi::export]` / `CoreError` / `parse_reference`) — só a **versão**.

## Passos sugeridos

1. Em `core/Cargo.toml`, trocar `uniffi = "0.31.2"` por **`uniffi = "=0.31.0"`**
   (pin EXATO, para casar com `uniffi_core =0.31.0` do runtime do ubrn). Manter
   `default-features` como está (a feature `wasm32` é ligada pelo wasm-crate do
   ubrn, não aqui).
2. Regenerar o lock: `cargo update -p uniffi --precise 0.31.0` (ou `cargo build`),
   confirmando no `core/Cargo.lock` que `uniffi`/`uniffi_core`/`uniffi_macros`
   ficam em **0.31.0**.
3. Qualidade da fronteira (de `core/`): `cargo fmt --check`,
   `cargo clippy -- -D warnings`, `cargo build`, `cargo test` — **verdes** sob
   0.31.0 (revalida F0.2/F0.3: `ping`, `parse_reference`, "Jo 3.16"=="John 3:16").
4. Revalidar a geração **JSI/nativa** (F0.4): rodar `./scripts/gen-bindings.sh` e
   confirmar que `bindings/` é repopulado (≥1 `.ts` referenciando
   `parseReference`/`ping`) **sem erro** sob a fronteira 0.31.0. (Os `.ts` são
   gerados/ignorados — não versionar.)
5. `DECISIONS.md`: atualizar o ADR-0003 (nota: versão re-alinhada de `0.31.2` →
   `=0.31.0` por compat. com o runtime web do ubrn) e escrever **ADR-0006**
   (decisão: pin `uniffi = "=0.31.0"` para o caminho web; impacto/risco; revalidação
   verde de F0.2/F0.3/F0.4).
6. Commit com prefixo do ID:
   `fix(F0.6c): alinhar uniffi da fronteira a =0.31.0 (compat. ubrn web; ADR-0006)`.

## Comando(s) de verificação

```sh
set -euo pipefail
ROOT="$(git rev-parse --show-toplevel)"

# 1) Pin exato =0.31.0 na fronteira
grep -qE '^uniffi[[:space:]]*=[[:space:]]*"=0\.31\.0"' "$ROOT/core/Cargo.toml" \
  && echo "OK  uniffi = \"=0.31.0\"" || { echo "FALTA pin uniffi =0.31.0"; exit 1; }

# 2) Lock em 0.31.0 (uniffi e uniffi_core)
grep -A1 'name = "uniffi_core"' "$ROOT/core/Cargo.lock" | grep -q '0.31.0' \
  && echo "OK  uniffi_core 0.31.0 no lock" || { echo "FALHA uniffi_core nao e 0.31.0"; exit 1; }

# 3) Qualidade + testes da fronteira sob 0.31.0
( cd "$ROOT/core" && cargo fmt --check && cargo clippy -- -D warnings && cargo build && cargo test ) \
  && echo "OK  fronteira verde sob uniffi 0.31.0"

# 4) Regressão JSI (F0.4): gen-bindings.sh roda e popula bindings/
"$ROOT/scripts/gen-bindings.sh" >/tmp/f06c-gen.log 2>&1 \
  && [ "$(find "$ROOT/bindings" -name '*.ts' -size +0c | wc -l | tr -d ' ')" -ge 1 ] \
  && echo "OK  JSI bindings regenerados sob 0.31.0" || { echo "FALHA gen-bindings.sh (regressao F0.4)"; tail -20 /tmp/f06c-gen.log; exit 1; }

# 5) ADR-0006 e working tree limpo
grep -q 'ADR-0006' "$ROOT/DECISIONS.md" && echo "OK  ADR-0006" || { echo "FALTA ADR-0006"; exit 1; }
cd "$ROOT"; test -z "$(git status --porcelain)" && echo "OK  working tree limpo" \
  || { echo "FALHA tree sujo:"; git status --porcelain; exit 1; }

echo "F0.6c verificada."
```

## Restrições

- **NÃO tocar/forkar o `the-light`.** Esta é uma mudança no nosso `core/` (fronteira)
  — só a versão do `uniffi`. A forma da fronteira (modo library, `parse_reference`,
  `CoreError`) **não** muda.
- **Não relaxar qualidade:** `fmt`/`clippy -D warnings`/`build`/`test` verdes antes
  do commit. Se 0.31.0 introduzir um warning/erro de clippy/compilação na fronteira,
  corrija o mínimo necessário **na fronteira** (não no core externo) e registre.
- **Rede só em dev/build** (baixar `uniffi 0.31.0`). Nada de rede no runtime; segredos
  nunca em git.
- **Se, sob 0.31.0, a fronteira ou a geração JSI (F0.4) regredirem de forma não
  trivial** (ex.: a API do `uniffi` 0.31.0 difere a ponto de exigir mudar a forma da
  fronteira ou revalidar decisão maior), **reporte** o desfecho real (failed/blocked)
  com a saída exata — não force. (Esperado: 0.31.0 é patch e deve ser verde.)
