## Objetivo

O smoke da F6.2 registrou (âncora de regressão) a realidade de CORS no BROWSER: **Gemini** e
**OpenAI** alcançam o provedor (200/401 = CORS ok), **Anthropic** bate parede CORS (`net::ERR_FAILED`),
**Ollama-web** exige config local. No NATIVO (reqwest) todos funcionam (sem CORS). A Anthropic
libera origem de browser SE a request enviar `anthropic-dangerous-direct-browser-access: true`
(aceitável em BYOK = chave do PRÓPRIO usuário). Esta tarefa (a) habilita esse header, (b) deixa a
capacidade por-provedor HONESTA na UI (sem prometer o que o browser não entrega), **sem proxy**
(preserva offline-first/no-server). Decisão humana aprovada: habilitar web com rótulos honestos.

## Escopo

1. **`app/web/ai-anchored.web.ts` (`anthropicHeaders`, ~:305-312):** adicionar
   `'anthropic-dangerous-direct-browser-access': 'true'` (junto de `x-api-key`/`anthropic-version`/
   `content-type`). Só afeta o alvo WEB (o transporte web). NÃO tocar o nativo (core).
2. **Rótulos de capacidade por provedor (honestos) na UI** — no seletor de provedor (Ask/Study/Chat/
   Compare) e/ou na tela de Ajustes (F6.6): quando `Platform.OS === 'web'`, indicar o estado real:
   Gemini/OpenAI/Anthropic = "funciona no navegador"; **Ollama = "requer config local (`OLLAMA_ORIGINS`)"**.
   No nativo, todos funcionam. Novas chaves `t()` pt/en; nada de hex; a11y. NÃO prometer o que o browser
   não entrega (ex.: não rotular Ollama-web como pronto).
3. **`DECISIONS.md` — ADR-0058:** decisão de habilitar IA no web (Gemini/OpenAI/Anthropic com o header
   opt-in de browser da Anthropic; Ollama-web = config local; nativo = todos; **sem proxy**), reconciliando
   com offline-first/no-server e BYOK (a chave é do usuário; o header "dangerous" é a via oficial e opt-in).
4. **Smoke — flipar a âncora (F6.2 `ai-reachability`):** com o header, a Anthropic passa de **CORS-wall**
   para **alcançada** (com a chave DUMMY → 401 do provedor). Atualizar a asserção de `ai-reachability`
   para esperar Anthropic ALCANÇADA (401/reachable) em vez de CORS-wall — provando a correção
   end-to-end no browser real. (Gemini/OpenAI seguem alcançados; nativo fora do escopo do smoke web.)

## Notas

- **Resposta REAL da Anthropic no browser** (chave real + CORS real) só é confirmável com chave BYOK →
  gate humano; o smoke prova a ALCANÇABILIDADE (401 com dummy = chegou no provedor, CORS liberado). Não usar chave real.
- Sem proxy: um proxy violaria offline-first/no-server. Se o usuário quiser Ollama no web, ele configura
  `OLLAMA_ORIGINS` do próprio lado — a UI só informa isso.
- O header `dangerous-direct-browser-access` é a via SANCIONADA da Anthropic p/ uso client-side com a
  chave do usuário (BYOK) — registrar essa justificativa no ADR-0058.
