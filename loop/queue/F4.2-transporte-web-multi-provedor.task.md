## Objetivo

Estender o **transporte web de IA** (`app/web/ai-anchored.web.ts::webLlmTransport`) para
tratar os provedores **anthropic**, **openai** e **ollama** — que **já existem no core**
(`ai::PROVIDERS`, nativo já funciona) — **além** de `gemini`/`mock`. Para CADA provedor: o
**corpo do request** (`*_body`) + a **extração da resposta** (`*_extract`) — **streaming e
não-streaming** — + o **endpoint/headers corretos**, com a chave BYOK **só no header
apropriado** de cada provedor. É a continuação natural da F4.1 (streaming token-a-token),
agora **por provedor**: o `onToken` (streaming) segue o formato **SSE/NDJSON** REAL de cada
API. Hoje o transporte só despacha `gemini`+`mock` (MVP F2.7b/ADR-0025); os demais lançam
"ainda não tem transporte web".

**App-side APENAS.** A fronteira Rust `ai_web_prepare`/`ai_web_finalize` (`ai-pure`,
cfg-free) **NÃO muda**: o `system`/`user` (prompt/RAG) e o `model` já vêm do
`ai_web_prepare` (que resolve `default_model` por provedor); o `cited_text` (texto bíblico
numerado, VERBATIM do store) sai do `prepare` e a `interpretation` final (concatenada) passa
pela MESMA `ai_web_finalize` (`rewrite_anchors`, mesma impl Rust). **Anti-alucinação
ZERO-DRIFT:** o transporte **só** monta o request (com o `system`/`user` do Rust) e extrai a
**interpretação** do modelo; nenhum prompt/citação/aparato é reimplementado em TS (PROIBIDO,
ADR-0029); o `cited_text` viaja SEPARADO. Espelhar os `*_body`/`*_extract` PRIVADOS do core
é **transporte = infra** (ADR-0023/D2, ADR-0025), não lógica de domínio.

**NÃO-BLOQUEANTE** (prova por MOCK de fetch/stream por provedor, sem chave/rede real).
**NÃO toca `the-light`** (`04b9b24` intacto). Um provedor **novo, inexistente no core**
(ex.: um LLM não suportado) exigiria `impl LlmProvider` no core → **PR futuro, fora de
escopo** (se acionado, é tarefa `gate:true`).

### Formatos REAIS por provedor — CONFIRMADOS na fonte do core (`04b9b24`, só-leitura)

Fonte: `~/.cargo/git/checkouts/the-light-9eb8809a6d68281a/04b9b24/crates/the-light-core/src/ai/providers.rs`

**Anthropic** (`anthropic_body`:155, `anthropic_extract`:182, `post`:224):
- Endpoint: `POST https://api.anthropic.com/v1/messages` (não-stream E stream — mesmo URL).
- Headers: `x-api-key: <key>`, `anthropic-version: 2023-06-01`, `content-type: application/json`.
- Body (não-stream): `{ "model": model, "max_tokens": 8192, "system": system,
  "thinking": {"type":"adaptive"}, "messages": [ {"role":"user","content": user} ] }`.
- Extract (não-stream): concatena `content[]` onde `type=="text"` → `text`; se
  `stop_reason=="refusal"` → erro; texto vazio → erro.
- **Streaming (SSE)** [não presente no core — F4.6 fará nativo; forma pública da Messages
  API]: body adiciona `"stream": true`; a resposta é `text/event-stream` com eventos
  `event: content_block_delta` cujo `data: {"type":"content_block_delta","delta":{"type":
  "text_delta","text":"..."}}`. **Delta = eventos `content_block_delta` com
  `delta.type=="text_delta"` → `delta.text`** (ignora `message_start`/`content_block_start`/
  `message_delta`/`message_stop`/`ping`).

**OpenAI** (`openai_body`:246, `openai_extract`:263, `post`:297):
- Endpoint: `POST https://api.openai.com/v1/chat/completions`.
- Headers: `authorization: Bearer <key>`, `content-type: application/json`.
- Body (não-stream): `{ "model": model, "max_tokens": 8192, "messages": [
  {"role":"system","content": system}, {"role":"user","content": user} ] }`.
- Extract (não-stream): `choices[0].message.content` (JSON pointer `/choices/0/message/content`);
  vazio → erro.
- **Streaming (SSE)** [forma pública Chat Completions]: body adiciona `"stream": true`;
  linhas `data: {"choices":[{"delta":{"content":"..."}}]}`, terminadas por `data: [DONE]`.
  **Delta = `choices[0].delta.content`** (ausente em eventos só-`role`/`finish_reason` →
  ignora).

**Ollama** (`ollama_body`:318, `ollama_extract`:335, `post`:364):
- Endpoint: `POST {host}/api/chat` — host default `http://localhost:11434` (nativo via env
  `LIGHT_OLLAMA_HOST`; **no web NÃO há env** → default `http://localhost:11434/api/chat`).
- Headers: `content-type: application/json` — **SEM chave** (é local; ollama não usa BYOK).
- Body (não-stream): `{ "model": model, "stream": false, "messages": [
  {"role":"system","content": system}, {"role":"user","content": user} ] }`.
- Extract (não-stream): `message.content` (JSON pointer `/message/content`); vazio → erro.
- **Streaming (NDJSON, NÃO SSE)**: body seta `"stream": true`; a resposta é **JSON por
  linha** (sem prefixo `data:`): `{"message":{"content":"..."},"done":false}` … linha final
  `{"message":{"content":""},"done":true}`. **Delta = `message.content` por linha.**

**Gemini** (referência — JÁ em TS, F2.7b/F4.1): endpoint
`.../models/{model}:generateContent` / `:streamGenerateContent?alt=sse`; header
`x-goog-api-key`; delta via `geminiPartText` (`candidates[0].content.parts[*].text`).

## Restrições

- **Anti-alucinação ZERO-DRIFT:** o transporte muda **só o request/extração**. NÃO
  reimplementar prompt/RAG/citação/aparato em TS (PROIBIDO, ADR-0029). `system`/`user`/`model`
  vêm do `ai_web_prepare` (Rust `ai-pure`); `cited_text` sai do `prepare` (store, VERBATIM,
  separado) e a `interpretation` completa passa pela MESMA `ai_web_finalize` (`rewrite_anchors`,
  Rust). Os `*_body`/`*_extract` espelham os PRIVADOS do core (transporte = infra,
  ADR-0023/0025) — nunca a lógica de domínio.
- **BYOK — chave por provedor, session-only, só no header:** anthropic `x-api-key`
  (+`anthropic-version`), openai `Authorization: Bearer`, gemini `x-goog-api-key`, **ollama
  sem chave** (localhost). NUNCA na URL/log. `fetch` INJETÁVEL; a prova usa MOCK — **nenhuma**
  rede/chave real no loop.
- **Offline-first:** IA web é opt-in; sem chave/rede o app segue offline. O caminho `"mock"`
  roda sem rede (inalterado).
- **NÃO tocar `the-light`** nem `core/src/lib.rs`/`core/Cargo.toml` (app-side apenas;
  `04b9b24` intacto; provedores já existem no core — nativo já funciona). Provedor **novo,
  inexistente no core** = PR futuro (fora de escopo). Se o transporte multi-provedor exigir
  mudar a fronteira Rust/o core → **HALT** (não é o caso: `LlmRequestParts` do
  `ai_web_prepare` é suficiente).
- **Qualidade por tarefa:** `tsc --noEmit` + `expo export --platform web` verdes (eslint não
  está configurado no repo — precedente F4.1); prova headless determinística (MOCK) por
  provedor verde; sem regressão dos testes web existentes.
- Se registrar os endpoints/headers/body/extract/streaming por provedor do transporte web,
  usar **ADR-0034** (próximo ADR livre).
