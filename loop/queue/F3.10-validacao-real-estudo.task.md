# F3.10 — Validação REAL do estudo profundo (⛔ GATE / BLOQUEANTE)

> **Precisa da CHAVE do usuário + rede opt-in.** A chave é um **segredo**: NUNCA pode
> entrar no chat, git ou log. A validação é **sua**, com a chave numa **env var** do seu
> terminal; o loop só recebe o **RESULTADO** (não-segredo). Desfecho = **HALT** até validar.

## O que já está pronto (não-bloqueante)
Toda a cadeia nativa de estudo profundo (F3.1–F3.9a): léxico STEP CC-BY, fronteira de
léxico/estudo/conversa, UI de estudo/conversa/comparação, export acadêmico, e a pesquisa
web Wikipedia opt-in no `deep_study`. Harness de host **`core/examples/study_real.rs`**
(compila offline, fmt/clippy limpos, **não imprime a chave**).

## Como você valida (a chave fica só na env var — nunca no chat/git/log)
No repo `/Users/butkeraites/Documents/the-light-app` (rode com `! <cmd>` ou no seu terminal):

Estudo profundo real (sua chave Gemini valida, modelo atual):
```
AI_PROVIDER=gemini AI_MODEL=gemini-2.5-flash AI_API_KEY="<sua-chave>" \
  cargo run --manifest-path core/Cargo.toml --example study_real
```
Com pesquisa web (Wikipedia keyless, opt-in) incluída:
```
AI_PROVIDER=gemini AI_MODEL=gemini-2.5-flash AI_RESEARCH=wikipedia AI_API_KEY="<sua-chave>" \
  cargo run --manifest-path core/Cargo.toml --example study_real
```
(troque `AI_PROVIDER`/chave por anthropic/openai se preferir; `AI_MODE`/`AI_LENS`/`AI_DEPTH`
opcionais.)

Opcional — validar também a **conversa** real (F3.4): rode o `ask_real.rs` da F2.6, ou avise
que quer um harness de conversa multi-turno e eu preparo.

## Critério de aceite (você confirma e reporta)
- [ ] Retorna uma **INTERPRETATION real** do provedor (não a do mock).
- [ ] O **PASSAGE_TEXT** é **João 3:16 verbatim do STORE** ("For God so loved…") — anti-alucinação.
- [ ] As **CITATIONS** vêm do banco/léxico (Strong STEP CC-BY) — e da web (Wikipedia) se
      `AI_RESEARCH=wikipedia` — **nunca do modelo**; o **ACADEMIC_MARKDOWN** > 0.
- [ ] Nenhum erro de chave/rede; a chave não apareceu em nenhum output.
- [ ] Você reporta ao loop: **provedor/modelo + "funcionou"** (e, se quiser, um trecho da
      interpretação/citações) — SEM colar a chave.

## Se falhar

do core trazem só nome do provedor/mensagem HTTP). O Driver ajusta num ciclo focado.
