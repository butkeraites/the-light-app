## Objetivo

Habilitar o **2º backend de pesquisa web — Tavily (BYOK)** no estudo profundo **WEB**,
como paridade da F4.3 (Tavily nativo). É **100% app-side** (glue TS `app/web/*` + UI +
self-test): **NÃO toca o `the-light`** nem o Rust `ai-pure`. Como `TavilyProvider`/
`build_research_provider` são `embedded`-only (reqwest → não compilam em wasm), no web a
busca é um `fetch` TS (molde EXATO de `research.web.ts::wikipediaSearch`, mas **com
chave** session-only) — e a montagem das citações `[W:n]`/`kind="Web"` continua vindo do
**MESMO Rust `ai-pure`** (`study_web_prepare`/`study_web_finalize`), **ZERO-DRIFT**: o TS
só faz o HTTP e entrega as **fontes cruas** (`StudyWebSourceInput[]`); NUNCA monta citação
em TS (ADR-0029).

Hoje (após F4.3):
- `study.web.ts::deepStudyOnHandle` já recebe `researchKey?` mas o **IGNORA** (`void
  researchKey`) — Tavily web era a F4.4.
- `study.web.ts::resolveWebSources` só trata `"wikipedia"` (keyless) e **lança** para
  qualquer outro backend (inclusive `"tavily"`).
- `reading.web.ts::deepStudy` já threada `researchBackend?`/`researchKey?` até
  `deepStudyOnHandle`.
- `ReaderStudyPanel.tsx` tem um toggle booleano **off/Wikipedia** (`webResearch` →
  `WIKIPEDIA_BACKEND`), sem opção Tavily nem campo de chave.

Semântica resultante desta tarefa:
- `researchBackend` ausente/vazio → `[]` (offline por padrão, **preservado**).
- `researchBackend === "wikipedia"` → `wikipediaSearch` (keyless, **sem regressão**).
- `researchBackend === "tavily"`:
  - com `researchKey` → `tavilyFetch(fetch, key, query, lang, limit)` → `POST
    https://api.tavily.com/search` (chave **só no CORPO** do POST, nunca na URL/log) →
    `StudyWebSourceInput[]` → `web_sources` do `study_web_prepare`/`study_web_finalize`.
  - **sem** `researchKey` → **erro explícito citando só "tavily"** (0 fetch), espelhando
    o padrão dos provedores de IA (`AiError::NoKey`) e do nativo F4.3.
- backend desconhecido → erro explícito (comportamento atual preservado).

**NÃO-BLOQUEANTE** (prova só por MOCK, sem chave/rede real). A **validação com a chave
Tavily REAL** (fontes reais citadas `[W:n]`) é a **F4.5** (⛔ gate, chaves/rede).

## Contexto / arquivos

- **Forma REAL da request/response do Tavily** (confirmada na fonte `04b9b24`, só-leitura,
  `crates/the-light-core/src/ai/research.rs:240-287`, `TavilyProvider::search` — o
  transporte TS ESPELHA isto):
  - **Request:** `POST https://api.tavily.com/search`, header `content-type:
    application/json`, corpo JSON:
    ```json
    { "api_key": "<key BYOK>", "query": "<reference_label>",
      "max_results": <limit clamped 1..10>, "search_depth": "basic" }
    ```
    A **chave BYOK vai NO CORPO** (campo `api_key`), **NÃO em header** e **NÃO na URL**.
    `query` = o rótulo da passagem (`<Livro EN> <cap>[:<verso>]`), igual ao
    `reference_label`/`researchQuery` do web hoje. `max_results` = `limit.clamp(1,10)`
    (default de pesquisa = 4, igual ao `DEFAULT_WIKIPEDIA_LIMIT`). `search_depth =
    "basic"`.
  - **Response parse:** `v["results"]` (array). Por item (pular se `url` vazio):
    `url = it["url"]`; `title = it["title"]` (default `"(sem título)"`); `snippet =
    it["content"]` (default `""`); `site` = host extraído da `url` (`url.split("://")[1]`
    até o primeiro `/`); `fetched_at` = agora. Mapeia para `WebSource{title,url,snippet,
    site,fetched_at}` — no web, o tipo é `StudyWebSourceInput{title,url,snippet,site,
    fetchedAt: bigint}` (o mesmo que `wikipediaSearch` já produz).

## Restrições

- **Anti-alucinação ZERO-DRIFT (ADR-0029):** as citações Web são montadas pelo Rust
  `study_web_finalize` (das URLs), NUNCA pelo TS. O transporte TS **só** faz o `fetch`
  Tavily e entrega as fontes cruas (`StudyWebSourceInput[]`). **PROIBIDO** espelhar
  montagem de citação/`[W:n]`/prompt/aparato em TS. O `passage_text` (João 3:16) vem
  VERBATIM do store, separado da `interpretation`.
- **BYOK (ADR-0025):** a chave Tavily (`research.tavily`) é **session-only / in-memory**
  no web — **nunca** persistida (Local/Session Storage, IndexedDB, disco), **nunca**
  logada/impressa, **nunca** em git. Vai **só no CORPO** do POST (`api_key`), nunca na
  URL/header/mensagem de erro. Sem chave e backend=tavily → erro citando só "tavily", 0
  fetch.
- **Tavily = rede OPT-IN, padrão OFF.** Nenhuma chamada de rede real no CI/self-test (o
  `fetch` é MOCK). A validação real com a chave do usuário é a **F4.5** (⛔ gate).
- **Wikipedia keyless preservado (ADR-0028).** Nenhuma regressão do caminho Wikipedia.
  **Offline-first:** sem backend → `[]`; a Fase 1 segue 100% offline.
- **NÃO tocar o `the-light` nem o Rust `ai-pure`.** Tavily web = transporte `fetch`
  (app-side), como toda a pesquisa web (ADR-0028/0025/0034). `TavilyProvider`/
  `build_research_provider` são `embedded`-only e permanecem intactos; `core/Cargo.toml`,
  o pin `04b9b24`, `study_web_prepare`/`study_web_finalize` **NÃO** mudam. Se algo exigisse
  tocar Rust/the-light → **`blocked`/HALT** (não stubar/forkar/copiar).
- **Escopo atômico:** transporte `tavilySearch` + threading em `study.web.ts` + UI toggle/
  chave + self-test MOCK. **Sem** validação real (F4.5), **sem** PR, **sem** Tavily nativo
  Keychain. Registrar **ADR-0035** (Tavily web: transporte `fetch` a `api.tavily.com/
  search` + chave BYOK session-only só no corpo + toggle UI opt-in) — reservado se ainda
  livre (a F4.3 não consumiu ADR).
