## Objetivo

Popular a tabela **`cross_references`** do banco `assets/data/bible.sqlite`
(gerado pela F1.1) com as **~344.799 referências cruzadas** do
**OpenBible.info / Treasury of Scripture Knowledge (TSK)**, **rodando o
importador canônico `xtask import-xref`** do `the-light` (rev pinado `8f66004`,
ADR-0002) — **sem modificar/forkar `the-light`**, de forma **reprodutível** (um
script) e **idempotente** (reimportar não duplica).

Esta é uma tarefa de **DADOS/pipeline**, análoga à F1.1: **sem UI** e **sem
fronteira** (a leitura de xref na fronteira é **F1.8**; a UI + atribuição visível
é **F1.9**). O dado de xref do OpenBible é **CC-BY** (não domínio público): exige
**atribuição obrigatória**. Esta tarefa **registra** essa origem/licença e a
**string de atribuição** em **ADR-0016** (a exibição visível é responsabilidade
da F1.9). Decide/confirma a **estratégia de armazenamento** (o `bible.sqlite` já é
**gerar-ignorado** por ADR-0013; o xref só o faz crescer alguns MB).

Anti-alucinação/licenciamento: os xrefs vêm **sempre** do importador canônico
sobre a fonte CC-BY registrada no `xtask` — **nunca** inventados, **nunca**
fonte não-livre. CC-BY exige atribuição; só dados **livres** (PD/CC0/CC-BY).

## Contexto / arquivos

**Subcomando REAL descoberto na fonte do xtask (rev `8f66004`):** é
**`import-xref`** (um subcomando dedicado), **não** `import --xref`. Confirmado em
`~/.cargo/git/checkouts/the-light-9eb8809a6d68281a/8f66004/xtask/src/main.rs`:
`Some("import-xref") => xref_import::run(&args[1..])`. A lógica vive em
`xtask/src/xref_import.rs` (leitura permitida — cópia read-only do rev pinado; o
repo canônico `/Users/butkeraites/Documents/the-light` é **leitura-apenas / não
tocar**).

**Flags de `import-xref`** (de `xref_import.rs::run`, parser próprio):
`--db <PATH>`, `--seed-dir <DIR>`, `--force`, `--offline`. **Não há `--version`**
(xref é independente de tradução; chaveado pela tríade canônica
`book/chapter/verse`). Qualquer outra flag → o xtask aborta
("argumento desconhecido para `import-xref`").

**Fonte (CC-BY, URL fixada no xtask — `XREF_URL` em `xref_import.rs`):**
`https://raw.githubusercontent.com/scrollmapper/bible_databases/master/sources/extras/cross_references.txt`
— TSV plano (3 colunas: `From Verse`, `To Verse`, `Votes`), espelho raw do
scrollmapper dos dados **OpenBible.info**. **Atribuir OpenBible.info, NÃO
scrollmapper** (o scrollmapper é só o mirror git-pinnable; a camada CC-BY
votada/compilada é da OpenBible). ~8,3 MB; **344.800** linhas = 1 cabeçalho +
**344.799** xrefs. O xtask grava/lê o arquivo em `<seed-dir>/cross_references.txt`
e baixa por rede **só** se ausente e sem `--offline` (rede em build permitida).

**Guarda de drift no próprio xtask:** se `parse_tsv` der **< 300.000** linhas
válidas, o `import-xref` **aborta** (`"apenas N … esperado ~344.799; fonte
incompleta?"`). Linhas com OSIS irresolúvel são filtradas no parse → a contagem
final pode ficar **ligeiramente abaixo** de 344.799 (snapshot do mirror +
filtragem). Trate o aceite como **guarda de drift** (≈ 344.799), não igualdade
exata.

**Schema `cross_references` (uma fonte da verdade — migração `v1_initial.sql` do
core, aplicada por `Store::open`):**
`cross_references(from_book, from_chapter, from_verse, to_book, to_chapter,
to_verse_start, to_verse_end, votes)` + `idx_xref_from`. **Não há coluna de
licença/atribuição** nesta tabela (diferente de `translations.license`) → a
atribuição CC-BY é registrada em **ADR-0016** (string canônica) e exibida pela UI
na **F1.9**. `votos` podem ser **negativos** (refs disputadas) — não filtrar aqui
(o threshold `min_votes` é decisão da fronteira F1.8).

**Destino = o MESMO `bible.sqlite` da F1.1.** A tabela `cross_references` é parte
do **mesmo** schema v1 e **já existe (vazia)** no `bible.sqlite` gerado pela F1.1
(o `import` da F1.1 e o `import-xref` usam ambos `Store::open`, que aplica as
migrações). O `import-xref` só a **popula**:
`import_rows` faz `DELETE FROM cross_references` e reinsere tudo em transação →
**idempotente** (reimportar → mesma contagem, sem duplicar). Ao final imprime
`✓ <n> referências cruzadas importadas (OpenBible.info, CC-BY).`

**Como rodar SEM tocar `the-light` (mesmo padrão da F1.1 / ADR-0013):** rodar o
member `xtask` do **checkout pinado do cargo**
(`$HOME/.cargo/git/checkouts/the-light-9eb8809a6d68281a/8f66004/xtask/Cargo.toml`)
com `CARGO_TARGET_DIR` **FORA** do checkout (`.cache/xtask-target`, ignorado) e
`--locked`. **Não** rodar dentro de `/Users/butkeraites/Documents/the-light`.

**Script (F1.1 = molde):** `scripts/gen-bible-db.sh` já roda o `xtask import`
(verses + FTS) com exatamente esse padrão (checkout pinado, `CARGO_TARGET_DIR`
fora, `--locked`, seed-dir `.cache/seed`, repasse de `--offline`/`--force`).
**Caminho recomendado:** **estender `scripts/gen-bible-db.sh`** para, **após** o
`import`, rodar também o `import-xref` no **mesmo** `--db assets/data/bible.sqlite`
e **mesmo** `--seed-dir .cache/seed` → um único pipeline reprodutível gera o banco
**completo** (verses + FTS + xrefs). (Alternativa: `scripts/gen-xref.sh`
dedicado, mas então ele **deve** rodar **após**/exigir `gen-bible-db.sh` — rodar
`import-xref` num `--db` inexistente criaria um banco **só com xrefs** e sem
versículos, um footgun; evite.)

**Armazenamento (decisão → ADR-0016):** o `bible.sqlite` já é **gerar-ignorado**
(ADR-0013; `.gitignore` linhas `assets/data/bible.sqlite*` + `/.cache/`). O xref
acrescenta ~344k linhas + índice (poucos MB). Confirmar que a decisão
**gerar-ignorado** continua valendo (medir o tamanho novo); **nenhuma** mudança de
`.gitignore` deve ser necessária (seed-dir e banco já ignorados). **Não** versionar
o binário.

**Escopo (guarda):** F1.7 popula `cross_references` **no `bible.sqlite`**.
**Não** antecipar: a propagação dos xrefs para o subset `reading-sample.sqlite`
(bundling no app) é da **F1.9**; a fronteira `cross_refs`/`for_verse` é **F1.8**;
a UI + atribuição visível é **F1.9**.

**Documentos:** `DECISIONS.md` (ADR-0002 pin `8f66004`; ADR-0013
bible.sqlite gerar-ignorado + como o xtask roda sem tocar the-light);
`IMPLEMENTATION_PLAN.md` (§F1.5 xrefs ~344k OpenBible/TSK, atribuição CC-BY

## Restrições

- **NÃO modificar nem forkar `the-light`** (`/Users/butkeraites/Documents/the-light`
  é leitura-apenas; consumo é git dep pinada `rev 8f66004`, ADR-0002). Rodar o
  `xtask import-xref` **sem** escrever no source dele (`CARGO_TARGET_DIR` fora do
  checkout + `--locked`). Se a **única** forma de invocar o importador exigir
  tocar/alterar o `the-light` → `blocked` com o erro EXATO (decisão humana) —
  **não** improvisar uma cópia paralela da lógica de xref.
- **Só dados LIVRES CC-BY** (OpenBible.info / TSK, exatamente a `XREF_URL`
  registrada no xtask). CC-BY exige **atribuição** — registrar a string canônica
  em ADR-0016 (exibição visível é F1.9). **Jamais** apontar para fonte não-livre;
  **nenhum** dado de xref inventado/hardcoded (anti-alucinação: vem verbatim da
  fonte canônica via xtask).
- **Offline-first é regra de RUNTIME:** rede só em **dev/build** (download do TSV).
  O app em runtime **não** faz rede. **Segredos nunca** em git/log.
- **Blocked legítimo (rede/seed):** o `cross_references.txt` **não está em cache**
  (a F1.1 só semeou os datasets de verses, não o xref) → o `import-xref` **baixa
  por rede**. Se o ambiente estiver **offline** e sem seed do xref, rodar com
  `--offline` falha → escrever `done/F1.7.result.md` com `status: blocked`, a
  **saída literal** do comando e a decisão pendente (origem/seed do dataset de
  xref). **Não** fabricar dados para "passar".
- **Armazenamento:** o `bible.sqlite` continua **gerar-ignorado** (ADR-0013); o
  seed-dir é **sempre ignorado**. Não commitar binário grande; registrar o tamanho
  novo em ADR-0016.
- **Escopo:** F1.7 entrega **só os DADOS** (popular `cross_references` no
  `bible.sqlite`). A leitura de xref na **fronteira** (`cross_refs`/`for_verse`/
  `passage_labels`) é **F1.8**; a **UI + atribuição CC-BY visível** é **F1.9**; a
  propagação dos xrefs ao `reading-sample.sqlite` (bundling) é **F1.9**. Não
  antecipar.
