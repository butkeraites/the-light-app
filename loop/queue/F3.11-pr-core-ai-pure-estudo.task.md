## Estratégia de desacoplamento (a MENOS arriscada — widen do gate + 1 `pub` novo)

Espelha o precedente **ADR-0024/F2.7**: **aditiva, opt-in, defaults byte-a-byte**. Em
vez de mover código, **ampliar o gate `#[cfg(feature="embedded")]` → `#[cfg(any(feature=
"embedded", feature="ai-pure"))]`** na superfície **pura** do estudo, e **gatear por
`embedded` só o que puxa SQLite/reqwest/disco**. Passos:

1. **`Cargo.toml` do core — `ai-pure` ganha `chrono` clock-free (p/ `WebSource`):**
   - Motivo: `StudyRequest.web_sources: Vec<WebSource>` e `from_web_results` exigem
     `WebSource` (que tem `fetched_at: DateTime<Utc>`) no grafo `ai-pure`. `chrono` é
     Rust puro e wasm-safe; **sem a feature `clock`** não puxa `wasm-bindgen`/`SystemTime`
     no caminho puro (o caminho puro nunca chama `Utc::now()` — o timestamp vem do
     TS/`from_timestamp`). Precisa de `std` (ou `alloc`) p/ `.format()`/`.to_string()`.
   - Forma recomendada (**weak-feature**, mantém o clock só no nativo — byte-a-byte):
     `chrono = { version="0.4.45", default-features=false, features=["serde","std"],
     optional=true }`; `ai-pure = ["dep:serde_json", "dep:chrono"]`; `embedded =
     ["ai-pure", "chrono/clock", "dep:directories", "dep:reqwest", "dep:rusqlite",
     "dep:serde_json", "dep:tempfile", "dep:toml"]` (o `chrono/clock` reintroduz o clock
     que `research`/`WikipediaProvider` usa no nativo — **comportamento nativo
     inalterado**). O Executor **confirma na fonte** o conjunto mínimo de features de
     chrono que faz `WebSource` + `from_web_results` compilarem no wasm.
   - `default = ["embedded"]` **intacto**; `ai-pure` **fora do default**.
2. **`lib.rs` do core — provavelmente NADA** (`pub mod ai` já sob `any(embedded,
   ai-pure)`). Se algum re-export de crate-root precisar (não esperado), é a única linha.
