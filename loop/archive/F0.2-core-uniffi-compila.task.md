## Objetivo

Criar o crate Rust `core/` (nome de pacote sugerido: **`the-light-app-core`**)
com **uma função UniFFI trivial que apenas COMPILA**, estabelecendo o esqueleto
da fronteira e o **padrão de erro UniFFI** que as tarefas seguintes vão reutilizar.

Esta tarefa **NÃO** tem lógica de produto e **NÃO** depende do `the-light-core`
ainda. Ela prova exclusivamente que um crate Rust com fronteira UniFFI compila,
formata e passa no clippy/test neste ambiente. O `parse_reference` real e a
delegação ao `the-light-core::reference` são da **F0.3** (ver transferência
abaixo).

## Contexto / arquivos

- Árvore-alvo (seção 1 do `IMPLEMENTATION_PLAN.md`): `core/Cargo.toml`,
  `core/src/lib.rs`. O diretório `core/` e `core/src/` já existem (com `.gitkeep`),
  criados na F0.1; **não há `core/Cargo.toml` ainda** — a guarda de escopo da F0.1
  reservou a criação dele para esta tarefa (F0.2). A partir da F0.2,
  `core/Cargo.toml` **deve** existir.
- A fronteira UniFFI pode usar **proc-macros** (`#[uniffi::export]` +
  `uniffi::setup_scaffolding!()`, modo *library*, sem `.udl`) **ou** um arquivo
  `.udl` + `build.rs`. Escolha o caminho que **compila limpo** neste ambiente e
  **registre a escolha** (nota no commit / mini-ADR em `DECISIONS.md`).
- **Ferramentas (F0.0):** o CLI gerador de bindings `ubrn`
  (`uniffi-bindgen-react-native`) **NÃO está instalado** — mas ele **não é
  necessário** nesta tarefa: a geração de bindings TS é da **F0.4**. A F0.2
  precisa apenas do **crate `uniffi`** (do crates.io) como dependência Rust, para
  `cargo build`/`test` compilarem a fronteira.
- **Versão do `uniffi`:** a F0.0/ADR-0001 deixaram a versão exata para ser
  **fixada no momento da instalação** (esta tarefa). Fixe uma versão explícita no
  `core/Cargo.toml` (ex.: `uniffi = "<X.Y>"`, sem `*`) e registre qual foi.
- **Rede em tempo de build/dev é permitida.** Offline-first/BYOK é uma regra do
  **PRODUTO em runtime**, não do desenvolvimento: baixar o crate `uniffi` do
  crates.io durante `cargo build` é legítimo. **Porém**, se **não houver rede**
  (nem cache local) para resolver/baixar o crate `uniffi`, isso é um **BLOQUEIO
  REAL**: o Executor deve reportar `blocked` (ferramenta/dependência indisponível)
  — **não** improvisar um stub falso de UniFFI nem fingir compilação.
- **Transferência registrada (resolução do `the-light-core` → F0.3):** o
  ADR-0002 anotou a *resolução real* do `the-light-core` (adicionar a git
  dependency pinada por commit e provar via `cargo metadata`) como "pendência a
  exercer na F0.2". **Esta tarefa transfere essa responsabilidade para a F0.3**,
  onde `parse_reference` de fato delega ao `the-light-core::reference`. Motivo: a
  definição autoritativa da F0.2 (backlog `PHASE-0.md`) é "crate com UniFFI
  compilando, **SEM LÓGICA**" — adicionar a dependência do core agora seria escopo
  da F0.3. Portanto a F0.2 **não** adiciona `the-light` como dependência; ela só
  prova a fronteira UniFFI vazia. O Executor deve anotar essa transferência no
  resultado.

## Passos sugeridos

1. Criar `core/Cargo.toml` com:
   - `[package] name = "the-light-app-core"`, `version`, `edition`.
   - `[lib] crate-type = ["lib", "cdylib", "staticlib"]` (necessário para a ponte
     nativa/WASM em tarefas futuras; já deixe pronto).
   - `[dependencies] uniffi = "<X.Y>"` com versão **fixada** (sem `the-light-core`).
   - (Opcional) `thiserror` para o enum de erro, se ajudar o padrão.
   - Se optar por `.udl`: a seção `[build-dependencies] uniffi = { ..., features = ["build"] }`
     e um `core/build.rs` com `uniffi::generate_scaffolding(...)`.
2. Criar `core/src/lib.rs` com:
   - Uma função UniFFI trivial, ex.: `#[uniffi::export] pub fn ping() -> String {
     "pong".to_string() }` (retorno **constante**, sem I/O, sem rede, sem lógica).
   - Um **enum de erro UniFFI mínimo** que estabelece o padrão da fronteira, ex.:
     `#[derive(Debug, uniffi::Error)] pub enum CoreError { ... }`. Para evitar
     warnings de `dead_code` sob `clippy -D warnings`, **exercite** o erro com uma
     função trivial que o usa, ex.: `#[uniffi::export] pub fn ping_checked() ->
     Result<String, CoreError> { Ok("pong".to_string()) }`.
   - Se usar modo *library* (proc-macros): `uniffi::setup_scaffolding!();`.
   - **Sem** nenhum `use the_light_core::...` e **sem** lógica de parsing.
3. Gerar/validar o scaffolding UniFFI conforme o fluxo escolhido (modo library via
   `setup_scaffolding!` **ou** UDL + `build.rs`). Não é necessário gerar bindings
   TS aqui (isso é F0.4).
4. Rodar, do diretório `core/`: `cargo fmt`, `cargo clippy -- -D warnings`,
   `cargo build`, `cargo test`. Tudo verde antes do commit.
5. Commitar com prefixo do ID, ex.:
   `feat(F0.2): crate core com fronteira UniFFI trivial (ping) compilando`.

## Critério de aceite

- [ ] `core/Cargo.toml` **existe** (a guarda anti-escopo da F0.1 deixa de valer) e
  declara o pacote `the-light-app-core` (ou outro nome escolhido, registrado).
- [ ] `core/src/lib.rs` exporta via UniFFI uma **função trivial** (`ping` ou

- [ ] Um **padrão de erro UniFFI** está definido (enum derivando `uniffi::Error`)
  e usado por ao menos uma função (sem warnings de `dead_code`).
- [ ] `cargo build` do crate `core` fica **verde**.
- [ ] `cargo fmt --check` limpo e `cargo clippy -- -D warnings` **sem warnings**.
- [ ] `cargo test` do crate passa (mesmo que trivial/zero testes de lógica).
- [ ] **Guarda de escopo:** `core/Cargo.toml` **não** depende do `the-light-core`
  nem aponta git dep para o repo `the-light` (filtrando a linha do nome do pacote,
  que contém a substring `the-light`). A resolução real do core fica para a F0.3.
- [ ] A versão fixada do crate `uniffi` está registrada (no Cargo.toml e no
  resultado/commit).
- [ ] `git status --porcelain` vazio após o commit (working tree limpo).

## Comando(s) de verificação

```sh
set -e
ROOT="$(git rev-parse --show-toplevel)"

# 0) core/Cargo.toml passa a existir nesta fase
test -f "$ROOT/core/Cargo.toml" \
  && echo "OK  core/Cargo.toml existe (criado na F0.2)" \
  || { echo "FALTA core/Cargo.toml"; exit 1; }

# 1) Qualidade Rust do crate core (rodar de dentro de core/)
cd "$ROOT/core"
cargo fmt --check                         && echo "OK  cargo fmt --check limpo"
cargo clippy -- -D warnings               && echo "OK  cargo clippy sem warnings"
cargo build                               && echo "OK  cargo build verde"
cargo test                                && echo "OK  cargo test verde"

# 2) Função UniFFI trivial e padrão de erro presentes no lib.rs
grep -qE "uniffi::export|\[uniffi::export\]" "$ROOT/core/src/lib.rs" \
  && echo "OK  função exportada via UniFFI presente" \
  || { echo "FALTA #[uniffi::export] em core/src/lib.rs"; exit 1; }
grep -qE "uniffi::Error|derive\(.*Error" "$ROOT/core/src/lib.rs" \
  && echo "OK  padrão de erro UniFFI definido" \
  || { echo "FALTA enum de erro UniFFI em core/src/lib.rs"; exit 1; }

# 3) Guarda de escopo: SEM dependência do the-light-core ainda (isso é F0.3).
#    O nome do pacote 'the-light-app-core' contém 'the-light'; por isso filtramos
#    primeiro QUALQUER linha que contenha o nome do pacote e só então procuramos
#    referência ao the-light (dep crate / git dep do repo the-light).
cd "$ROOT"
if grep -vF 'the-light-app-core' core/Cargo.toml | grep -q 'the-light'; then
  echo "FALHA escopo: core/Cargo.toml referencia o the-light (dep pertence à F0.3):"
  grep -vF 'the-light-app-core' core/Cargo.toml | grep 'the-light'
  exit 1
else
  echo "OK  core/Cargo.toml não depende do the-light(-core) ainda (transferido p/ F0.3)"
fi

# 4) Versão do uniffi fixada (sem '*')
grep -qE '^[[:space:]]*uniffi[[:space:]]*=' core/Cargo.toml \
  && ! grep -qE '^[[:space:]]*uniffi[[:space:]]*=[[:space:]]*"\*"' core/Cargo.toml \
  && echo "OK  dependência uniffi presente e versionada (sem '*')" \
  || { echo "FALTA dependência uniffi com versão fixa em core/Cargo.toml"; exit 1; }

# 5) Working tree limpo após o commit
test -z "$(git status --porcelain)" \
  && echo "OK  working tree limpo" \
  || { echo "FALHA working tree sujo:"; git status --porcelain; exit 1; }

echo "F0.2 verificada."
```

## Restrições

- **Não tocar nem forkar o repositório `the-light` / `../the-light`.** Esta tarefa
  não consome o core; qualquer necessidade futura de alterar o core é PR + ADR.
- **Escopo estrito (SEM lógica):** nada de `parse_reference`, parsing ou I/O — só a
  função trivial e o padrão de erro. `parse_reference` + delegação ao
  `the-light-core::reference` e a **resolução real da git dep** são da **F0.3**.
- **Não adicionar `the-light-core` como dependência ainda.** A guarda de escopo da
  verificação prova isso.
- **Offline-first / BYOK:** regra de **runtime do produto**, não do dev. Baixar o
  crate `uniffi` do crates.io durante o build é permitido. Nenhuma chamada de rede
  pode ser introduzida no **código** do crate (a função é constante).
- **Bloqueio legítimo, não improviso:** se faltar **rede/cache** para resolver o
  crate `uniffi`, ou faltar tooling indispensável para o scaffolding UniFFI
  compilar, **reporte `blocked`** com o motivo exato — **não** crie um stub falso
  nem finja que a fronteira UniFFI compilou.
- **Qualidade obrigatória:** `cargo fmt`/`cargo clippy -- -D warnings`/`cargo
  build`/`cargo test` verdes antes do commit.
- **Segredos nunca em git**; nada de chaves, tokens ou `.env` no crate.
