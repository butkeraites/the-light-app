# F2.6 — Validação REAL do `ask` (⛔ GATE / BLOQUEANTE)

> **Precisa da CHAVE do usuário + rede opt-in.** A chave é um **segredo**: NUNCA
> pode entrar no chat, no git, nem em log/transcript de subagente. Por isso a
> validação é feita **pelo humano**, com a chave numa **variável de ambiente** do
> seu próprio terminal; o loop só recebe o **RESULTADO** (a interpretação, que não é
> segredo). Desfecho do ciclo = **HALT** até o humano validar.

## O que já está pronto (não-bloqueante)
- Fronteira: `ask_anchored`/`ask_anchored_stream` (F2.1/F2.3a) + GeminiProvider no
  core (`133077a`, F2.3).
- Harness de host **`core/examples/ask_real.rs`** (compila offline, fmt/clippy limpos):
  lê `AI_PROVIDER`/`AI_API_KEY`/`AI_MODEL`/... de env vars, chama `ask_anchored` de
  verdade e imprime `cited_text` (do STORE) + `interpretation` (do LLM). **A chave
  nunca é impressa** (o harness só mostra "(presente — não exibida)").

## Como o humano valida (a chave fica só na env var — nunca no chat/git/log)
No repo `/Users/butkeraites/Documents/the-light-app`, no SEU terminal (ou via `! <cmd>`):

```
AI_PROVIDER=gemini AI_API_KEY="<sua-chave-Gemini>" \
  cargo run --manifest-path core/Cargo.toml --example ask_real
```
(troque `AI_PROVIDER` por `anthropic`/`openai` e a chave conforme o provedor;
opcional `AI_MODEL=gemini-2.0-flash`, `AI_QUESTION="..."`.)

## Critério de aceite (o humano confirma e reporta)
- [ ] O comando retorna uma **INTERPRETATION real** do provedor (não a resposta do mock).
- [ ] O **CITED_TEXT** impresso é **João 3:16 verbatim do STORE** ("For God so loved
      the world…") — prova de anti-alucinação (texto bíblico do banco, LLM só interpreta).
- [ ] Nenhum erro de chave/rede; a chave não apareceu em nenhum output (só
      "(presente — não exibida)").
- [ ] O humano reporta ao loop: **provedor usado + "funcionou" + (opcional) o texto da
      interpretação** — SEM colar a chave.

## Se falhar
- Erro de auth/quota/modelo → reportar a mensagem (o harness NÃO imprime a chave; os
  erros do core trazem só nome do provedor/mensagem HTTP). O Driver ajusta (ex.: modelo
  default, endpoint) num ciclo focado; se for limite do provedor (ex.: Gemini exige
  billing/faturamento), registrar e decidir provedor alternativo.

## Restrições
- **Chave nunca no chat/git/log** (só env var no terminal do humano; o loop recebe só o
  resultado). Nenhum subagente recebe a chave.
- Rede só nesta validação opt-in, com a chave do usuário. Offline-first intacto (o resto
  do app não exige rede).
- `the-light`/core não mudam nesta tarefa (`133077a`); é só rodar o harness.

## Desfecho
Humano valida e reporta → o Driver marca F2.6 aceita, arquiva, e segue para **F2.7**
(paridade web de IA — que começa pelo PR D2/wasm ao core) e depois **F2.8 (Marco 2)**.
Enquanto não houver validação, **HALT**.
