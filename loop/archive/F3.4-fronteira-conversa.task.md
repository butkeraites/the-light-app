## Objetivo

Expor na fronteira UniFFI (`core/src/lib.rs`) a **conversa ancorada com follow-up**
e o **refinamento de escopo**, **delegando** a `the_light_core::ai::{ask_session,
refine_scope, parse_refinement}` (rev pinado `c8ecb2f`). Três funções:

1. **`ask_session_anchored(...)`** — conversa multi-turno sobre uma passagem/estudo,
   mantendo o **contexto ancorado** (montado do **store local**: capítulo numerado +
   rótulos de xref); devolve a **interpretação** (LLM) separada do **texto citado**
   (store). Delega a `ai::ask_session`.
2. **`refine_scope(...)`** — a partir de um assunto (`brief`) + respostas anteriores,
   pede ao provedor UMA rodada de refinamento (pergunta + opções) e a devolve
   estruturada (`Refinement`). Delega a `ai::refine_scope`.
3. **`parse_refinement(raw)`** — parser **puro, determinístico, SEM provedor**: uma
   string de rodada (`PERGUNTA:` + `- opção`) → `Refinement`. Delega a
   `ai::parse_refinement`.

**Prova determinística por MOCK** — sem chave e sem rede. Molde F2.1 (`ask_anchored`)
+ F3.3 (`deep_study`). Só o **NÚCLEO** (fronteira + teste de host): a UI de conversa é
F3.6; a validação real com chave é F3.10; a paridade web é F3.12.

### Investigação da fonte do core (`c8ecb2f`, só-leitura via Read) — assinaturas/tipos REAIS

- **`ask_session` — PURO/ai-pure** (`ai/study.rs:626`; re-exportado na superfície
  **pura** `ai/mod.rs:27-30`, **NÃO** `#[cfg(embedded)]`):
  ```rust
  pub fn ask_session(
      provider: &dyn LlmProvider,
      lang: Lang,
      context: &str,
      turns: &[ChatMessage],
      study: Option<(StudyMode, Denomination)>,
  ) -> Result<String>
  ```
  Chama `provider.chat(&system, &messages)` (`system` = `study_followup_system_prompt`
  quando `study.is_some()`, senão `ask_system_prompt`). **Invariante do core:** embute o
  `context` **apenas na PRIMEIRA** mensagem de usuário (`ask_user_prompt`); os turnos
  seguintes ficam como diálogo puro. Devolve **só a `String`** da interpretação (o texto
  citado/âncora é o `context`, que **a fronteira monta do store**).
- **`refine_scope` — PURO/ai-pure** (`ai/study.rs:411`; re-exportado puro em `mod.rs:28`):
  ```rust
  pub fn refine_scope(
      provider: &dyn LlmProvider,
      mode: StudyMode,
      lang: Lang,
      brief: &str,
      prior: &[(String, String)],   // pares (pergunta, resposta) anteriores
      round: u8,                    // 1..=3
  ```

  **CORREÇÃO do backlog:** o rascunho supôs `refine_scope` "pura sem provider" — **é
  FALSO**. `refine_scope` **RECEBE um provider** (`provider.complete(&system,&user)` gera
  a rodada, depois `parse_refinement` a analisa). Note: **NÃO tem parâmetro `lens`** (só
  `mode`). Quem é **determinística SEM provider** é a `parse_refinement` abaixo.
- **`parse_refinement` — PURO/ai-pure, SEM provider** (`ai/study.rs:346`; re-export puro
  `mod.rs:28`): `pub fn parse_refinement(raw: &str) -> Refinement`. Parser tolerante:
  `PERGUNTA:`/`Pergunta:` (ou a 1ª linha não-opção) vira a pergunta; linhas `- …`/`* …`
  viram opções (**deduplicadas, sem vazias**). Determinística → prova sem provedor.
- **`Refinement` — PURO** (`ai/study.rs:70`, `#[derive(Debug,Clone,PartialEq,Eq)]`, **sem
  `cfg`**; re-export puro `mod.rs:29`): `pub struct Refinement{ question: String, options:
  Vec<String> }`.
- **`ChatMessage`/`ChatRole` — PUROS** (`ai/mod.rs:339`/`ai/mod.rs:320`, **sem `cfg`**;
  acessíveis `the_light_core::ai::{ChatMessage,ChatRole}`): `ChatMessage{ role: ChatRole,
  content: String }`; `ChatRole{ User, Assistant }` (`as_str()`→`"user"`/`"assistant"`).
- **`LlmProvider::chat` — PURO** (`ai/mod.rs:380`, default no trait): dobra o histórico
  num transcript e chama `complete`. **`MockLlmProvider` NÃO sobrescreve** `chat` → com
  MOCK, `ask_session` → transcript → `complete` → **resposta FIXA** (o `system` de
  `ask_session` **não** contém `"PERGUNTA:"`, então o mock devolve `self.response`).
- **`MockLlmProvider` — PURO/ai-pure** (`ai/mod.rs:399`, **sem `cfg`**;
  `MockLlmProvider::default()` → `name()=="mock"`, `model()=="mock-1"`). **CRÍTICO:** no
  `complete`, se `system.contains("PERGUNTA:")` devolve uma **rodada canônica**
  (`PERGUNTA: Qual recorte você prefere?\n- Efésios 2.8-9\n- Panorama temático da graça\n
  - Aplicação pastoral`) — e o `refine_system_prompt` **contém** `"PERGUNTA:"` → logo

- **`build_provider` — EMBEDDED-only** (`ai/mod.rs:38`, `#[cfg(embedded)]`): providers
  reais (`anthropic`/`openai`/`gemini`, via `reqwest`) **não existem no wasm**. O

### DECISÃO DE GATING (byte-verificada; justificativa por função)

O rascunho do backlog afirma "ask_session/refine_scope PURAS → nativo+web sem PR".
**Confirmado byte a byte**, com a nuance da fonte do provider e da âncora:

- **`parse_refinement(raw) -> RefinementOut`: CFG-FREE.** Puro, sem provider, sem store →
  compila **e roda** no wasm E no nativo. **ANTECIPA** a paridade web do parser (zero PR,
  zero stub).
- **`refine_scope(...) -> RefinementOut`: mock CFG-FREE, provider real `cfg(not(wasm32))`.**
  `ai::refine_scope` é pura e **NÃO toca o store**; o único ponto embedded é a **criação
  do provider real** (`build_provider`). Logo: para `provider_name == "mock"`, instanciar
  **`MockLlmProvider::default()` direto** (ai-pure) → **corpo CFG-FREE** → o caminho MOCK
  **compila E roda no wasm** (ANTECIPA a paridade web do refinamento). Para provider real,
  `build_provider(...)` sob `cfg(not(wasm32))`; no wasm com provider real → `CoreError`
  stub (transporte `fetch` = F3.12, molde F2.7b). Records `RefinementOut`/`RefinementRound`
  + `From` são **cfg-free**.
- **`ask_session_anchored(...) -> AiAnswer`: corpo `cfg(not(wasm32))` + stub web** (molde
  F3.3). Embora `ask_session` **em si** seja pura, a **âncora** (`context`) é montada do
  **store nativo** (`Store::open` + `EmbeddedSource::passage` + `numbered_passage` +
  `xref::passage_labels` — `rusqlite`, embedded). No wasm o store é `wa-sqlite`/OPFS via
  TS → a paridade web (F3.12) montará o `context` do store web e chamará a **`ask_session`
  pura** (exatamente como a F2.7b passou versículos verbatim). Records `ChatTurn`/`ChatRole`
  + `From` são **cfg-free**; o retorno reusa o `AiAnswer` (já existe).

**Resumo:** grafo wasm **puro** (só símbolos ai-pure: `ask_session`, `refine_scope`,
`parse_refinement`, `ask_context`, `numbered_passage`, `MockLlmProvider`, `ChatMessage`/
`ChatRole`, `Refinement` — nenhum arrasta `reqwest`/`rusqlite`). **NÃO** force cfg-free
onde a realidade exige embedded (`ask_session_anchored` = store nativo).

**Conclusões:** F3.4 depende **só de F3.3** (aceita, `49919c3`); é **NÃO-BLOQUEANTE** (só
`mock`, sem chave/rede); **NÃO exige PR ao the-light** (`ask_session`/`refine_scope`/
`parse_refinement`/`Refinement`/`ChatMessage`/`ChatRole`/`MockLlmProvider` já `pub` e
ai-pure em `c8ecb2f`; `build_provider` `pub` sob embedded). ADR-0023/0024/0026
(anti-alucinação; ai-pure vs embedded; léxico STEP CC-BY) valem sem mudança de escopo.

## Restrições

- **NÃO tocar o `the-light`.** `ask_session`/`refine_scope`/`parse_refinement`/`Refinement`/
  `ChatMessage`/`ChatRole`/`MockLlmProvider` já são `pub` e **ai-pure**, e `build_provider`
  é `pub` sob `embedded`, em `c8ecb2f`. Se algo exigir função **não-`pub`** ou wasm-safe
  inexistente hoje → **`blocked` + HALT** (seria PR molde F2.7/F3.11), **não** stubar/forkar/
  copiar/reimplementar.
- **NÃO reimplementar** a lógica de conversa/refinamento (sem `ask_context`/`numbered_passage`/
  transcript de `chat`/parsing de `PERGUNTA:` à mão): **delegar** ao core. A fronteira só
  **monta** o contexto (âncora) a partir do store e **adapta** tipos/erros.
- **Anti-alucinação (LEI):** o `cited_text` (âncora) vem do **store local verbatim** (o LLM
  só conversa); `AiAnswer` **separa** `cited_text` de `interpretation`. No refinamento não há
  texto bíblico (é escopo — pergunta + opções); ainda assim, nada de fabricar versículos.
- **SEM rede e SEM chave:** só o provedor `"mock"` (`key: None`). A chave real/rede é a F3.10
  (gate). Nada de `ai::research` (pesquisa web = F3.9).
- **Gating conforme a REALIDADE (não force cfg-free onde exige embedded):** `parse_refinement`
  cfg-free; `refine_scope` mock cfg-free / provider real `cfg(not(wasm32))` + stub web;
  `ask_session_anchored` corpo `cfg(not(wasm32))` (âncora do store nativo) + stub web
  (paridade web = F3.12). Usar **`MockLlmProvider::default()` direto** no caminho mock (NÃO
  `build_provider("mock",…)`, que é embedded-only e quebraria o caminho cfg-free/wasm).
  **Grafo wasm puro** (sem `rusqlite`/`reqwest`).
- **offline-first**; uniffi `=0.31.0` e o pin `c8ecb2f` intactos; `core/Cargo.toml`
  inalterado. Sem ADR novo (consome o core pinado; próximo ADR livre = ADR-0027).
