## Resumo

A fronteira (`core/src/lib.rs`, único arquivo tocado, +450) passa a **rotear
`"gemini"`** e a **expor streaming** via `ask_anchored_stream(...)` com o callback
UniFFI `AiTokenCallback`, delegando ao `LlmProvider::complete_stream` do core
(the-light pinado em `133077a`). O contexto ancorado é montado pela **mesma rota**
da `ask_anchored`/F1.2 (store → `numbered_passage` → `ask_context`) — **nada** de
prompt/RAG/SQL é reimplementado. A **anti-alucinação** é preservada e provada: o
`cited_text` vem **verbatim do store** (segue o versículo do fixture), enquanto os
**tokens** transmitidos são a **interpretação do modelo/mock** (invariantes ao texto
bíblico), provado por 3:16 vs 3:17. Roteamento `"gemini"` alcança o arm de
`build_provider` **sem rede** (no-key → `CoreError` cuja mensagem contém `"gemini"`),
tanto em `ask_anchored` quanto no caminho de streaming. Corpo `cfg(not(wasm32))` +
**stub web**; grafo wasm permanece **puro** (sem `rusqlite`/`reqwest`) e o
`core/Cargo.toml` (re-pin da F2.3) **não** foi alterado.

**Linha-chave:** streaming → tokens acumulados == `interpretation` == resposta fixa
do `MockLlmProvider`; `cited_text` == KJV verbatim do store (João 3:16).

### 3. Anti-alucinação / delegação (leitura de lib.rs)

`ask_anchored_stream` (lib.rs:1340-1438) monta o contexto pela **mesma rota** da
`ask_anchored`: `parse_reference` → `Store::open` → `EmbeddedSource::passage` →
`ai::numbered_passage` (o `cited_text`) → `ai::ask_context(&label, &cited_text, &[])`;
depois chama `provider.complete_stream(&context, &question, &mut sink)` do core — **não**
reimplementa prompt/RAG/SQL. O `sink` repassa `tok` a `on_token.on_token(...)` (tokens da
**interpretação**). O `AiAnswer` separa `cited_text` (store) de `interpretation` (modelo).
Corpo `cfg(not(target_arch="wasm32"))` + stub web (F2.7) que consome os args. **Nenhum
texto bíblico hardcoded** na produção: os literais KJV (`"For God so loved..."`) só
aparecem em `const`s dentro de módulos `#[cfg(test)]` / `#[cfg(all(test,
not(target_arch="wasm32")))]` (linhas 1505, 1578-1580, 1805-1807, 2491-2494, 2745).

## Decisões / ADRs

- **Sem ADR novo.** A tarefa **consome** o re-pin da F2.3 (the-light `133077a`:
  `GeminiProvider` + `LlmProvider::complete_stream`); `core/Cargo.toml` não muda.
- **Streaming exposto via callback UniFFI sobre o default de `complete_stream`:**
  `#[uniffi::export(callback_interface)] AiTokenCallback` + `ask_anchored_stream` chamam
  `complete_stream` do core. O default do core é **não-quebrante** (emite a resposta
  inteira 1× e devolve a `String` completa) — com o mock rende exatamente a resposta
  canônica de `ai::ask`. Provedores de rede reais (F2.6) sobrescrevem com SSE **sem mudar**
  esta fronteira.
- **Nota (mapeamento system/user):** o core **não** expõe `ask_stream` público nem o
  construtor do prompt de `ai::ask`; para não reimplementar a lógica do core (regra "uma
  fonte da verdade"), a fronteira compõe apenas peças públicas — `system` = bloco de
  contexto ancorado (`ask_context`, derivado do store) e `user` = `question`. A **paridade
  fina de prompt** com provedores de rede é **F2.6**. A anti-alucinação é preservada: o
  `cited_text` é do store; o LLM só interpreta.
