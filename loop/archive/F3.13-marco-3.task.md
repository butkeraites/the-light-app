## Regras não negociáveis — TODAS mantidas
- **Anti-alucinação (LEI):** o texto do versículo e o **léxico** vêm sempre do store local
  verificado; o LLM só interpreta; âncoras `[V:Strong]`/`[W:n]` inválidas são **descartadas
  pelo core**; `passage_text`/`citations` separados de `interpretation`. **ZERO DRIFT
  nativo↔web** — prompt/citação/aparato/`verify` são a MESMA impl Rust (`ai-pure`), provado
  por testes de paridade `assert_eq!` (estudo e conversa).
- **BYOK / chave nunca em git/log:** Keychain nativo (F2.4) / session-only web (ADR-0025);
  validação real via env var no terminal do humano. Login-de-conta OAuth rejeitado (ADR-0023).
- **Offline-first:** IA e pesquisa web são **opt-in aditivos**; leitura/busca/xref/notas +
  léxico/export seguem offline; rede só no `fetch`/HTTP com a chave (ou Wikipedia keyless opt-in).
- **`the-light` só via PR+ADR:** **3 PRs sancionados** na Fase 2/3 (Gemini+streaming →
  `133077a`; ai-pure ask → `c8ecb2f`; ai-pure study → `04b9b24`), cada um mergeado por
  sign-off + re-pin; **nunca forkado**. Grafo wasm sempre puro (sem rusqlite/reqwest/wasm-bindgen).
- **Licenciamento:** léxico STEP Bible **CC-BY 4.0** com atribuição visível; xref OpenBible
  CC-BY; texto domínio público. Qualidade por tarefa verde + review independente.
