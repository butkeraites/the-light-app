## Objetivo

Habilitar o **2º backend de pesquisa web — Tavily (BYOK)** no estudo profundo
nativo, **sem tocar o `the-light`**. Hoje a fronteira `deep_study`
(`core/src/lib.rs`) já roteia `research_backend: Option<String>` (F3.9a) mas chama
`build_research_provider(&backend, None, lang.code())` com a **chave `None`
HARD-CODED** (`lib.rs:1438`) — o que basta para `"wikipedia"` (keyless) e `"mock"`
(sem rede), mas **impede** o Tavily, que **exige a chave** (`build_research_provider`
retorna `AiError::NoKey("research.tavily")` se `None`). Esta tarefa faz o **threading
de uma chave de pesquisa**: adiciona um parâmetro **final** `research_key:
Option<String>` que é repassado a `build_research_provider(&backend, research_key,
lang.code())`.

Semântica resultante (o core já implementa; a fronteira só encaminha a chave):
- `research_backend = None` → sem pesquisa (`web_sources = vec![]`, offline; F3.9a
  preservado). **Padrão desligado.**
- `research_backend = Some("mock")` (chave ignorada) → `MockResearchProvider::canned()`
  (**sem rede**, 2 `WebSource` fixos) — a **prova determinística** desta tarefa
  (comportamento F3.9a intacto, com ou sem `research_key`).
- `research_backend = Some("wikipedia")` → `WikipediaProvider` (**keyless**, a chave é
  **ignorada** pelo core; F3.9a preservado) — rede opt-in do usuário (F4.5).
- `research_backend = Some("tavily")`:
  - `research_key = Some(<chave>)` → `TavilyProvider::new(key)` (BYOK, endpoint
    `https://api.tavily.com/search`) — **rede opt-in do usuário, validada na F4.5**
    (NÃO exercitada neste teste).
  - `research_key = None` → `CoreError` (espelha `AiError::NoKey`) — **sem rede, sem
    panic**: a prova negativa desta tarefa.

**NÃO-BLOQUEANTE** (só MOCK, sem chave/rede real) e **NÃO exige PR** — toda a superfície
de `research` (`build_research_provider`/`TavilyProvider`/`ResearchProvider`/`WebSource`)
já é `pub` sob `#[cfg(feature="embedded")]` em `04b9b24`. **NÃO toca o `the-light`.**

## Restrições

- **NÃO tocar o `the-light`.** `build_research_provider`/`TavilyProvider`/
  `ResearchProvider`/`WebSource`/`MockResearchProvider` já são `pub` sob `embedded` em
  `04b9b24`. Esta tarefa é **só o threading de `research_key` na fronteira** (app-side).
  Se algo exigisse símbolo **não-`pub`** ou mudança no core → **`blocked`/HALT** (não
  stubar/forkar/copiar). O pin `04b9b24`, `uniffi=0.31.0` e `core/Cargo.toml` **não**
  mudam.
- **BYOK — a chave nunca em git/log.** `research_key` é repassada **direto** a
  `build_research_provider` (que a coloca só no corpo do `fetch` Tavily, dentro do
  core). **Nunca** logar/imprimir/versionar a chave. O teste usa `None` ou um valor
  **fictício** — **jamais** uma chave Tavily real (validação real = F4.5).
- **Tavily = rede OPT-IN.** Nenhuma chamada de rede no CI/teste: o caminho Tavily é
  exercitado **só** no ramo `NoKey` (chave `None` → erro ANTES de qualquer HTTP). A
  busca Tavily real (com a chave do usuário) é a **F4.5** (gate, bloqueante).
- **Wikipedia keyless preservado (ADR-0028).** Para `"wikipedia"` a chave é ignorada
  pelo core; F3.9a segue intacto. **Offline-first:** padrão `None` → sem rede.
- **Anti-alucinação.** `passage_text` e as citações (`[W:n]`) vêm **do store / das URLs
  buscadas**, nunca do modelo; `[W:n]` fora do intervalo é sinalizado pelo core. **Não**
  reimplementar montagem de prompt/citação/busca (vive no core).
- **Grafo wasm puro.** A pesquisa/Tavily (`research`, embedded-only/`reqwest`) fica
  **dentro** do bloco `cfg(not(target_arch="wasm32"))`; o stub web só ignora o novo
  param (a busca Tavily web por `fetch` + chave session-only + UI toggle é a **F4.4**).
- **Escopo atômico:** só o threading `research_key → build_research_provider` + testes
  MOCK + higiene TS/bindings. **Sem UI de Tavily** (F4.4), **sem** validação real
  (F4.5), **sem** PR. Registrar **ADR-0035** só se formalizar a decisão do threading da
  chave de pesquisa (Tavily BYOK).
