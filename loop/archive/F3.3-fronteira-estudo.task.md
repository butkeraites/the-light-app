## Objetivo

Expor na fronteira UniFFI (`core/src/lib.rs`) a função **`deep_study`**, que produz
um **estudo profundo** de uma passagem (modo × lente × profundidade) **delegando** a
`the_light_core::ai::study::study(&provider, &StudyRequest)` (rev pinado `c8ecb2f`).
A fronteira **monta** o `StudyRequest` com FATOS do **store local** (passagem, léxico
verificado, rótulos de xref) e chama o `study` do core, que separa `passage_text`
(banco, verbatim) de `interpretation` (LLM). **Prova determinística por MOCK**
(`build_provider("mock", None, None)`) — **sem chave e sem rede**. Molde F2.1
(`ask_anchored`) + F3.2 (`lexical_entries`). Só o **NÚCLEO** (fronteira + teste de
host): a UI é F3.5; a pesquisa web é F3.9; a validação real com chave é F3.10; a
paridade web é F3.12.

- **NÃO tocar o `the-light`.** `study`/`StudyRequest`/`StudyResult`/`build_provider`/
  `xref::passage_labels`/`verified_lexicon`/`format_reference` já são `pub` sob

  **wasm-safe** que não existe hoje → **`blocked` + HALT** (seria PR molde F2.7/F3.11),
  **não** stubar/forkar/copiar/reimplementar.
- **NÃO reimplementar** prompt/RAG/estudo/SQL/citação (sem `system_prompt`,
  `user_prompt`, `numbered_passage` à mão, sem SQL de léxico/xref/passagem): **delegar**
  ao core. A fronteira só **monta** o `StudyRequest` a partir dos fatos do store e
  **adapta** tipos/erros.
- **Anti-alucinação (LEI):** `passage_text` e as `citations`/léxico vêm do **store
  local verificado**; o LLM/mock só **interpreta** (`interpretation`); os `warnings`
  do core sinalizam Strong/`[W:n]` inventados. `StudyResultOut` **separa**
  `passage_text` de `interpretation`.
- **SEM rede e SEM chave:** só o provedor `"mock"` (`key: None`). A chave real/rede é a
  F3.10 (gate); a pesquisa web é a F3.9 (gate) → `web_sources: vec![]`.
- **Só nativo:** corpo DB/IA `cfg(not(target_arch="wasm32"))` + **stub web**; o estudo
  web é a F3.12. **Grafo wasm puro** (sem `rusqlite`/`reqwest`).
