## Resumo

A fronteira `the-light-app-core` foi re-alinhada de `uniffi = "0.31.2"` para o pin
EXATO `uniffi = "=0.31.0"`, casando com `uniffi_core = "=0.31.0"` que o runtime web
do `ubrn 0.31.0-3` (`uniffi-runtime-javascript`) exige. Isso elimina a causa-raiz
que reprovou a F0.6b (attempt1): no grafo do wasm-crate, dois pins exatos
divergentes (`=0.31.2` vs `=0.31.0`) sobre o mesmo `uniffi_core` eram
irreconciliáveis.

Verificação independente do bloco 1–5 da task: **toda verde**. Só a **versão** do
`uniffi` mudou — a **forma** da fronteira é preservada: `core/src/lib.rs` tem diff
vazio no commit (modo library, `setup_scaffolding!`, `parse_reference`, `CoreError`,
`crate-type` intactos). O commit 7b98644 toca apenas `DECISIONS.md`,
`core/Cargo.lock`, `core/Cargo.toml` (+ a task). O `the-light` **não** foi tocado: é
consumido como git dep PINADA — o rev `8f6600460c3680a537d2f5df81b6980ed7e630d5`
permanece inalterado em `core/Cargo.toml`. Resultado: **passed**.

## Decisões / ADRs

- **ADR-0006 (aceito, F0.6c):** pin `uniffi = "=0.31.0"` na fronteira para compat.
  com o runtime web do `ubrn 0.31.0-3` (`uniffi-runtime-javascript` fixa
  `uniffi_core = "=0.31.0"`). Alinha os dois caminhos do ubrn (bindgen `=0.31.0` no
  host — ADR-0004 — e runtime `=0.31.0` no web). Patch dentro de `0.31.x` (mesmo
  contract version), baixo risco; revalidação F0.2/F0.3/F0.4 verde sob 0.31.0.
- **ADR-0003 atualizado:** nota registrando o re-alinhamento `0.31.2` → `=0.31.0`.
- the-light **não** modificado (consumido como git dep pinada, rev 8f66004 intacto);
  offline-first preservado (rede só em dev/build para baixar `uniffi 0.31.0`).
