## Resumo

A F0.4 entrega um script reprodutível (`scripts/gen-bindings.sh`) que gera os
bindings TypeScript da fronteira UniFFI do crate `the-light-app-core` (`core/`)
via `uniffi-bindgen-react-native` (CLI `ubrn`) **fixado na versão exata
`0.31.0-3`** como devDependency npm (`package.json` + `package-lock.json`). O
fluxo é host-only / modo *library*: `cargo build --manifest-path core/Cargo.toml`
produz a cdylib do host, de onde o `ubrn generate jsi ... --library --no-format`
extrai a metadata e emite `bindings/the_light_app_core.ts` +
`the_light_app_core-ffi.ts` (mais o C++ JSI em `bindings/cpp/`). Verifiquei a
**reprodutibilidade de forma independente**: apaguei os `.ts`/`cpp` gerados
(restando só `.gitkeep`) e rodei o bloco de verificação 1–6 exato — o script
recriou os bindings do zero, exit 0, e o bloco terminou em "F0.4 verificada.". A
fronteira do core e a versão pinada `uniffi = "0.31.2"` (ADR-0003) **não** foram
tocadas; o `the-light` permanece intacto; nenhum binding foi escrito à mão;
nenhum segredo foi versionado. **Resultado: passed.**

## Decisões / ADRs

- **ADR-0004 (aceito, F0.4)** — Geração reprodutível dos bindings TS:
  - Ferramenta + versão exata: `uniffi-bindgen-react-native@0.31.0-3` (sem
    `^`/`~`/`*`), última publicada no npm.
  - Mecanismo: devDependency npm, fixada em `package.json` + `package-lock.json`
    (não `cargo install`); o wrapper `bin/cli.cjs` compila o binário `ubrn` sob
    demanda em `node_modules/` (ignorado). Rede só em dev/build — offline-first
    de runtime preservado.
  - Invocação host-only / modo *library*: `cargo build` da cdylib do host +
    `ubrn generate jsi bindings --library --no-format --ts-dir <ts> --cpp-dir
    <cpp> <cdylib>`, a partir de `core/`. `ubrn.config.yaml` é a fonte da verdade
    dos caminhos. **NÃO** se compila wasm/iOS/Android aqui (F0.6/7/8).
  - `tsc --noEmit`: **ADIADO** p/ F0.5 (os `.ts` importam o runtime `@ubjs/core`,
    que chega com o app Expo). Porta dura da F0.4 = script rodar limpo e popular
    `bindings/`.
  - Compatibilidade com `uniffi 0.31.2` (ADR-0003): o `ubrn 0.31.0-3` embute
    `uniffi =0.31.0`; a geração leu a metadata da cdylib `0.31.2` sem erro de
    contrato/checksum (mesmo contract version na série `0.31.x`). Confirmado
    empiricamente nesta revisão (geração regenerada do zero, exit 0). A fronteira
    do core e a versão pinada **não** foram alteradas.
  - Tabela de ambiente atualizada: gerador `ubrn` de "NÃO INSTALADO" → `0.31.0-3`
    (INSTALADO na F0.4).
