## Resumo

F6.8 (última da trilha de IA) habilita IA de nuvem no WEB via o header opt-in
oficial da Anthropic (`anthropic-dangerous-direct-browser-access: true`) SÓ no
transporte web, deixa a capacidade por-provedor HONESTA na tela de Ajustes (F6.6),
registra o ADR-0058 e FLIPA a âncora de regressão da F6.2 (`ai-reachability`):
a Anthropic passa de CORS-wall a ALCANÇADA no browser real. Verifiquei de forma
independente rodando todos os gates e inspecionando o diff. Tudo verde. A âncora
flipada é uma asserção REAL (não afrouxada): o smoke faz a requisição de browser
de verdade contra `api.anthropic.com` com chave DUMMY e observa 401 (CORS ok) —
se ainda estivesse murada (`net::ERR_FAILED`), a asserção flipada FALHARIA.

## Verificação da substância (adversarial)

1. **Header é o fix real + web-only.** `ai-anchored.web.ts` (transporte web) envia o
   header opt-in. `the-light/core` (reqwest) não contém o header — nativo intocado.
   É a via SANCIONADA da Anthropic p/ uso client-side com a chave do próprio usuário (BYOK).
2. **Flip é REAL, não asserção afrouxada.** O código antes só REGISTRAVA o estado
   ("CORS-wall é ESPERADO até a F6.8 → não falha"); agora percorre
   anthropic/openai/gemini e empurra um `problem` (→ throw → vermelho) se
   `reached = responses.length>0 && failures.length===0` for false. O smoke seleciona
   o provedor real, insere a chave DUMMY, submete e escuta `response`/`requestfailed`
   nos hosts REAIS (`api.anthropic.com` etc.). O 401 observado prova que o browser
   RECEBEU resposta HTTP da Anthropic (CORS liberado). Se ainda murado, `requestfailed`
   com `net::ERR_FAILED` → `reached=false` → vermelho. Verde em dev E dist = prova.
3. **Rótulos honestos.** Ollama-web = "Requer configuração local (OLLAMA_ORIGINS) — o
   navegador não alcança o Ollama sem isso" (NÃO prometido pronto); nuvem
   (anthropic/openai/gemini) = "Funciona no navegador"; nativo = "Funciona neste
   aparelho". Casa com o que a F6.2 mediu + o header habilita.
4. **Sem proxy / offline-first intacto.** O commit só adiciona um header + rótulos +
   testes + ADR. Nenhum servidor/proxy introduzido; o app chama o provedor DIRETO do
   cliente; recursos essenciais seguem sem rede/conta.
5. **Não negociáveis.** ADR-0058 presente e coerente (justificativa BYOK — o "dangerous"
   é expor a chave no cliente, que no BYOK já é a chave do usuário; sem proxy;
   reconcilia offline-first e anti-alucinação). Sem chave/segredo real. Nativo/core
   intocados; the-light @ `225b8c9` (working tree limpo). `.claude/settings.json` NÃO
   está no commit (modificado só no working tree). Commit `9011c32` toca apenas 6
   arquivos, todos em `app/` + `DECISIONS.md`.
