## Resumo

A camada de **store NO WEB** (Opção A do ADR-0011) está provada de forma
independente. O `getPassage` web (TS) resolve a referência **PELO RUST (wasm)**
— reusa `parseReference` (`app/web/reference.web.ts` → bindings gerados) — e lê o
**texto KJV verbatim** de João 3:16 de um **`wa-sqlite`** (build SYNC, sem
`SharedArrayBuffer`/COOP-COEP) rodando o **SELECT espelhado** de
`EmbeddedSource::passage` (`SELECT verse, text FROM verses WHERE
translation_id=? AND book_number=? AND chapter=? AND verse=? ORDER BY verse`)
sobre os bytes de `assets/data/sample.sqlite`. No browser o backend é OPFS
(persistência) + VFS de memória hidratado; a prova headless em node usa o **mesmo
VFS de memória + a mesma `queryPassage`/`readPassage`** de produção. O texto vem
**sempre do store local** — não há string bíblica no código de produção (a
constante KJV existe só na asserção do teste). Sem regressão (F0.6b web, F0.9
store nativo, `tsc`, `expo export`). `the-light` intacto e working tree limpo.

## Decisões / ADRs

- **ADR-0011 (Opção A):** store web = `wa-sqlite` + OPFS com a leitura de passagem
  em TS, **sem mudar o `the-light-core`**. `parse_reference` continua no Rust
  (uma fonte da verdade); o `SELECT` de passagem é infraestrutura espelhada.
- **ADR-0012 (subdecisões da F0.10):** `wa-sqlite@1.0.0` **build SYNC** (sem
  `SharedArrayBuffer`/COOP-COEP — evita o gatilho de bloqueio do export estático);
  query ISOLADA do VFS (`queryPassage`/`readPassage`) reusável em node e browser;
  empacotamento Metro do `.wasm` e do `.sqlite` como assets locais; **symlink**
  `app/assets/data/sample.sqlite → ../../../assets/data/sample.sqlite`
  (fonte única dentro do projectRoot, asset bundled byte-idêntico ao canônico);
  runtime browser = OPFS-persistência + leitura em VFS de memória hidratado
  (`createSyncAccessHandle` é Worker-only); prova headless honesta em node com VFS
  de memória sobre os mesmos bytes + a mesma query.

## Bloqueios / perguntas

Nenhum bloqueio. Notas de fronteira (esperadas e honestas):

- **OPFS é runtime browser-only** — não existe em node. A prova determinística
  cobre `parseReference` (Rust/wasm) + `queryPassage`/`readPassage` de produção
  sobre os mesmos bytes do `sample.sqlite` num VFS de memória; o caminho
  OPFS-ao-vivo (`sqlite-opfs.web.ts`) não é exercitado headless por design
  documentado no teste e no ADR-0012. Cobertura: query + leitura do sample.
- **Escopo Fase 0:** só a leitura de **passagem única** (Single). Store web com
  search/xref completos e VFS OPFS "ao vivo" em Worker ficam como evolução futura
  (reavaliar Opção B do ADR-0011) — fora do escopo de viabilidade desta tarefa.
