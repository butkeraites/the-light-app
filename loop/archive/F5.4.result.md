## Resumo

F5.4 expõe pela fronteira UniFFI a PERSISTÊNCIA do progresso do plano de leitura
ativo, delegando ao `the_light_core::userdata::plans::PlanStore` (fs) que grava o
único plano ativo em `<data_dir>/reading-plans/active.json`. Adiciona o Record
puro `ReadingPlanProgress { plan_id, start_date (ISO), completed }` (todos os
alvos) + `From<PlanProgress>` só no nativo, e 4 fns de fronteira (corpo nativo
`cfg(not wasm32)` + stub web `cfg(wasm32)`): `reading_plan_progress`,
`start_reading_plan`, `set_reading_plan_completed`, `clear_reading_plan`. Glue TS
nativo (delega) + stub web. Commit toca EXATAMENTE os 3 arquivos-produto
(`core/src/lib.rs`, `app/web/reading.ts`, `app/web/reading.web.ts`); bindings
regenerados são gitignored. Verificação independente: 100% verde.

## Decisões / ADRs

- **Desvio Result-wrapping (APROVADO).** O executor entregou
  `reading_plan_progress -> Result<Option<ReadingPlanProgress>, CoreError>` e
  `clear_reading_plan -> Result<bool, CoreError>` em vez dos `Option<...>`/`bool`
  crus esboçados nos Passos. Verdito: **aceitável e correto** — na verdade
  EXIGIDO pelas Restrições, que mandam "REUSE the F1.10 notes pattern exactly".
  O core `2fc2dab` define `PlanStore::load() -> Result<Option<PlanProgress>>` e
  `clear() -> Result<bool>` (plans.rs:137/153); mapear para os tipos crus obrigaria
  a `.unwrap()`/engolir erro → JSON corrompido/erro de I/O viraria `None`/`false`
  silenciosamente. O wrapping é propagação de erro estritamente melhor e idêntica
  ao molde de notas F1.10 (`get_note -> Result<Option<Note>>`, `delete_note ->
  Result<bool>`). O Record `ReadingPlanProgress` tem exatamente os 3 campos do
  core `PlanProgress` (sem campos extras — Restrição respeitada).
- **Glue TS `async` (razoável).** Diferente do F5.1 (geração = compute puro,
  sync), o F5.4 é I/O (fs) → wrappers `async` seguem o molde de notas F1.10.
  Assinatura idêntica entre glue nativo e stub web (zero drift). Sem ADR novo.
- **Sem drift de serialização/layout.** Grep confirmou que nem a fronteira nem
  os dois `.ts` reimplementam serde/day-index/escrita atômica; o único ponto de
  construção de caminho é `plan_store()` (`reading-plans/active.json`). O
  `NaiveDate` do core vira ISO `String` via `Display` (dep transitiva `chrono`
  não nomeada — `core/Cargo.toml` intocado).
