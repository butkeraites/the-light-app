## Resumo

F5.23 entrega o motor PURO `app/lib/userdataSnapshot.ts` — snapshot JSON
round-trippável dos dados do usuário (notas + marcações + progresso de plano) +
import-com-merge determinístico — mais a prova headless `test:web:snapshot`. É a

## Decisões / ADRs

- **ADR-0051** (novo, número livre; ponteiro bumpado p/ ADR-0052): formato do
  snapshot + política de merge. Aceito.
- **Veredito da regra de colisão (crux).** Os Records `Note`/`Highlight` genuinamente
  NÃO têm timestamp (verificado nas bindings geradas), então o "last-modified wins"
  literal do ADR-0036 é IMPOSSÍVEL. A adaptação "importado vence + `max(completed)`
  p/ progresso" é **ACEITÁVEL**: é (1) determinística (merge repetível → idêntico),
  (2) idempotente (`merge(s,s)==s`; re-import = no-op 0/0/false), (3) nunca regride
  progresso de plano (`max`), e (4) honestamente documentada em ADR-0051 com a
  justificativa explícita (import é a autoridade; sem campo de tempo p/ desempatar;
  `exportedAt` é informativo e não participa do merge). Uma melhoria futura (campo de
  timestamp) seria mudança no core — fora do escopo desta tarefa. Flag: em colisão
  de notas, o corpo local diferente é substituído pelo importado sem 3-way merge —
  aceitável dado o shape dos dados e o design "import aplica sobre o local".
- **Escopo (deferimento correto, não buraco).** F5.23 = motor puro + prova headless.
  Sem Drive OAuth/PKCE (F5.24), push/pull (F5.25), Share/file-picker/toggle UI +
  wiring do store nativo (F5.26). Confirmado no diff (5 arquivos, sem UI/auth/native
  store) e explicitado em ADR-0051. O motor é cross-target; o web já o exercita
  fim-a-fim na prova.
