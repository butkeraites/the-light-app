   `codeChallenge = base64UrlEncode(SHA-256(codeVerifier))` (driveAuth.ts L96-98). O
   teste usa o vetor OFICIAL RFC 7636 Ap. B: verifier
   `dBjftJeZ4CVP-mB92K27uhbUJU1p1r_wW1gFWFOEjXk` → challenge
   `E9Melhoa2OwvFrEMTJguCHaoeK1t8URWbuGJSstw-cM`. O mock `getRandomValues` injeta
   OCTETOS **derivados por decode BASE64URL do verifier**, e o challenge é a SHA-256
   **REAL** (`webcrypto.subtle.digest` do node) — NÃO uma string hardcoded igual a si
   mesma. `buildAuthUrl` inclui `response_type=code`, `code_challenge_method=S256`,
   scope `drive.appdata`, `state`, `redirect_uri` (+ `access_type=offline`/`prompt=consent`).
