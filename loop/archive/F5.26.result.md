## Checagens adversariais (as 8)

1. **Opt-in OFF por padrão (crítico) — REAL.** `syncPrefs.ts::getSyncOptIn()` retorna `raw === 'true'`;
   KV vazio → `getPref` = `null` → `false`. Desligar REMOVE a chave (volta ao default). A prova
   afirma o default sobre um KV genuinamente vazio (`assert kv.has(SYNC_OPTIN_PREF_KEY) === false`),
   round-trip enable→relê `true`→disable→relê `false`, e valor-lixo `'sim'` lê OFF. `optin_default_off=ok`
   e `optin_persist=ok` são substantivos. Toggle da UI usa `value={optIn === true}` (não truthy acidental).

2. **Opt-in OFF ⇒ ZERO rede — confirmado.** Boot lê só `getSyncOptIn()` (KV, sem rede). O 2º effect
   (link do Drive) só roda com `optIn === true && isWeb`; com OFF, seta `driveLinked=false` e retorna.
   A seção do Drive só renderiza com `optIn === true`. Nenhum `fetch`/`driveSync` eager: o único `fetch`
   está em `onDriveSyncNow`, gated por `!driveLinked`. `driveAuth.isLinked` só lê o TokenStore de sessão.

3. **`SnapshotStore` ligado ao store REAL — confirmado.** `snapshotStore.ts`/`.web.ts` injetam as fns
   de fronteira reais de `web/reading` (`listNotes`/`listHighlights`/`readingPlanProgress`/`putNote`/
   `addHighlight`/`startReadingPlan`/`setReadingPlanCompleted`), nome EN via `listBooks()` do core e
   validação REAL via `parseReference` síncrono ANTES de escrever. A prova headless usa o MESMO
   `createSnapshotStore` do produto sobre as `*Fs`/`*PlanFs` de produção (só o backend de storage é
   in-memory) — não há no-op silencioso em produção.

4. **Privacidade honesta + offline proeminente — reais, não TODOs.** `sync.offlineFirst` em banner de
   destaque ("funciona 100% offline sem isto / totalmente opcional"). `sync.privacySyncs` = notas+
   marcações+progresso. `sync.privacyNever` = sessões de IA, banco bíblico, chaves/segredos, texto de
   versículo (só a referência canônica viaja). `sync.noTelemetry` = "sem telemetria". Strings localizadas
   PT+EN completas.

5. **Nenhuma chamada real ao Google / nenhum segredo — confirmado.** `DRIVE_CLIENT_ID` vem de
   `EXPO_PUBLIC_GOOGLE_DRIVE_CLIENT_ID` (ausente → `DRIVE_CONFIGURED=false`); botão Connect tem
   `onPress={undefined}` + `disabled`. Token só no `sessionTokenStore` (memória). `grep` no repo por
   `client_secret`/`GOCSPX-`/`*.apps.googleusercontent.com`/`AIza...`/`ya29.` → NENHUM. Zero `console.*`
   nos arquivos F5.26 (nenhum vazamento de token/snapshot).

6. **Anti-alucinação / escopo de dados — confirmado.** UI roteia export por `exportSnapshot`/
   `serializeSnapshot` (F5.23), import por `importSnapshotIntoStore` (merge determinístico F5.23), sync por
   `createDriveSync` (F5.25). Snapshot carrega só referência canônica + corpo/cor/tag/progresso — nunca
   texto bíblico/sessão/banco/chave (a prova assere `!json.includes('For God so loved')`). Nenhum novo data path.

   driveAuth + driveSync + userdataSnapshot são `import()` sob demanda (chunks async, medidos separadamente).
