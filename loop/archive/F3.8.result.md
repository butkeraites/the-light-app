## Resumo

Exportação acadêmica (SBL → Markdown + sidecar de citações) + UI Share entregue via
**via (C)**: o Record `StudyResultOut` ganhou o campo `academic_markdown: String`,
populado no `From<the_light_core::ai::study::StudyResult>` (gateado
`cfg(not(target_arch = "wasm32"))`) por `r.to_academic_markdown(r.language)`. O
Markdown acadêmico (SBL) vem **inteiro do core** (`StudyResult::to_academic_markdown`,
`pub` porém `embedded`-only, callável na fronteira nativa) — **fonte única, zero
drift**, uma única chamada (reusa o estudo já rodado, importante p/ BYOK real na
F3.10). O app **não** reimplementa SBL/proveniência: `app/lib/studyExport.ts`
(`buildStudyExport`, função PURA molde F1.11) só **agrega** os `StudyCitation` +
`sources` do léxico + metadados no sidecar JSON. Botão "Exportar (acadêmico)"
(`testID=study-export-academic`) no `ReaderStudyPanel` → Share nativo `{markdown,
sidecar}`; web = stub (F3.12). FOLLOW-UP ADR-0027 **resolvido na raiz**: 9
doc-comments `**puro**/` → `**puro** /` em `core/src/lib.rs`. Prova host (MOCK, 60
testes) + prova headless no device (`TLA_EXPORT`, sem regressão). the-light intocado
(`c8ecb2f`); `core/Cargo.toml` inalterado; grafo wasm puro.

Verificação **independente reproduzida do zero** — toda verde.

## Decisões / ADRs

- **Via (C) — `academic_markdown` do core no `StudyResultOut`.** O Markdown SBL é
  produzido pela mesma impl do core (`StudyResult::to_academic_markdown`, `pub`
  `embedded`-only) e exposto como campo do Record. **Fonte única / zero drift**
  (nada de SBL reimplementado no app) e **uma única chamada** (reusa o estudo já
  rodado) — decisão importante para o **BYOK real da F3.10**, onde a via (A)
  re-rodaria `study()` = 2ª chamada paga. Campo `String` inócuo no wasm (o stub web
  de `deep_study` retorna `CoreError`, nunca constrói o Record; `From` gateado
  `cfg(not wasm32)`).
- **Sidecar app-side (molde F1.11).** `buildStudyExport` PURA agrega os Records
  retornados (`StudyCitation` + `sources` + metadados) num JSON round-trippável —
  sem nova fronteira, sem reimplementar serialização do core. Consistente com
  ADR-0016/0017/0026/0027; **nenhum ADR novo obrigatório** (a task previa ADR-0028
  opcional; não foi necessário — a forma do export reusa o molde existente).
- **FOLLOW-UP ADR-0027 RESOLVIDO NA RAIZ.** Os 9 doc-comments `**puro**/` do core
  (que fechavam prematuramente o bloco JSDoc dos bindings gerados) viraram
  `**puro** /`. Correção na FRONTEIRA (`core/src/lib.rs`, que podemos editar), NÃO
  no `the-light`. O saneamento `**/` dos geradores fica redundante (mantido,
  defensivo). `tsc` verde nos bindings regenerados = sintoma extinto na origem.
