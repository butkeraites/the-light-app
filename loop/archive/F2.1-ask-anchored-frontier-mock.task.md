## Objetivo

Abrir a **Fase 2 (IA BYOK)** expondo, na fronteira UniFFI (`core/src/lib.rs`), a
**pergunta ancorada** — `ask_anchored(...)` — que **delega** à camada de IA do
`the-light-core` (`ai::{build_provider, ask, ask_context, numbered_passage}`),
provando a espinha ponta a ponta com o **provedor MOCK** do core (sem chave, sem
rede). O texto do versículo vem **sempre do store local** (verbatim, F1.2/F0.9);
o LLM apenas **interpreta**. O tipo de retorno **separa** texto citado (store) de
interpretação (modelo) — anti-alucinação materializada no contrato.

Esta é a menor tarefa elegível que **avança sem bloquear**: usa `MockLlmProvider`
(via `build_provider("mock", None, None)`), então **não** precisa da chave real
nem de rede, e a prova é um **teste Rust de host determinístico**. Gemini,
armazenamento de chave e UI vêm depois (F2.3/F2.4/F2.5), após o **gate estratégico
F2.2**.

## Passos sugeridos

1. **Record de retorno** (puro, todos os alvos):
   `pub struct AiAnswer { reference: Reference, cited_text: String, interpretation:
   String, provider: String, model: String }` (`#[derive(uniffi::Record)]`).
   `cited_text` = passagem **numerada, verbatim do store**; `interpretation` = saída
   do LLM. Reutilize o Record `Reference` já existente na fronteira.
2. **Função** `#[uniffi::export] pub fn ask_anchored(db_path: String, translation:
   String, reference: String, question: String, provider_name: String, key:
   Option<String>, model: Option<String>, lang: String) -> Result<AiAnswer,
   CoreError>`.
   - **Corpo `#[cfg(not(target_arch = "wasm32"))]`:**
     1. `let r = reference::parse_reference(&reference).map_err(...)?;`
     2. abrir o store e obter a `Passage` verbatim pela **mesma** rota da F1.2
        (`Store::open(&db_path)` → `EmbeddedSource`/`BibleSource::passage(&r)` com
        a `translation`).
     3. `let numbered = ai::numbered_passage(&passage);`
        `let context = ai::ask_context(&label, &numbered, &[]);` (related vazio ⇒
        "(nenhuma)"; contexto enriquecido por xref fica p/ depois).
     4. `let lang = parse_lang(&lang);` (`"pt"|"en"` → `Lang`; default sensato).
     5. `let provider = ai::build_provider(&provider_name, key, model)
        .map_err(...)?;`
     6. `let interpretation = ai::ask(provider.as_ref(), &question, &context, lang)
        .map_err(...)?;`
     7. devolver `AiAnswer { reference: r.into(), cited_text: numbered,
        interpretation, provider: provider.name().into(), model:
        provider.model().into() }`.
   - **Stub web** (`#[cfg(target_arch = "wasm32")]`): `Err(CoreError::Generic {
     message: "ai não disponível no alvo web (F2.7)".into() })`. Mantém o grafo wasm

## Restrições (regras rígidas relevantes)

- **Anti-alucinação (LEI):** o texto do versículo vem **sempre** do store local
  (verbatim); o LLM só interpreta, recebendo o texto numerado como **contexto**. A
  fronteira **nunca** deixa o modelo gerar/editar texto bíblico. A referência é
  canônica (`parse_reference`). O `cited_text` e a `interpretation` ficam separados
  no `AiAnswer`.
- **BYOK / chave nunca em git/log:** a chave é **argumento** de `build_provider`;
  esta tarefa usa `key = None` (MOCK). Nada de `secrets.toml`/`KeyStore`; nenhum
  log da chave (não há chave aqui). Armazenamento seguro real = F2.4.
- **Offline-first / rede só opt-in:** o MOCK **não** faz rede; nenhuma capacidade
  essencial passa a exigir rede. A rede real de IA (com a chave do usuário) é F2.6
  (bloqueante).
- **`the-light` intocado:** consumo pinado `8f66004`; **não** reimplementar
  prompt/ancoragem/RAG/SQL na fronteira nem em TS (vem do `ai::study`/store —
  uma fonte da verdade). Se a IA ancarada exigir mudar o core, **`blocked` + HALT**
  com o erro EXATO (decisão estratégica = F2.2/ADR-0023), **não** improviso.
- **Qualidade por tarefa:** `fmt`/`clippy -D warnings`/`test` verdes; grafo wasm
  puro; `gen-bindings-web.sh` verde. Commit com prefixo `feat(F2.1): ...`.
