## Resumo

O transporte web de IA (`app/web/ai-anchored.web.ts::webLlmTransport`) foi estendido para
despachar **anthropic / openai / ollama** (além de gemini/mock), cada um com
endpoint + headers (chave BYOK só no header correto; ollama sem chave) + body + extract,
espelhando os `*_body`/`*_extract` PRIVADOS do core (`04b9b24`, só-leitura). Streaming SSE
para anthropic (`content_block_delta`) e openai (`choices[].delta.content`), NDJSON para
ollama (`message.content` por linha). Helpers compartilhados `postJson`/`readLineStream`/
`parseSseData`; o Gemini foi refatorado para reusá-los sem regressão.

**ZERO-DRIFT confirmado independentemente:** o transporte só monta o request (com
`system`/`user`/`model` do `ai_web_prepare`) e extrai a interpretação do modelo; o
`cited_text` (João 3:16 KJV verbatim do store) sai do `prepare` SEPARADO e a interpretação
completa passa pela MESMA `ai_web_finalize` (Rust). `ai_web_prepare`/`ai_web_finalize`
(core Rust `ai-pure`) e o `the-light` (`04b9b24`) **inalterados**; nenhum binding novo;
grafo wasm puro. Chave session-only por provedor, só no header, nunca na URL/log (ollama
sem chave). Todas as provas headless (mock) + testes web verdes; `tsc`/`expo export` verdes.

**Decisão: passed** — todo o critério de aceite cumprido e a verificação independente saiu
verde.
