## Objetivo

Entregar, no NATIVO (iOS/Android), uma **UI de comparação multi-IA**: a partir de uma
passagem selecionada no Reader, o usuário faz **UMA pergunta ancorada** e escolhe **N
provedores** (≥2, dentre `anthropic`/`openai`/`gemini`/`ollama` + `mock`) para respondê-la
**lado a lado**. Cada coluna faz uma chamada **independente** à fronteira **`ask_anchored`**
(F2.1/F2.3a, já exposta em `reading.ts`) com o provedor daquela coluna, sobre a **mesma
âncora** (mesma `reference`/passagem do store). A UI renderiza N respostas com o
`cited_text` (âncora, VERBATIM do store) exibido **UMA vez**, comum/rotulado e **separado**
de cada `interpretation` (LLM) — **anti-alucinação visível** (molde F2.5/F3.5/F3.6). Prova
**determinística com MOCK** (sem chave, sem rede): self-test no device emite `TLA_COMPARE`.
Só nativo — a **comparação web é a F3.12** (o self-test web é SKIP).

Escopo desta tarefa (atômico): **UI de comparação (`ReaderComparePanel`) + wire no Reader
(`ReaderVersePanel` + `[chapter].tsx`) + self-test de comparação + asserção no
`run-ios-selftest.sh`**. NÃO cria função nova na fronteira Rust (reusa `ask_anchored` da
F2.1/F2.3a). NÃO cria glue novo (o wrapper `askAnchored` já existe em `reading.ts` e
`reading.web.ts`). NÃO usa chave/rede (só `"mock"` na prova). NÃO faz a comparação web (F3.12).

### DECISÃO DE DESENHO (central — registrada e justificada)

O `MockLlmProvider` devolve uma resposta **FIXA** → comparar mock×mock é **degenerado** em
conteúdo. Portanto a prova por MOCK cobre o **WIRING de N provedores**, não a diferença de
respostas:

- **A UI entrega a superfície de comparação + o wiring.** Ela permite escolher N provedores
  e dispara **N chamadas independentes** a `ask_anchored` (uma por coluna) sobre a **mesma
  âncora**, montando N `AiAnswer` (cada um com seu `provider`/`model`/`interpretation`).
- **Prova por MOCK (≥2 colunas "mock"):** o self-test confirma que **N chamadas retornam N
  `AiAnswer`**, que **todos os `cited_text` são IGUAIS** (o mesmo texto numerado do store) —
  **anti-alucinação** + prova de que as N chamadas leram a MESMA âncora do store — e que o
  layout compõe N respostas rotuladas por provedor. O `cited_match=true` é exatamente o
  invariante que a comparação precisa (mesma passagem para todos os modelos).
- **A comparação de respostas DIFERENTES (provedores reais) é a F3.10** (chave real, gate).
  F3.7 **NÃO finge** comparação real com mock: entrega a UI + wiring; o conteúdo real é
  validado na F3.10.

**Layout escolhido (justificado):** o `cited_text` (âncora, verbatim do store) é **IDÊNTICO
em todas as colunas** → é exibido **UMA vez**, como **bloco de âncora comum** no topo,
rotulado "Passagem (texto bíblico)", **separado** das N colunas de interpretação. Isso torna
a **anti-alucinação MAIS visível** (prova que todos os N modelos receberam o MESMO texto do
store, sem N cópias redundantes) e é exatamente o invariante `cited_match` que o self-test
comprova. Cada coluna abaixo mostra **provedor/modelo** + sua `interpretation`. **Custo
estimado é OMITIDO** — a fronteira **não expõe `estimate_cost_usd`** (simplificação idêntica
à da F2.5; nada de custo hardcoded/inventado).

**Chave (BYOK) por coluna:** para provedores **reais** (não-mock), a chave da coluna **viria
do keystore** (F2.4, `getKey(provider)`), passada como argumento a `ask_anchored`; se a chave
faltar, a coluna mostra um estado "sem chave (BYOK — F3.10)" e **não** chama a fronteira.
Nesta entrega, a **prova usa só `"mock"`** (sem chave, sem rede). A chave **nunca** é logada.
