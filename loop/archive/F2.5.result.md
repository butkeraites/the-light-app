## Resumo

A UI nativa de estudo assistido ancorado (`ask`) foi entregue e provada ponta a
ponta no device com o provedor determinístico `"mock"` (sem chave, sem rede). O
`ReaderAskPanel` (novo) consome EXCLUSIVAMENTE a fronteira de IA já entregue
(F2.1/F2.3a) via glue nativo → bindings gerados → JSI → `the_light_core::ai`:
`askAnchored`/`askAnchoredStream` com wiring do `AiTokenCallback` (`{ onToken }`)
→ callback JS incremental. NÃO reimplementa prompt/RAG/citação/streaming em TS
(uma fonte da verdade). Anti-alucinação VISÍVEL confirmada: o `citedText`
(passagem verbatim do store, rótulo "Passagem (texto bíblico)") é exibido
SEPARADO da `interpretation` (LLM, rótulo "Interpretação (IA) — confira nas
Escrituras"); nenhuma resposta de IA é hardcoded no app. A chave BYOK é lida sob
demanda via `keystore.getKey` só para provedores reais e NUNCA é logada (grep
limpo); a prova usa `"mock"` sem chave. Glue web = STUB explícito (IA web = F2.7).
`tsc --noEmit` e `expo export --platform web` verdes; `the-light` intocado
(`133077a`), `core/src/lib.rs`/`core/Cargo.toml` não modificados. Prova headless
no simulador iOS: `TLA_ASK` presente com `provider="mock"`, `streamed=true`,
`cited_prefix` de João 3:16 KJV e `interp_len=94`, SEM regressão dos demais TLA_*.

## Decisões / ADRs

- **Sem ADR novo.** A UI de ask consome a fronteira já entregue
  `ask_anchored`/`ask_anchored_stream` + `AiTokenCallback` (F2.1/F2.3a) via
  UniFFI/JSI; nenhuma função nova de fronteira foi criada. O pipeline
  (referência → passagem verbatim do store → contexto ancorado → provedor →
  interpretação) vive no core (uma fonte da verdade).
- **Anti-alucinação VISÍVEL** materializada na UI: `citedText` (store, verbatim) e
  `interpretation` (LLM) em blocos com rótulos distintos; o texto bíblico nunca é
  apresentado como saída do LLM. Confirmado no device (cited_prefix == KJV do store).
- **Custo simplificado**: a fronteira NÃO expõe `estimate_cost_usd`; o painel exibe
  uma estimativa aproximada de tokens da interpretação (`~text.length/4`) com o
  aviso "custo exato indisponível". Nenhuma função de fronteira criada só p/ custo.
- **BYOK**: chave lida sob demanda via `keystore.getKey(provider)` apenas para
  provedores reais; `"mock"` = sem chave/sem rede (caminho da prova). Chave nunca
  logada/impressa/versionada.
- **Paridade web = stub** (`reading.web.ts` lança "IA web = F2.7"); IA não ligada ao
  wasm nesta tarefa.

## Bloqueios / perguntas

- **A PRÓXIMA tarefa (F2.6 — validação REAL com a chave do usuário: Claude/GPT/
  Gemini) é BLOQUEANTE / gate.** Exige credencial BYOK real + rede opt-in — fora do
  escopo autônomo offline-first do loop. Ao chegar nela, o Driver deve escrever
  `loop/HALT` aguardando decisão/credencial humana (nunca chamar provedores reais
  no loop/CI).
- **Ask no web = F2.7** (após o PR D2/wasm que habilita a camada de IA no web); hoje
  é stub explícito, sem regressão.
- **Streaming real token-a-token por provedor** (SSE incremental de Claude/GPT/
  Gemini) é evolução: o mock emite a interpretação 1× pelo callback; a assinatura de
  streaming já está pronta para o incremento real na F2.6.
- Nada mais pendente para esta tarefa (F2.5 não-bloqueante): **passed**.
