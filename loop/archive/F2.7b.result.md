## Resumo

Paridade web de IA entregue e verificada de forma independente. O caminho web do
`ask` ancorado está **destubado** e funciona **ponta a ponta** com **zero drift**
anti-alucinação: prompt/RAG/citação vêm do **mesmo Rust `ai-pure`** (wasm) no web e
no nativo, e o transporte é `fetch` no TS. A fronteira nova (`core/src/lib.rs`:
`ai_web_prepare`/`ai_web_finalize` + Records `AiWebRequest`/`AiVerseInput`) é
**cfg-free** (só a superfície `pub` do `ai-pure`) → o grafo wasm segue **puro** (sem
`rusqlite`/`reqwest`). O `cited_text` sai **sempre do store** (`numbered_verses`) e a
`interpretation` do provedor; a citação anti-alucinação (`rewrite_anchors`) roda em
Rust no `finalize`. Chave web = **session-only / in-memory** (`Map` de módulo, perdida
no reload), **nunca** `localStorage`/`sessionStorage`/`IndexedDB`/log; no transporte a
chave vai **só** no header `x-goog-api-key`. `the-light` intacto em `c8ecb2f`,
`core/Cargo.toml` não alterado, sem regressão nativa (43 testes) nem web.

**Linha-chave (prova mock, sem rede/chave real):** `cited_text = "16 For God so loved
the world, …everlasting life."` (João 3:16 KJV **verbatim do store**) e
`interpretation = "Resposta simulada do provedor (MOCK): …"` (texto do **mock**, com a
âncora espúria `[V:G9999]` removida por `rewrite_anchors`). Zero-drift provado no host
por `ai_web_prepare_matches_native_anchored_cited_text_zero_drift` (cited_text/system
idênticos ao `ask_anchored` nativo).

## Decisões / ADRs

- **ADR-0025** (aceito): política de chave web **session-only / in-memory** (`Map` de
  módulo, perdido no reload; interface idêntica ao nativo) + padrão **prepare → fetch →
  finalize** (prompt/RAG/citação em Rust `ai-pure` = anti-alucinação com **zero drift**;
  `body`/`extract` em TS porque os `gemini_body`/`gemini_extract` do core são **privados**
  sob `ai-pure` — transporte = infra, ADR-0023/D2). Depende de ADR-0024 (`ai-pure`),
  ADR-0023 (D2 transporte / D3 chave), ADR-0011 (infra TS no web).
- O `system`/`user` EXATOS são capturados pela rota **pública** `ai::ask` dirigida por um
  `CaptureProvider` local (o `ask_user_prompt` do core é privado) — mesma moldura de
  prompt do nativo, confirmada idêntica pelo teste host de paridade.
