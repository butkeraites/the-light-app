## Decisões / ADRs

- **D2 do ADR-0023 concretizada como feature `ai-pure`:** as partes PURAS do `ai`
  (prompt/RAG + `ask` ancorado + `citation::rewrite_anchors` anti-alucinação + `default_model`/
  `estimate_cost_usd`) agora compilam em `wasm32` sem `reqwest`/`rusqlite`. A montagem de
  prompt+citação é UMA única impl Rust compartilhada nativo/web (sem drift) — anti-alucinação
  preservada (o texto do versículo vem do store; o LLM só interpreta). Feature aditiva e
  opt-in (`default` byte-a-byte; `embedded ⊃ ai-pure`).
- **Fix `default_model` gemini → `gemini-2.5-flash`** (o `2.0-flash` foi retirado 3/mar/2026);
  preço NÃO inventado (2.5-flash cai em `starts_with("gemini") => None`, honesto). Testes que
  fixavam o antigo ajustados (verdes).
- **`#![cfg_attr(not(embedded), allow(dead_code))]` em `providers.rs`:** as helpers puras
  (`*_body`/`*_extract`) ficam sem chamador no build ai-pure até a **F2.7b** (que liga o
  caminho de IA web por `fetch`). É um `allow` gateado SÓ em `not(embedded)` — sob o default
  (embedded) o `dead_code` continua ativo; não relaxa a qualidade do build nativo.
- **ADR novo pendente:** **o Driver deve registrar `ADR-0024`** no `DECISIONS.md` (feature
  `ai-pure` + fix default gemini) e escrever a spec do PR em
  `loop/proposals/the-light-PR-ai-pure-wasm.md` — ainda não existem (máximo atual = ADR-0023).
