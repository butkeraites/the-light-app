## Resumo

A fronteira `lexical_entries(db_path, book, chapter, verse: Option<u16>, lang, limit:
Option<u32>) -> Result<VerifiedLexiconOut, CoreError>` foi adicionada em
`core/src/lib.rs` (commit `7f469d4`, +370, um só arquivo). Ela **delega** ao core
embedded `the_light_core::ai::lexicon::verified_lexicon(store.conn(), &Reference, &[],
lang, limit)` via `Store::open` — **sem reimplementar** SELECT/JOIN/COALESCE/agregação de
léxico na fronteira (o SQL de léxico aparece apenas em doc-comments descrevendo o core).
Records **puros** `LexEntry{strongs,lemma,translit,gloss,occurrences,testament}` e
`VerifiedLexiconOut{entries,sources}` com dois `From` **cfg-free** (tipos-fonte
`ai::LexicalEntry`/`ai::VerifiedLexicon` são `ai-pure`, ADR-0024); só o **corpo** que abre
o store é `cfg(not(target_arch = "wasm32"))` + **stub web** (`léxico indisponível no alvo
web (F3.12)`). Verificação independente **verde**: `fmt --check` (0), `clippy -D warnings`
(0, limpo), `cargo test` **48 passed / 0 failed** (43 anteriores + 5 novos
`lexicon_tests`), grafo wasm **puro** (sem `rusqlite`/`reqwest`), `gen-bindings-web.sh`
exit 0 gerando `lexicalEntries`/`LexEntry`/`VerifiedLexiconOut`, `the-light` intacto
(`c8ecb2f`), `core/Cargo.toml` não modificado (pin count 2, sem diff).

Linha-chave verificada: **Gn 1:1 → Strong base `H0430` (de `H0430G`), `gloss == Some("God")`,
`lemma` presente, `testament == "OT"`, `sources` com `STEP Bible`** — do léxico local
verificado (STEP CC-BY), nunca do LLM. O teste bônus com o `bible.sqlite` real (F3.1)
**rodou e passou** (`H0430 → gloss contém "God"` no corpus real).

## Decisões / ADRs

- **Sem ADR novo.** A fronteira apenas **consome** `ai::lexicon::verified_lexicon`, já
  `pub` sob `#[cfg(feature="embedded")]` no rev pinado `c8ecb2f` — nenhuma função nova
  exigida do core, nenhum PR ao `the-light`.
- **Delegação total ao core (uma fonte da verdade):** `lexical_entries` só abre o store,
  monta o `Reference` e chama `verified_lexicon`; a query, o `COALESCE` da glosa, a
  agregação por Strong base e a ordenação por frequência vivem em `ai/lexicon.rs`.
- **`verified_lexicon` é infalível** (não devolve `Result` — engole erros de SQLite e
  devolve vazio); o **único** ponto de erro da fronteira é `Store::open` (I/O local).
  Passa-se `verse_numbers = &[]` e deixa o core derivar do `Reference`.
- **Sem parâmetro `translation`:** o léxico é independente de tradução (chaveado por
  `book_number/chapter[/verse]`), como a xref da F1.8; recebe `book/chapter/verse`
  numéricos. O `lang` é aceito por fidelidade à assinatura real mas atualmente ignorado
  por `verified_lexicon` (zero drift futuro).
- **`From` cfg-free (divergência vs. `CrossRef`/`Note`):** os tipos-fonte
  `ai::LexicalEntry`/`ai::VerifiedLexicon` são `ai-pure` (presentes no wasm, ADR-0024),
  logo os dois `From` **não** são gateados (como `From<model::SearchHit>`); só o corpo que
  abre o store SQLite é `cfg(not(wasm32))`. Divergência documentada no doc-comment.
- **Anti-alucinação + licenciamento:** glosas/lemas/Strong vêm do léxico local verificado
  (STEP Bible / TBESH–TBESG, CC-BY, F3.1); as `sources` (atribuição verbatim) são
  preservadas para a **exibição obrigatória** da UI da F3.5.
