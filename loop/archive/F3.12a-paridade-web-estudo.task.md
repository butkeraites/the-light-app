## Objetivo

Levar o **estudo profundo** ao **web (wasm)** com **fonte única em Rust / ZERO
DRIFT** (ADR-0029 D2), consumindo a superfície pura do estudo que a **F3.11
mergeou** (`the-light` @ **`04b9b24`**, feature `ai-pure` estendida) via o padrão
**prepare → (fetch em TS) → finalize** (molde EXATO F2.7b/ADR-0025). Nesta 1ª
fatia entregar: **estudo profundo web** (`deepStudy`), **léxico verificado web**
(`lexicalEntries`) e **export acadêmico web** (reusa `buildStudyExport`), com o
texto do versículo e as glosas do **store web** (subset `reading-sample.sqlite`,
que tem léxico desde a F3.5/ADR-0027) e a interpretação vinda do `fetch` ao LLM
(chave **session-only**). O painel compartilhado `ReaderStudyPanel` passa a
funcionar no web só pelo destub do glue.

**Fora de escopo (fatia F3.12b, backlog):** conversa ancorada web
(`askSessionAnchored`), **pesquisa web Wikipedia** (fetch opt-in → `WebSource`) e
comparação multi-IA web. A fronteira desta fatia **já aceita** `web_sources`
(vazio aqui) para que a F3.12b seja **app-side apenas** (fetch Wikipedia → passar
`web_sources`), sem re-tocar o Rust.

**Regra dura (ADR-0029):** proibido espelhar em TS o anti-alucinação do estudo
(prompt/citação/aparato/verify). O prompt de estudo (`system_prompt_in` +
`user_prompt`), a validação (`verify`/`cited_web_indices`), as citações
(`CitationCollector`/`from_verified_lexicon`/`from_web_results`) e o markdown
acadêmico (`to_academic_markdown` com `rewrite_anchors`) vêm do **MESMO Rust
`ai-pure`** no web e no nativo. Só a **recuperação do store** (SELECT do léxico) e
o **transporte** (`fetch`) são infra TS (ADR-0011 / ADR-0025).

### Desenho das funções de fronteira web (NÃO tocam the-light — só `core/src/lib.rs`)

Corpo **cfg-free** (só `ai-pure`), espelhando `ai_web_prepare`/`ai_web_finalize`:

- Records novos (puros, uniffi):
  - `StudyLexEntryInput { strongs, lemma: Option<String>, translit: Option<String>,
    gloss: Option<String>, occurrences: u32, testament: String }` (espelha
    `LexicalEntry`) — vem da recuperação TS do store web.
  - `StudyWebSourceInput { title, url, snippet, site, fetched_at: i64 }` (espelha
    `WebSource`; `fetched_at` = timestamp p/ `from_timestamp`) — **vazio na F3.12a**;
    populado por Wikipedia fetch na F3.12b.
  - `StudyWebRequest { reference, reference_label, passage_text, system, user,
    provider, model }` (saída do prepare; passage_text = numerado verbatim do store).
- `study_web_prepare(reference, mode, lens, depth, lang, provider_name,
  model: Option<String>, verses: Vec<AiVerseInput>, lexicon_entries:
  Vec<StudyLexEntryInput>, lexicon_sources: Vec<String>, web_sources:
  Vec<StudyWebSourceInput>) -> Result<StudyWebRequest, CoreError>`:
  1. `reference = parse_reference(&reference)`; `lang.parse::<Lang>()` default Pt;
  2. `passage_text = ai::numbered_verses(verses)` (verbatim do store);
  3. montar `Passage` (model) dos `verses` p/ `passage: Some(&p)`;
  4. montar `VerifiedLexicon { entries, sources }` dos `lexicon_entries`/`_sources`;
  5. montar `Vec<WebSource>` dos `web_sources` (vazio na F3.12a);
  6. `cross_references` = `[]` nesta fatia (ou passados como `Vec<String>` opcional);
  7. `system = prompts::system_prompt_in(mode, lens, depth, lang, None)`;
  8. `user = study::user_prompt(&req, &passage_text)`;
  9. `reference_label = format_reference(&reference, lang)`;
  10. `model = model|default_model(provider)`.
- `study_web_finalize(reference, reference_label, mode, lens, depth, lang,
  passage_text, provider, model, raw_llm_response, lexicon_entries:
  Vec<StudyLexEntryInput>, lexicon_sources: Vec<String>, web_sources:
  Vec<StudyWebSourceInput>) -> Result<StudyResultOut, CoreError>`:
  1. reconstruir `VerifiedLexicon` + `Vec<WebSource>` (mesmos inputs do prepare);
  2. `sections = split_sections(&raw)`;
  3. `warnings = verify(&raw, &vl).warnings` (se `mode.wants_lexical()`) +
     validação `cited_web_indices` fora do intervalo (mesma regra do `study()`);
  4. `citations = CitationCollector { from_verified_lexicon(&vl) +
     from_web_results(&ws) }.into_vec()` se `mode.emits_apparatus()`, senão `[]`;
  5. montar `StudyResult { …, passage_text, interpretation=raw, sections, warnings,
     citations, provider, model }` e `academic_markdown = to_academic_markdown(lang)`;
  6. mapear p/ `StudyResultOut` (o Record já existente, incl. `academic_markdown`).

Os corpos usam **só** a superfície `ai-pure` (nenhum `Store`/rusqlite/reqwest) → o
grafo wasm da fronteira segue **puro** (como `ai_web_prepare`). Regenerar bindings
web (`gen-bindings-web.sh`).

### Recuperação do léxico no web (infra TS — ADR-0011 precedente)

Novo `app/web/sqlite-lexicon.web.ts` (par de `sqlite-xref.web.ts`): abre o subset
(wa-sqlite/OPFS), roda o `SELECT` de `original_tokens` (por book/chapter[/verse])
+ JOIN `lexicon` (glosa: `gloss_pt`→`gloss`→token do original) + `scholarly_sources`
(atribuição), agrega por **Strong base** (remove letras de desambiguação à direita,

por Strong) e trunca ao `limit` — **espelhando o shape** de
`ai::lexicon::verified_lexicon` (retrieval, ADR-0011: SELECT+shaping = infra TS, o
mesmo precedente do passage/xref/search web). Produz `StudyLexEntryInput[]` +
`sources[]`. `lexicalEntries` web = essa recuperação → `VerifiedLexiconOut` (o
Record de retorno já existe; sem lógica de anti-alucinação em TS — as glosas são
VERBATIM do store).

## Restrições (regras rígidas relevantes)

- **NÃO tocar `the-light`** — consome `ai-pure` pinado `04b9b24` (a superfície pura
  do estudo veio da F3.11 mergeada). Única mudança em `core/` = a fronteira web
  nova em `core/src/lib.rs`; `core/Cargo.toml` **inalterado** (re-pin já feito).
- **Anti-alucinação ZERO-DRIFT (ADR-0029):** prompt/citação/aparato/verify do
  **MESMO Rust `ai-pure`**; **PROIBIDO** espelhar em TS. Só recuperação de store
  (SELECT do léxico, ADR-0011) e transporte (`fetch`, ADR-0025) são infra TS.
  `passage_text`/glosas SEMPRE do store; `interpretation` só do LLM.
- **BYOK / chave web session-only** (ADR-0025) — nunca em git/log; só no header do
  `fetch`. Sem chave/sessão, o app segue 100% offline (estudo é opt-in).
- **Offline-first** — a única rede em runtime é o `fetch` ao LLM (opt-in). Pesquisa
  web Wikipedia (rede opt-in) = **F3.12b**.
- **Subset A1** — usar `reading-sample.sqlite` (com léxico STEP CC-BY, F3.5).
- **Se a fronteira web exigir símbolo do core ausente em `ai-pure`** (ex.: um
  aggregador de léxico só-`embedded` sem equivalente puro e cuja replicação em
  TS/app-Rust seria drift do anti-alucinação) → **`blocked` + HALT** com a decisão
  (outro PR ao core). NÃO improvisar. A recuperação de léxico (SELECT + shape) É
  infra TS sancionada (ADR-0011); o que NÃO pode virar TS é prompt/verify/citação.
- **Qualidade por tarefa** verde: Rust `fmt`/`clippy -D warnings`/`test`; TS
  `tsc --noEmit`/`eslint`; `expo export web` 0. Diante de conflito: **HALT**.
