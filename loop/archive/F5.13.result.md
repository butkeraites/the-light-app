## Resumo

F5.13 adiciona lembretes LOCAIS diários OPT-IN (OFF por padrão) do plano de leitura
ativo, estritamente offline. A lógica pura vive em `app/lib/planReminders.shared.ts`
(`createPlanReminders` sobre um `NotificationsBackend` INJETÁVEL + KV de prefs F5.2),
sem `expo-notifications`/`react-native`. O backend NATIVO (`planReminders.ts`) é um
wrapper 1:1 sobre `expo-notifications` LOCAL (import lazy); o WEB (`planReminders.web.ts`)
é um no-op documentado que mantém o módulo nativo FORA do bundle web. A pref
(`enabled`+`time`+`id`) persiste no KV OFFLINE da F5.2 sob `tla.pref.plans.reminder`,
SEPARADA do `active.json` do core (`PlanProgress` intacto). A UI (`plans/index.tsx`)
ganha `ReminderControls` (Switch opt-in + chips de horário) sob o plano ativo, só no
ramo nativo; `clearReadingPlan` cancela o lembrete. Corpo/título = cromo i18n + NOME do
plano do core (anti-alucinação). Re-verifiquei todos os gates de forma independente:
verde. A prova crítica offline-first (LOCAL-only, 0 refs no bundle web) confere.

## Decisões / ADRs

- **ADR-0042** (F5.13) — lembretes LOCAIS opt-in do plano ativo: `expo-notifications` LOCAL
  (agenda no device, sem servidor/conta/push token), pref app-side na F5.2, web degrada.
  Número livre (1 ocorrência em DECISIONS.md), aceito. Adiciona a dep de runtime
  `expo-notifications` ~56.0.19 (app-side; NÃO toca `the-light`).
- **Julgamento mock-vs-device:** aceito. A lógica compartilhada (`createPlanReminders`) é
  bem coberta (opt-in OFF, ligar→1 diária, trocar→cancela+reagenda, desligar→cancela+remove,
  negada→nada+OFF, já-concedida→sem re-prompt, persistência ao reabrir, `parseHHMM`/
  `formatHHMM`, grep estrutural offline, sem `console.*`). O backend NATIVO é um wrapper
  FINO 1:1 sobre a API LOCAL de `expo-notifications`, e o `tsc` (exit 0) valida seus nomes/
  assinaturas contra os type defs REAIS do módulo instalado (56.0.19) — não há complexidade
  escondida sem cobertura. O mock representa fielmente a API LOCAL (permissão + agenda/
  cancela), sem tocar push token/rede.
- `eslint` = N/A neste repo (sem config/dep; precedente ADR-0038). Gate real de TS = `tsc`.
