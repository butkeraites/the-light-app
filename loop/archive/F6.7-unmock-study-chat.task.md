## Objetivo

Dois dos 4 painéis de IA estão travados no mock: `ReaderStudyPanel.tsx:217` e `ReaderChatPanel.tsx:134`
hardcodam `MOCK_PROVIDER` e não têm seletor de provedor. O transporte real JÁ EXISTE nos dois alvos
(nativo `core ai::providers`; web `ai-anchored.web.ts`) e as fns de fronteira (`deepStudy`,
`askSessionAnchored`) já ACEITAM `providerName`+`key` como args — o painel Ask (`ReaderAskPanel`) é o
modelo pronto. Esta é a maior lacuna "existe mas está retido" e a menor mudança real (troca de 2 args
literais + cromo de seletor).

## Escopo

1. **`ReaderStudyPanel.tsx`:** adicionar um SELETOR de provedor (espelhar `ReaderAskPanel.tsx:239-311`:
   chips sobre `[MOCK_PROVIDER, ...SUPPORTED_PROVIDERS]` + badges do `useConfiguredAiProviders`, já
   importado ~:167). Trocar os args hardcoded em `:217` (`MOCK_PROVIDER, undefined, undefined`) pelo
   `provider` selecionado + `key` lido sob demanda via `getKey(provider)`; se provedor real sem chave →
   mostrar `ask.needKeyError` (ou equivalente) + o CTA p/ `/settings` (F6.6), pulando a chamada (igual
   Ask `:154-160`). `ai.mockProviderNote` só quando `provider === 'mock'`. `deepStudy` segue não-streaming.
   NÃO confundir com a chave de web-research Tavily (session-only `useState`) — é OUTRA coisa, mantém.
2. **`ReaderChatPanel.tsx`:** mesmo padrão — seletor + trocar `MOCK_PROVIDER, undefined, undefined` no
   `askSessionAnchored(...)` `:134` por `provider`+`getKey`. Respeitar a ordem real dos args
   (`:122-137`: `lang` antes de `turns`; `studyMode`/`studyLens` antes de `providerName`).
3. **Anti-alucinação (inegociável):** `passageText`/`lexicalEntries`/`citations`/`citedText` seguem do
   STORE (verbatim); só a `interpretation` é do modelo. Blocos citação/interpretação separados como hoje.
4. **Mock = default/offline:** manter `mock` como opção default (responde offline, determinístico) — o
   usuário escolhe um provedor real quando tiver chave. Reusar `AiProviderNotice` quando sem provedor.
5. **i18n/a11y:** qualquer cromo novo via `t()` pt/en; seletor com role/label; sem hex.
