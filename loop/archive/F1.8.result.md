## Resumo

A fronteira UniFFI (`core/src/lib.rs`) passa a expor `cross_refs(db_path, book,
chapter, verse, min_votes: Option<i64>, limit: Option<u32>) -> Result<Vec<CrossRef>,
CoreError>`, que **delega** à função livre `the_light_core::xref::for_verse(store.conn(),
…)` (via `Store::open`) e adapta cada `xref::CrossRef` para o novo Record UniFFI
`CrossRef { reference: Reference, votes: i64 }`. **Nenhum** SQL/ranking de xref é
reimplementado na fronteira de produção: a query, a ordenação por votos (DESC) e o
filtro `votes >= min_votes` vivem no core (`xref.rs`). Os defaults vêm do core
(`min_votes None` → `DEFAULT_MIN_VOTES` = 1, oculta disputadas; `limit None` →
`DEFAULT_LIMIT` = 20). O Record existe em **todos** os alvos (só tipos puros
`Reference`/`i64`), mas o `impl From<the_light_core::xref::CrossRef>` é gated
`#[cfg(not(target_arch = "wasm32"))]` (o módulo `xref` do core é `embedded`-only); o
corpo DB de `cross_refs` é gated com stub web retornando `CoreError`. Verificação
independente reproduzida do zero: `fmt`/`clippy -D warnings`/`test`(26) verdes, grafo
wasm sem `rusqlite`, bindings web (exit 0) com `crossRefs`, `the-light` intacto em
`8f66004`, árvore limpa (só bookkeeping do loop). Veredito: **passed**.

## Decisões / ADRs

- **Sem ADR novo.** Reusa ADR-0005 (matriz de features por alvo; `embedded` só no
  nativo), ADR-0010 (store nativo + **gating de corpo** + anti-alucinação) e ADR-0016
  (xref OpenBible **CC-BY** + string de atribuição). Mesmo molde de F0.9/F1.2/F1.5.
- **Divergência de gating (vs. F1.5) confirmada e correta:** o tipo-fonte
  `the_light_core::xref::CrossRef` é `embedded`-only (módulo `xref`), então o
  `From<xref::CrossRef>` está sob `#[cfg(not(target_arch = "wasm32"))]` (lib.rs 302),
  enquanto o Record `CrossRef` (puro, `Reference`/`i64`) vale para todos os alvos — o
  módulo `xref` **não** entra no grafo wasm (verificado: `cargo tree` wasm sem
  `rusqlite`; build web exit 0).
- **Atribuição CC-BY:** a fronteira **só entrega os dados** (referência + votos); a
  string visível `Cross references courtesy of OpenBible.info (CC-BY)` é exibida pela
  **UI da F1.9** (registrado no doc-comment de `cross_refs`/`CrossRef`). A obrigação
  CC-BY do produto não desaparece — muda de camada.
- **Anti-alucinação:** xref é só referência (book/chapter/verse + votos); nenhum texto
  bíblico é gerado. INSERTs em `cross_references` são fixture de teste (como
  `xref.rs::seeded` no core), não produto.
