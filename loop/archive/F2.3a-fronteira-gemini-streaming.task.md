## Objetivo
1. **Rotear `"gemini"`**: a `ask_anchored` (F2.1) já passa `provider_name` a
   `build_provider` — que agora conhece `"gemini"`. Adicionar teste de host provando o
   roteamento SEM rede: `ask_anchored(..., provider_name="gemini", key=None, ...)` →
   `CoreError` (o core devolve `NoKey` para gemini/anthropic/openai sem chave), provando
   que o roteamento alcança o arm gemini. (NÃO chamar gemini real — sem key/rede.)
2. **Expor streaming (D4)**: nova função de fronteira
   `ask_anchored_stream(db_path, translation, reference, question, provider_name,
   key: Option<String>, model: Option<String>, lang, on_token: Box<dyn AiTokenCallback>)
   -> Result<AiAnswer, CoreError>` que:
   - monta o contexto ancorado igual à `ask_anchored` (passagem do store via
     `EmbeddedSource::passage` + `numbered_passage`/`ask_context`),
   - chama `provider.complete_stream(system, user, &mut |tok| callback.on_token(tok))`
     (o default do core emite a resposta inteira 1× → o callback recebe o texto todo),
   - retorna o mesmo `AiAnswer` (cited_text do STORE + interpretation do LLM/mock).
   - Callback interface UniFFI: `#[uniffi::export(callback_interface)] pub trait
     AiTokenCallback { fn on_token(&self, token: String); }` (ou o equivalente idiomático).
   - Corpo `#[cfg(not(target_arch="wasm32"))]` + **stub web** (streaming no web = F2.7;
     stub consome args, retorna CoreError). Grafo wasm permanece puro.

## Anti-alucinação (LEI)
No streaming, o **cited_text vem do store** (verbatim), a `interpretation` é o que o
provider (mock) emitiu; o callback transmite tokens da **interpretação** (não do texto
bíblico). O teste prova que `cited_text` é do store e que os tokens recebidos compõem a
resposta do MOCK (não texto bíblico gerado).
