## Objetivo

Ligar a **pesquisa web opt-in** (ADR-0028 / gate F3.9 D1) ao estudo profundo na
fronteira `deep_study` (`core/src/lib.rs`), **sem tocar o `the-light`**. Hoje a
fronteira passa `web_sources: vec![]` ao `study()`; esta tarefa adiciona um parâmetro
**`research_backend: Option<String>`** que, quando `Some(...)`, roda
`the_light_core::ai::build_research_provider(backend, None, lang_code)` +
`provider.search(query, limit)` e injeta o `Vec<WebSource>` resultante em
`StudyRequest.web_sources`. O `study()` do core então injeta o bloco "FONTES
SECUNDÁRIAS" citável por `[W:n]` e monta as citações **das URLs**
(`CitationCollector::from_web_results`, **nunca do modelo**; anti-alucinação embutida).

- `research_backend = None` (ou omitido) → `web_sources = vec![]` (comportamento atual,
  offline). **Padrão desligado.**
- `research_backend = Some("wikipedia")` → `WikipediaProvider` (**keyless, rede real**) —
  opt-in do usuário, validado na **F3.10** (chave/rede reais; fora desta tarefa).
- `research_backend = Some("mock")` → `MockResearchProvider::canned()` (**sem rede**) —
  a **prova determinística** desta tarefa.

**NÃO-BLOQUEANTE** (só mock, sem chave/rede) e **NÃO exige PR** (toda a superfície de
`research` já é `pub` sob `embedded` em `c8ecb2f`).

## Restrições

- **NÃO tocar o `the-light`.** `build_research_provider`/`ResearchProvider`/`WebSource`/
  `MockResearchProvider`/`study`/`StudyRequest` já são `pub` sob `embedded` em `c8ecb2f`.
  Se algo exigisse função **não-`pub`** ou mudança no core → **`blocked`/HALT** (não
  stubar/forkar/copiar). O pin `c8ecb2f` e `uniffi=0.31.0` **não** mudam.
- **Offline-first / opt-in:** o padrão é **desligado** (`None` → sem rede). A pesquisa
  web (`"wikipedia"`) é **rede opt-in do usuário** — nesta tarefa **só o mock** é
  exercitado (sem rede/chave). Nenhuma chamada de rede no CI/teste.
- **Anti-alucinação:** `passage_text` e `citations` (léxico + `[W:n]`) vêm **do
  store/das URLs buscadas**, nunca do modelo; `[W:n]` fora do intervalo é sinalizado
  pelo core. **Não** reimplementar montagem de prompt/citação/busca (vive no core).
- **BYOK / segredos:** nenhuma chave nesta tarefa (`key=None`; Wikipedia é keyless).
  Nenhuma chave em git/log.
- **Grafo wasm puro:** a busca web (`research`, embedded-only/`reqwest`) fica **dentro**
  do bloco `cfg(not(target_arch="wasm32"))`; o stub web só ignora o novo param. Paridade
  web (busca por `fetch`) é a **F3.12** (após o PR `ai-pure` da F3.11).
- **Escopo atômico:** só a fiação `research_backend → web_sources` + teste MOCK + higiene
  TS. **Sem UI de pesquisa web** (F3.5 futura), **sem** validação real (F3.10), **sem** PR.
