## Resumo

A UI de referências cruzadas NATIVA está entregue e provada no device. Ao
selecionar um versículo no Reader (`read/[book]/[chapter]`), o `ReaderXrefPanel`
(Modal/bottom-sheet, tokens de tema) lista as xrefs vindas EXCLUSIVAMENTE da
fronteira F1.8 `cross_refs` (glue `reading.ts` → binding gerado `crossRefs` → JSI
→ `the_light_core::xref::for_verse`, já ordenado por votos DESC pelo core); cada
item mostra a referência de destino + votos (`String(cr.votes)`, robusto a
`bigint`) e é `Pressable` → `router.push` ao capítulo alvo (rota F1.3). A
atribuição CC-BY EXATA (ADR-0016) é exibida sempre que há xrefs. Nenhum
SQL/`ORDER BY votes`/filtro de xref é reimplementado no TS de produção; a UI só
apresenta o `Vec<CrossRef>`. O subset bundled `reading-sample.sqlite` tem
`cross_references` populada pelo gerador (copiada do `bible.sqlite`, ambos os
lados em {1,19,43}): 22413 xrefs, 9 em João 3:16 — LIDO do banco, não hardcoded.
A prova headless no iPhone 17 emitiu o marcador central
`TLA_XREF verse="John 3:16" count=9 first_ref="John 3:15" first_votes=439` (do
retorno real de `cross_refs`) sem regressão de TLA_SELFTEST PT/EN, TLA_READ,
TLA_PARALLEL, TLA_SEARCH. `tsc`/`expo export` verdes; `the-light` (8f66004) e a
fronteira `core/src/lib.rs` intactos. **Decisão: passed.**

## Decisões / ADRs

- **Sem ADR novo.** Reusa **ADR-0014** (bundling do subset no app nativo),
  **ADR-0015** (tema/tokens) e **ADR-0016** (xref OpenBible CC-BY + string de
  atribuição obrigatória — exibida nesta F1.9).
- `votes` é `i64` no core → `bigint` no binding ubrn (`FfiConverterInt64`). UI, glue
  e self-test formatam via `String(...)`, robusto a `number`/`bigint` — não assumem
  `number`. O parâmetro `minVotes` também é tipado `bigint` no glue/binding.
- Pipeline de DADOS (não xref): o gerador `gen_reading_sample_db.rs` copia as xrefs
  do `bible.sqlite` cujos AMBOS os lados caem em {1,19,43}, garantindo que o capítulo
  de destino exista no subset (o clique não cai em "Capítulo não encontrado"). A
  query/ordenação/filtro continuam no core (`xref::for_verse`), chamados pela
  fronteira `cross_refs`. Asserções de sanidade no próprio gerador.

## Bloqueios / perguntas

- **Nenhum bloqueio.** A paridade web da xref é a **F1.15** (pós-gate estratégico
  F1.12); aqui o glue web (`reading.web.ts`/`xref-selftest.web.ts`) é stub que lança
  em runtime, mantendo `tsc`/Metro web verdes (sem regressão).
- Nota operacional: antes da prova no device foi necessário matar Metro órfão em
  `:8081` (`pkill -f "expo start"; lsof -ti:8081 | xargs kill -9`) para evitar flake
  de porta. O run-ios-selftest.sh rodou limpo numa única execução (EXIT=0), sem
  necessidade de `SKIP_BUILD=1`. A mensagem `Terminated: 15` no fim do log é o
  `cleanup()` encerrando o Metro de background após o sucesso (esperado).
