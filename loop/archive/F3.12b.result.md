## Resumo

Paridade WEB do estudo profundo fechada nas 3 frentes, com **fonte única em Rust
`ai-pure` / ZERO DRIFT** (ADR-0032). Verificação independente reproduzida integralmente
e verde:

1. **Conversa ancorada web** — 1 fronteira web nova `session_web_prepare` em
   `core/src/lib.rs` (corpo cfg-free, só a superfície `pub` do `ai-pure`), que dirige
   `ai::ask_session` por um `CaptureProvider` (sobrescreve só `complete`) → captura o par
   `(system, user)` **EXATO** que o nativo `ask_session_anchored` enviaria. Teste de host
   de **paridade** prova `system`/`user`/`cited_text` idênticos ao caminho nativo (zero
   drift). Reusa `AiWebRequest` + `ai_web_finalize` (F2.7b) — nenhum finalize novo.
   Glue `session.web.ts` destuba `askSessionAnchored` → `ReaderChatPanel` funciona no web.
2. **Pesquisa Wikipedia (opt-in, keyless)** — `research.web.ts::wikipediaSearch` faz
   `fetch` à API pública keyless → `StudyWebSourceInput[]` → alimenta `web_sources` do
   `study_web_prepare/finalize` (F3.12a já aceita). As citações `[W:n]`/`kind="Web"` vêm
   das URLs pelo Rust `ai-pure` — nunca do modelo. Toggle **DESLIGADO por padrão** + aviso
   de privacidade no `ReaderStudyPanel`.
3. **Comparação multi-IA web** — reuso puro de `askAnchored` (já web-ok, F2.7b);
   un-SKIP dos self-tests + prova headless (2× `askAnchored` mock, `cited_match=true`).

`the-light` intocado (`04b9b24`), `core/Cargo.toml` inalterado; a única mudança em `core/`
é `core/src/lib.rs`. Grafo wasm puro; chave session-only só no header; sem regressão.

## Decisões / ADRs

- **ADR-0032** (aceito) — conversa web via `session_web_prepare` dirigindo `ai::ask_session`
  por um `CaptureProvider` que captura o `(system, user)` EXATO do nativo → **ZERO DRIFT**
  (provado por teste de paridade host). Reuso puro de `AiWebRequest` + `ai_web_finalize`
  (nenhum finalize novo). Única mudança em `core/` = `core/src/lib.rs`; `core/Cargo.toml`
  intocado.
- **Wikipedia keyless fetch opt-in** (ADR-0028) alimentando `web_sources` do
  `study_web_prepare/finalize` (F3.12a) → citações `[W:n]`/`kind="Web"` montadas pelo Rust
  a partir das URLs, nunca do modelo. Padrão **OFF** + aviso de privacidade; sem backend =
  0 citação Web.
- **Comparação web por reuso** de `askAnchored` (F2.7b) — sem glue/fronteira nova; só
  un-SKIP dos self-tests.
- **`related = []` no MVP** (ADR-0032): o contexto/prompt é do MESMO Rust `ai::ask_context`;
  popular `related` (rótulos de xref do store web) é follow-up — não é drift do
  anti-alucinação (é recuperação de store, infra TS, ADR-0011).
