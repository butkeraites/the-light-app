`reqwest`+`rusqlite`+`chrono`+`directories`+`toml`+`tempfile`+`serde_json`
(`Cargo.toml [features]`). **Achados (o que é PURO vs. o que puxa peso):**

- **`citation.rs` — puro exceto 1 função.** `use super::research::WebSource;` (l.18) e

  puro** (`serde_json` + `crate::model::Lang` + `VerifiedLexicon` que é tipo PURO):
  `rewrite_anchors` (anti-alucinação), `Citation`, `CitationCollector::{new,push,
  from_verified_lexicon,into_vec}`, `sbl_footnote`/`sbl_bibliography_entry`,
  `provenance_footer`, `to_json`/`from_json`.

  usam **rusqlite** (`use rusqlite::{params, Connection}` l.14). Mas os **tipos e a
  verificação são PUROS**: `LexicalEntry`, `VerifiedLexicon`, `format_verified_block`,
  `verify`/`VerifiedOutput`, `cited_strongs`, `base_strong`, `EMPTY_SENTINEL`.
- **`providers.rs` — misto.** `blocking_client`/`send_json`/os `post()` e os `impl
  LlmProvider` concretos (Anthropic/OpenAI/Ollama/Gemini) + `build_provider` usam
  **reqwest**. Mas `default_model` (l.26), `estimate_cost_usd` (l.74), os `*_body`/
  `*_extract` (puros, testáveis sem rede), `parse_api_response`, `api_error_msg`,
  `messages_json` são **puros** (só `serde_json`).

  usam `directories`; todo o resto é **puro** (`ask_system_prompt`,

- **`keys.rs` — embedded puro-e-simples.** `KeyStore` usa `directories`+`toml`+
  `crate::util::atomic_write` (fs). **Nada da Fase 2 web usa o `KeyStore` do core**
  (a chave é BYOK app-side, F2.4). É Fase 3/estudo local.

  default não-quebrante, já mergeado na F2.3), `MockLlmProvider`, `ChatMessage`/

  os produz. `Io`/`NoConfigDir` são só `std` (ok no wasm).

**Conclusão da investigação:** as partes que a Fase 2 (ask ancorado + anti-alucinação)
precisa no web são **genuinamente puras** e dependem apenas de `crate::model` (sempre
disponível) + `serde_json` (wasm-safe). O que puxa `reqwest`/`rusqlite`/`chrono`/
`directories`/`toml` está **isolável** (módulos `research`/`keys` inteiros + funções
específicas de `providers`/`lexicon`/`study`/`citation`/`prompts`). **Não precisa
tocar nada fora de `ai/` (+ 1 linha em `lib.rs` + o bloco `[features]` do Cargo.toml).**

## Estratégia de desacoplamento (a MENOS arriscada — feature fina `ai-pure`)

Introduzir uma **feature fina `ai-pure`** que compila SÓ a superfície pura do `ai`
(montagem de prompt/RAG + `ask` ancorado + stripping de citação anti-alucinação),
**gateando por `embedded`** tudo que puxa `reqwest`/`rusqlite`/`chrono`/`directories`/
`toml`/`tempfile`. Justificativa: é aditiva, opt-in, e mantém os defaults byte-a-byte
(CLI/TUI/xtask compilam exatamente como antes). Passos:

1. **`Cargo.toml` do core — nova feature fina:**
   - `ai-pure = ["dep:serde_json"]`  (serde_json é Rust puro, compila p/ wasm).
   - `embedded = ["ai-pure", "dep:chrono", "dep:directories", "dep:reqwest", "dep:rusqlite", "dep:serde_json", "dep:tempfile", "dep:toml"]`
     (embedded **inclui** `ai-pure` → sob default TUDO do `ai` segue compilando).
   - **`ai-pure` NÃO entra no `default`** — permanece `default = ["embedded"]`.
2. **`lib.rs` — 1 linha:** `#[cfg(feature = "embedded")] pub mod ai;` →
   `#[cfg(any(feature = "embedded", feature = "ai-pure"))] pub mod ai;`.
   (Os demais módulos pesados seguem `#[cfg(feature="embedded")]`; `model`/`reference`
   já são sempre-on e são o único `crate::` que a superfície pura do `ai` usa.)
3. **`ai/mod.rs`:**
   - Gatear os **submódulos pesados** por `embedded`: `#[cfg(feature="embedded")] pub mod
     research;` e `#[cfg(feature="embedded")] pub mod keys;` (ambos Fase 3/4; puxam
     reqwest+chrono / directories+toml). Manter `citation`/`lexicon`/`prompts`/`providers`/
     `study` sempre em `ai` (as partes pesadas deles são gateadas item-a-item, abaixo).
   - Dividir os `pub use` (l.17-25): manter os **puros** (ex.: `study::{ask, ask_context,
     numbered_passage, numbered_verses, split_sections, parse_refinement, refine_scope}`,
     `citation::{Citation, CitationCollector, CitationKind}`, `providers::{default_model,
     estimate_cost_usd}`, `lexicon::{VerifiedLexicon, LexicalEntry, verified_lexicon?...}`)
     e **gatear por `embedded`** os pesados (`keys::KeyStore`, `research::{...}`,
     `providers::build_provider`, `study::{study, StudyRequest, StudyResult}`,
     `lexicon::verified_lexicon` — que é rusqlite).
   - Gatear as variantes `AiError::Toml`/`AiError::TomlSer` por `#[cfg(feature="embedded")]`
     (puxam `toml`; só `keys` as usa). Manter `Io`/`NoConfigDir`/demais (só `std`).
4. **`ai/providers.rs`:** manter puros `default_model`, `estimate_cost_usd`, `*_body`/
   `*_extract`, `parse_api_response`, `api_error_msg`, `messages_json`; **gatear por
   `embedded`** `blocking_client`, `send_json`, os `struct`+`impl LlmProvider` concretos
   (Anthropic/OpenAI/Ollama/Gemini com `post()` reqwest) e `build_provider` (fábrica dos
   providers de rede). No wasm, o provider real é TS/`fetch` (F2.7b); o prompt/custo vêm
   do Rust.
5. **`ai/lexicon.rs`:** `use rusqlite::…` + `verified_lexicon`/`collect`/`attributions_for`/
   `resolve_verses`/`Agg` sob `#[cfg(feature="embedded")]`; manter puros os tipos +
   `format_verified_block`/`verify`/`cited_strongs`/`base_strong`/`EMPTY_SENTINEL`.
6. **`ai/study.rs`:** gatear por `embedded` o `use super::lexicon::{self,…}` e `use
   super::research::WebSource` (mova-os para blocos `#[cfg(feature="embedded")] use …`),
   e os itens pesados: `StudyRequest`, `StudyResult`, `study()`, `user_prompt`,
   `cited_web_indices`, `to_markdown`/`to_academic_markdown`. Manter puros:
   `numbered_verses`, `numbered_passage`, `ask_context`, `ask_user_prompt`, `ask`,
   `ask_session`, `split_sections`, `StudySection`, `parse_refinement`, `Refinement`,
   `refine_scope`/`refine_system_prompt`/`refine_user_prompt`. Gatear o `#[cfg(test)]`
   pesado (ou seus casos que usam `study`) por `embedded` — sob default nada muda.
7. **`ai/citation.rs`:** gatear por `embedded` o `use super::research::WebSource` e
   `from_web_results`; manter o resto (usa `serde_json` + `VerifiedLexicon` puro +
   `model::Lang`).
8. **`ai/prompts.rs`:** gatear por `embedded` `prompts_dir()` e `system_prompt()` (usam
   `directories`); manter `ask_system_prompt`/`study_followup_system_prompt`/
   `system_prompt_in`/`builtin`/`section_blueprint`/… (puros; `system_prompt_in` usa só
   `std::fs`, que compila no wasm).

**Regra de ouro:** nenhum item compilado sob `ai-pure` pode referenciar um item gateado
por `embedded`. A prova é o build wasm (abaixo). Se algum acoplamento forçar mover algo
para fora de `ai/` OU exigir refator grande/quebrante, **NÃO improvisar**: preferir gatear
MAIS coisas por `embedded` (variante conservadora) e, se ainda assim inviável, `blocked`
+ HALT com o erro EXATO.
