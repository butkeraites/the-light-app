## Objetivo

Entregar, no NATIVO (iOS/Android), uma **UI de conversa multi-turno** ancorada numa
passagem/estudo: a partir de uma passagem selecionada no Reader, o usuário mantém uma
**conversa com follow-up** (histórico de turnos User/Assistant) sobre aquela passagem.
Cada follow-up chama a fronteira **`ask_session_anchored`** (F3.4) via o glue nativo →
JSI, mantendo o **contexto ancorado do STORE** (a âncora = texto do versículo, montada do
store local pelo core). A UI exibe a conversa com o `cited_text` (âncora, verbatim do
store) **separado/rotulado** de cada `interpretation` (LLM) — **anti-alucinação visível**
(molde F2.5/F3.5). Prova **determinística com MOCK** (sem chave, sem rede): self-test no
device emite `TLA_CHAT`. Só nativo — a **conversa web é a F3.12** (o glue web é stub).

Escopo desta tarefa (atômico): **UI de conversa + glue de conversa + self-test + bindings
regenerados**. NÃO toca a fronteira Rust (F3.4 já entregou `ask_session_anchored`; nenhuma
função nova). NÃO usa chave/rede (só `"mock"`). NÃO faz a conversa web (F3.12).

## Contexto / arquivos

### Fronteira REAL de F3.4 (CONFIRMADA em `core/src/lib.rs` — NÃO modificar; NÃO é escopo)
`ask_session_anchored` (l.1656) — **atenção à ordem REAL dos argumentos** (contra
alucinação): `lang` vem **ANTES** de `turns`; `study_mode`/`study_lens` vêm **DEPOIS** de
`turns` e **ANTES** de `provider_name`; a passagem é `book/chapter/verse` **NUMÉRICOS**
(u8/u16), **não** a string "John 3:16":

```rust
pub fn ask_session_anchored(
    db_path: String,
    translation: String,
    book: u8,
    chapter: u16,
    verse: Option<u16>,
    lang: String,
    turns: Vec<ChatTurn>,
    study_mode: Option<StudyMode>,
    study_lens: Option<StudyLens>,
    provider_name: String,
    key: Option<String>,
    model: Option<String>,
) -> Result<AiAnswer, CoreError>
```

- Retorno **REUSA** `AiAnswer { reference, cited_text, interpretation, provider, model }`
  (l.2121). `cited_text` = passagem **numerada, verbatim do store** (a âncora, montada pelo
  core via `Store::open`+`EmbeddedSource::passage`+`numbered_passage`); `interpretation` =
  saída **só do modelo** (o mock). **NÃO há campo `turns` no retorno** → o marcador
  `turns=<n>` do self-test vem do **tamanho do histórico de ENTRADA** (a conversa que o app
  montou), não do retorno.
- `ChatTurn { role: ChatRole, content: String }` + `ChatRole { User, Assistant }` (l.1438/
  l.1463) — puros/`ai-pure`, já nos bindings gerados (nativo e web) por F3.4.
- Invariante do core (byte-verificada em F3.4): o `context` (âncora) é embutido **só no 1º
  turno de usuário**; o mock devolve uma interpretação FIXA. A conversa mantém a âncora
  porque a UI passa **sempre** o mesmo `book/chapter/verse` do store.
- **Opcionais** (NÃO obrigatórios ao aceite): `refine_scope` (l.1578) e `parse_refinement`
  (l.1624) para sugerir refinamentos de escopo. `RefinementOut { question, options }`,
  `RefinementRound { question, answer }` já estão nos bindings gerados.

## Restrições

- **NÃO tocar `the-light` nem a fronteira Rust** (`core/src/lib.rs`, `core/Cargo.toml`).
  F3.4 já entregou `ask_session_anchored`/`ChatTurn`/`ChatRole` (superfície `pub`,
  `ai-pure`, em `c8ecb2f`) — **nenhuma função/tipo novo** é necessário. Se, e SOMENTE se,
  a UI de conversa exigir mudar o core: **`blocked` + HALT** com o erro EXATO (não
  stubar/forkar/copiar o core).
- **SEM chave/rede**: só `provider="mock"` (determinístico, offline). A chave real + rede
  são a **F3.10** (gate). **Chave nunca logada** (BYOK/offline-first — regra não negociável).
- **Anti-alucinação VISÍVEL**: o `cited_text` (âncora) vem **sempre** do store verbatim,
  exibido **separado/rotulado** de cada `interpretation` (LLM). O texto bíblico **nunca**
  vem do modelo.
- **Só NATIVO**: a conversa web é a **F3.12** (o glue web é stub `= F3.12`; sem arrastar
  `ai`/store/`rusqlite` para o bundle web). Prova em ≥1 alvo nativo (iOS); o mesmo gancho de
  `selftest.ts` roda no Android (bônus, sem regressão).
- **Ordem REAL dos argumentos** de `ask_session_anchored` (contra alucinação): `lang`
  **antes** de `turns`; `study_mode`/`study_lens` **depois** de `turns`; passagem
  **numérica** (book/chapter/verse), não string.
- **Qualidade por tarefa**: `tsc --noEmit` limpo; `expo export web` 0; eslint sem erros
  novos. Verde antes do commit (prefixo `F3.6` na mensagem).
- **FOLLOW-UP do ADR-0027 (NÃO é escopo da F3.6):** corrigir os doc-comments com
  `**puro**` (asterisco colado) em `core/src/lib.rs` fica **pendente** para quando alguma
  tarefa **tocar `lib.rs`** (ex.: F3.12). F3.6 **não** toca `lib.rs`, então **não** o
  resolve aqui — apenas registrado para rastreio.
