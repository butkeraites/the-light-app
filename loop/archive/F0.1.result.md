## Decisões / ADRs

- **ADR-0001 (aceito, F0.1):** stack = core Rust consumindo `the-light-core` numa
  crate de fronteira `core/`, app Expo/React Native + TypeScript (Web/iOS/Android),
  fronteira via UniFFI (`ubrn`), store `rusqlite` bundled no nativo e
  `wa-sqlite`+OPFS no web, transporte de IA plugável (reqwest/fetch) BYOK. Risco
  técnico central (SQLite no WASM) a validar na Fase 0; bindings gerados não
  versionados (geração reprodutível fica para a F0.4).
- **ADR-0002 (aceito, F0.1):** consumir `the-light-core` como **git dependency do
  Cargo pinada por commit** (`{ git = "…/the-light", rev = "<commit>" }`), e
  **não** como submódulo — por reprodutibilidade sem subárvore, atualização
  auditável via `rev` em commit/ADR, fronteira limpa (core intacto; mudança no
  core só via PR no `the-light` + ADR) e tooling padrão (`cargo metadata`).
  Trade-off: desenvolvimento simultâneo do core usa `[patch]` local temporário,
  sem commitar.
- Verificação independente confirma que nenhuma decisão exigiu tocar/forkar o
  `the-light`: a F0.1 entrega só a decisão documentada.
