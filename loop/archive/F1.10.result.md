## Resumo

A F1.10 expõe na fronteira UniFFI (`core/src/lib.rs`) o CRUD de **notas** e
**marcações/highlights** por referência, **delegando** integralmente à camada
file-based `the_light_core::userdata` (situação A confirmada na fonte pinada):
`put_note`/`get_note`/`delete_note`/`list_notes` via
`userdata::notes::NoteStore` e `add_highlight`/`remove_highlight`/`list_highlights`
via `userdata::highlights::HighlightStore`. As 7 funções recebem um `data_dir`
gravável e derivam `notes/` + `highlights.json` do mesmo layout default do core;
**nenhuma recebe `db_path` nem toca o `bible.sqlite`** (separação de dados). A
referência chega como `String` e é canonicalizada pelo core
(`reference::parse_reference`) — PT e EN caem na MESMA nota/arquivo, sem `From`
reverso inventado. Records `Note`/`Highlight` puros (todos os alvos), com `From`
e corpos gated `cfg(not(target_arch = "wasm32"))` + stub web (mesma mensagem-padrão
das demais funções DB-backed). Verificação independente: fmt/clippy/test verdes
(34 testes, 8 novos `userdata_tests`), web puro (sem `rusqlite` no grafo wasm),
`gen-bindings-web` verde com as 7 funções como stub, `the-light` intacto
(`8f66004`), `core/Cargo.toml` inalterado. Commit `b6381d0` é **pura adição**
(623 inserções, 0 remoções) tocando só `core/src/lib.rs`. **Decisão: passed.**

## Decisões / ADRs

- **Sem ADR novo.** Situação **A** confirmada: a fronteira **delega** à camada
  file-based já existente do `the-light-core::userdata` — `NoteStore` (um `.md`
  por referência em `notes/`, slug canônico EN) + `HighlightStore`
  (`highlights.json`, array legível). Nenhum store reimplementado na fronteira.
- **`data_dir` gravável separado do `bible.sqlite` só-leitura:** as 7 funções não
  recebem `db_path`; derivam `notes/`+`highlights.json` do mesmo layout default do
  core (formato aberto/exportável p/ F1.11/F1.16). A fronteira nunca chama
  `data_dir()`/`open_default`/`load_default` → `UserDataError::NoDataDir` não ocorre.
- **Sem mudança no `Cargo.toml`:** `embedded` já trazia `serde_json`/`directories`/
  `tempfile` desde a F1.2; nada novo entra no grafo wasm. `uniffi = "=0.31.0"` intacto.
- Aderência a ADR-0002 (git dep pinada), ADR-0005 (matriz de features; `userdata`
  nativo-only), ADR-0010 (gating de corpo + stub web).
