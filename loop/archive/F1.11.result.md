## Resumo

UI de notas/highlights NATIVA entregue e provada de forma independente. O painel
`ReaderVersePanel` (editor de nota Markdown + chips de cor da paleta nomeada +
export + xref) abre pelo MESMO gesto de seleção de versículo da F1.9; os
indicadores por versículo (cor de fundo do highlight + marcador `✎` de nota) entram
no `ReaderChapterView` via props OPCIONAIS retrocompatíveis. Notas/highlights
passam SEMPRE pela fronteira `userdata` (F1.10): o glue nativo `reading.ts` estende
com `putNote`/`getNote`/`deleteNote`/`listNotes`/`addHighlight`/`removeHighlight`/
`listHighlights` (+ tipos `Note`/`Highlight`) delegando aos bindings gerados →
JSI → Rust; nenhuma reimplementação de I/O/serialização/slug/ordenação de userdata
em TS. Persistência em `${documentDirectory}userdata/` (app/lib/userdata.ts),
diretório SEPARADO e gravável, distinto do `reading-sample.sqlite` só-leitura. O
export (`notesExport.ts`) é função PURA que apresenta os Records de
`list_notes`/`list_highlights` (sem reescrever o store), compartilhado via `Share`
do react-native (sem `expo-sharing`). A prova HEADLESS no device emitiu
`TLA_NOTES note_ref="John 3:16" note_len=55 highlights=1 persisted=true
export_ok=true` SEM regressão dos demais marcadores; `tsc`/`expo export` verdes;
the-light pinado (8f66004) e core/src/lib.rs + core/Cargo.toml intactos.

## Decisões / ADRs

- **ADR-0017 (NOVO, aceito):** (1) `data_dir` de userdata em
  `${documentDirectory}userdata/` (gravável, idempotente, path real sem `file://`),
  SEPARADO do `reading-sample.sqlite` só-leitura — `ensureUserDataDir()` distinto de
  `ensureReadingDb()`; (2) EXPORT = agregado Markdown montado dos Records via
  `buildNotesExport` (função pura, apresentação), NÃO serialização do store; (3)
  Share via `react-native` `Share` (texto), SEM `expo-sharing` — `package.json`
  inalterado; (4) UI no `ReaderVersePanel` (mesmo gesto F1.9) + indicadores
  retrocompatíveis no `ReaderChapterView`; (5) persistência provada por 2ª leitura
  independente (`persisted=true`).
- Reusa **ADR-0016** (atribuição CC-BY do xref, `XREF_ATTRIBUTION` preservada),
  **ADR-0014** (bundling/`documentDirectory`), **ADR-0015** (tema), **ADR-0010/0005**
  (matriz por alvo; anti-alucinação aplica-se à referência, não ao corpo da nota).

## Bloqueios / perguntas

- **Nenhum bloqueio.** A paridade web de notas/highlights é a **F1.16** (pós-gate
  F1.12) — aqui o glue/serviço/self-test web são stubs que lançam em runtime,
  mantendo `tsc`/build web verdes (sem `expo-file-system` no bundle web).
- **PRÓXIMA É O GATE F1.12** (`gate: true` — store web do corpus completo:
  FTS5/OPFS, Opção A vs B, notas/highlights no web). Com a F1.11 aceita, esta é a
  última tarefa nativa da Fase 1: o loop deve **PARAR (HALT)** p/ sign-off
  humano/auditoria antes de executar a F1.12. Não há outra tarefa nativa pendente
  entre F1.11 e o gate.
