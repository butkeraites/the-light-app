## Objetivo

Fechar a **paridade web do estudo profundo** (2ª e última fatia da F3.12), sobre a
fronteira que a **F3.12a** já entregou (`study_web_prepare`/`study_web_finalize`,
que **já aceita `web_sources`**, vazio na F3.12a). Três entregas **app-side**, com
**fonte única em Rust / ZERO DRIFT** (ADR-0029):

1. **Conversa ancorada web** (`askSessionAnchored`): destubar em `reading.web.ts`
   para o `ReaderChatPanel` funcionar no browser. `ai::ask_session` **já é `ai-pure`**
   (F3.4) — a menor via zero-drift é **UMA fronteira web nova em `core/src/lib.rs`**
   (`session_web_prepare`, app-side, **NÃO** toca `the-light`) que dirige a `ask_session`
   PURA por um `CaptureProvider` (o `chat` default DOBRA o transcript → `complete`
   captura o `system`/`user` EXATOS que o nativo enviaria), + transporte `fetch` +
   **reuso** de `ai_web_finalize`.
2. **Pesquisa web Wikipedia (opt-in, ADR-0028)**: um helper TS (`research.web.ts`)
   que faz `fetch` à API pública da Wikipedia (**keyless**) → `WebSource[]` →
   alimenta `web_sources` do `study_web_prepare`/`study_web_finalize` (F3.12a) → o
   estudo web ganha as citações `[W:n]` (como o nativo F3.9a/F3.10). **Padrão
   DESLIGADO + aviso de privacidade** no `ReaderStudyPanel`. **Sem re-tocar o Rust**
   (a fronteira já aceita `web_sources`).
3. **Comparação multi-IA web**: `askAnchored` **já foi destubada** (F2.7b) → o
   `ReaderComparePanel` já funciona no web; **un-SKIP** do `compare-selftest.web.ts`
   e prova headless (N `askAnchored` mock, mesma âncora).

**Regra dura (ADR-0029):** proibido espelhar em TS o anti-alucinação (prompt/
citação/verify/aparato). A montagem da conversa (`ask_session` → `ask_context` →
`ask_user_prompt` → dobra do transcript) e o bloco `[W:n]`/citações web
(`user_prompt` + `from_web_results` + `cited_web_indices`) vêm do **MESMO Rust
`ai-pure`** no web e no nativo. Só a **recuperação do store** (SELECT léxico,
ADR-0011) e o **transporte** (`fetch` ao LLM e à Wikipedia, ADR-0025/ADR-0028) são
infra TS.

### Desenho da fronteira web NOVA (só em `core/src/lib.rs`, cfg-free/`ai-pure`)

`session_web_prepare(book: u8, chapter: u16, verse: Option<u16>, lang: String,
turns: Vec<ChatTurn>, study_mode: Option<StudyMode>, study_lens: Option<StudyLens>,
provider_name: String, model: Option<String>, verses: Vec<AiVerseInput>,
related: Vec<String>) -> Result<AiWebRequest, CoreError>` (corpo cfg-free, espelha
`ask_session_anchored` 2–6):
1. `reference` = `Reference::single(book,chapter,v)` ou `whole_chapter`; `lang` default Pt;
2. `cited_text = ai::numbered_verses(verses)` (VERBATIM do store web);
3. `label = reference::format_reference(&reference, lang)`;
   `context = ai::ask_context(&label, &cited_text, &related)`;
4. `study = study_mode.zip(study_lens).map(|(m,l)| (m.into(), l.into()))`;
5. `messages = turns.into_iter().map(Into::into).collect()` (`Vec<ChatMessage>`);
   `model = model|default_model(provider)`;
6. `CaptureProvider` dirigido por `ai::ask_session(&cap, lang, &context, &messages, study)`
   → captura `(system, user)`; devolve `AiWebRequest { reference, cited_text, system,
   user, provider, model }` (**reusa o Record de `ai_web_prepare`**).

Finalize = **reusar `ai_web_finalize(reference_label, cited_text, provider, model,
interpretation)`** (rewrite_anchors com válidas vazio → limpa âncoras espúrias; o
mesmo que o `ask`). O corpo usa **só** `ai-pure` → grafo wasm segue puro. Regenerar
bindings (`gen-bindings-web.sh`).
