## Objetivo

Entregar a **exportação acadêmica** de um estudo profundo (F3.3/F3.5): a partir de
um `StudyResultOut` (retorno de `deep_study`, provedor **mock**), produzir um
**Markdown acadêmico (SBL)** — metadados YAML, texto citado do store, análise com
**notas de rodapé SBL**, bibliografia e **rodapé de procedência com a atribuição
STEP CC-BY** — mais um **sidecar de citações** (metadados/atribuições/proveniência),
e permitir **compartilhar/salvar** pelo caminho de Share nativo da F1.11. Só
**nativo (iOS/Android)**; web = stub (F3.12). **Prova determinística por MOCK**
(sem chave/rede), molde F3.5/F1.11.

**Regra de ouro (anti-drift / anti-alucinação):** o Markdown SBL é produzido pela
**mesma impl do core** (`StudyResult::to_academic_markdown`, rev pinado `c8ecb2f`)
— **NÃO** reimplementar a serialização SBL/proveniência no app. O texto citado e as
citações vêm **do store/banco**; a interpretação é **rotulada como gerada por IA**.

## Contexto / arquivos

### O que a investigação da FONTE do core (`c8ecb2f`, só-leitura) determinou
Checkout do cargo `~/.cargo/git/checkouts/the-light-9eb8809a6d68281a/c8ecb2f/crates/the-light-core/src/ai/`
(**não** o `/Users/butkeraites/Documents/the-light` bloqueado):

- **`StudyResult::to_academic_markdown(&self, lang: Lang) -> String`** (`ai/study.rs:508`)
  é **`pub`**, dentro de `impl StudyResult` marcado `#[cfg(feature = "embedded")]`
  (`ai/study.rs:425`). Ou seja: **`pub` porém `embedded`-only** → **callável a
  partir da fronteira NATIVA** (que compila com `embedded`, `core/Cargo.toml:42`),
  **sem PR ao the-light**. Produz: bloco YAML (`title`/`author`/`lang`), `## Texto
  (acervo local)` (= `passage_text` verbatim do store), `## Análise` (=
  `interpretation`/`sections` com âncoras `[V:Strong]`/`[W:n]` **reescritas
  deterministicamente** para `[^chave]` — o modelo só emite âncoras, o **core
  valida e descarta as inválidas** via `citation::rewrite_anchors`), `## Notas`
  (SBL, `citation::sbl_footnote`), `## Bibliografia` (`citation::sbl_bibliography_entry`),
  e `citation::provenance_footer` que **separa verificável (acervo local, com a
  atribuição STEP CC-BY das citações `Source`) do gerado por IA** ("a análise e a
  interpretação são geradas por {provider}/{model} … podem conter erros — confira
  sempre as fontes primárias").
- **`StudyResult::to_markdown(&self) -> String`** (`ai/study.rs:432`) é o Markdown
  **simples** (não-SBL); NÃO é o alvo desta tarefa (queremos o **acadêmico**).
- **`ai::citation`** (`citation.rs`): `Citation` (struct serde com
  `kind,key,author,title,locus,publisher,year,url,license,attribution,accessed,quote`
  — doc do core: "round-trippável p/ o sidecar `.citations.json`"), `CitationKind`
  {Scripture,Lexicon,Source,Web}, `sbl_footnote`/`sbl_bibliography_entry`/
  `provenance_footer`/`rewrite_anchors`/`to_json`/`from_json` — **toda a lógica SBL
  vive AQUI, no core**. Reimplementá-la no app = drift proibido.
- **Atribuição STEP CC-BY** aparece no `provenance_footer` porque `study()` constrói
  citações `Source` a partir do `verified_lexicon` (`CitationCollector::from_verified_lexicon`),
  cujas `sources` são a atribuição STEP verbatim (`Credit it to 'STEP Bible' … CC BY 4.0`).
  João 3:16 tem léxico no subset (F3.5/ADR-0027: `lexicon=21`) → a atribuição
  **estará** no Markdown acadêmico → prova robusta.

### DECISÃO — expor via fronteira (fonte única, zero drift) — PREFERIDA
Como `to_academic_markdown` é **`pub`** (só `embedded`-gated, o que é OK para a
fronteira nativa), o Markdown SBL deve ser **produzido pelo core** (fonte única),
**não** formatado no app. **Recomendação (primária):**

- **(C) Estender o Record `StudyResultOut` com um campo `academic_markdown: String`,
  populado no `From<StudyResult>` existente** (l.1256, `cfg(not wasm32)`) via
  `academic_markdown: result.to_academic_markdown(result.language)`. Vantagens:
  (i) **fonte única / zero drift** (o core formata); (ii) **UMA única chamada** —
  reusa o estudo que a UI já rodou (crítico para BYOK real na F3.10: **não** re-roda
  o LLM); (iii) **sem reconstrução reversa** de tipos. O campo é `String` no Record
  (harmless no wasm — o stub web de `deep_study` retorna `CoreError`, nunca
  constrói o Record). A computação extra é só formatação de string (barata).
- **Alternativa aceitável (A):** nova função de fronteira
  `deep_study_academic_markdown(...mesmos args de deep_study...) -> Result<String,
  CoreError>` que compartilha um helper privado com `deep_study` (constrói o
  `StudyRequest`, roda `study()`) e retorna `result.to_academic_markdown(lang)`.
  **Caveat:** re-roda `study()` → com provedor REAL isso é uma **2ª chamada paga**
  (na F3.10). Escolha (A) só se preferir não alterar a forma do Record; **então**
  documente o caveat. Em ambos os casos: **toca `core/src/lib.rs`**, corpo/`From`
  sob `cfg(not(target_arch="wasm32"))` + **stub web** (paridade = F3.12), grafo
  wasm puro (sem `rusqlite`/`reqwest`).

**NÃO** escolher a via (b) app-side (formatar o Markdown SBL a partir dos campos)
— isso reimplementaria `rewrite_anchors`/`sbl_footnote`/`sbl_bibliography_entry`/
`provenance_footer` (drift proibido).

### Sidecar de citações — app-side (molde F1.11), SEM nova fronteira
O **sidecar** é **metadados** (não é a serialização SBL). Como `deep_study` **já
retorna** `StudyResultOut.citations` (do banco), o sidecar é montado **app-side**
por uma função **pura** (molde `notesExport.buildNotesExport`): um JSON com
`{ reference, provider, model, generated_by_ai_disclaimer, attributions: [STEP
CC-BY, …], citations: StudyCitation[] }`. Isso **agrega Records já retornados** (não
reimplementa a serialização SBL do core). O doc do core confirma que `Citation` é
"round-trippável p/ o sidecar `.citations.json`".

### Compartilhar/salvar (molde F1.11)
Botão **"Exportar (acadêmico)"** (`testID=study-export-academic`) no
`ReaderStudyPanel`, habilitado quando há `result`; ao acionar, passa
`{ markdown: academicMarkdown, sidecar }` ao **Share do `react-native`** (mesmo
caminho da F1.11 — sem dependência pesada nova). Web = stub (F3.12).

### FOLLOW-UP ADR-0027 (obrigatório porque ESTA tarefa toca `core/src/lib.rs`)
A F3.5 saneou nos GERADORES (`scripts/gen-bindings-{ios,web}.sh`, um
`perl -pe 's{\*\*/}{** /}g'`) um defeito de origem: **9** doc-comments em
`core/src/lib.rs` contêm a sequência `**puro**/` (negrito seguido de barra, ex.:
`tipo **puro**/\`ai-pure\``, linhas 846/874/993/1036/1085/1122/1146/1433) que fecha
prematuramente o bloco JSDoc dos bindings gerados. **Como esta tarefa TOCA
`lib.rs`**, corrija a RAIZ: troque `**puro**/` → `**puro** /` (um espaço antes da
barra) nas 9 ocorrências (é só comentário; não altera tipo/assinatura/comportamento).
Com isso o saneamento `**/` dos geradores fica **redundante** — pode ser mantido
(defensivo) ou removido; a acceptance é `tsc --noEmit` verde nos bindings
regenerados. **NÃO** tocar o `the-light` (o defeito está no `core/src/lib.rs` da
FRONTEIRA, que podemos editar).

## Restrições

- **NÃO tocar o `the-light`.** `to_academic_markdown` já é **`pub`** (só
  `embedded`-gated, OK para a fronteira nativa) → **expor via fronteira SEM PR**. Se
  o executor descobrir que algo exigido é **não-`pub`** ou wasm-inviável, então
  **`blocked` + HALT** com o erro EXATO (PR sancionado = molde F2.7/F3.11). **Não**
  stubar/forkar/copiar o core.
- **NÃO reimplementar a serialização SBL/proveniência** do core no app: o Markdown
  acadêmico vem **inteiro** de `StudyResult::to_academic_markdown`; o sidecar só
  **agrega** os `StudyCitation` já retornados (molde F1.11 `buildNotesExport`).
- **NÃO** escolher a via (b) app-side de formatação do Markdown (drift proibido).
- **Atribuições obrigatórias no export:** STEP CC-BY (léxico) presente no rodapé de
  procedência; xref OpenBible CC-BY / domínio público quando citados; visíveis.
- **Anti-alucinação:** o texto citado e as citações vêm do **store/banco**
  (verbatim); a **interpretação é rotulada como gerada por IA** ("confira sempre as
  fontes primárias"); âncoras inválidas do modelo são descartadas pelo core.
- **BYOK / offline-first:** prova **só com `"mock"`** — **sem chave, sem rede**;
  chave **nunca** em git/log. A validação com provedor real = F3.10 (gate).
- **Só nativo.** O export web (helper/Share) = **F3.12** (stub); o corpo/`From` que
  toca `study`/`StudyResult` é `cfg(not(target_arch="wasm32"))` + stub web; grafo
  wasm puro.
- **Como esta tarefa TOCA `core/src/lib.rs`, resolver o FOLLOW-UP do ADR-0027**
  (`**puro**/` → `**puro** /`, 9 ocorrências), tornando o saneamento `**/` dos
  geradores redundante.
- `core/Cargo.toml` **inalterado**; qualidade por tarefa verde (Rust
  `fmt`/`clippy -D warnings`/`test`; TS `tsc --noEmit`/eslint).
- **Sem ADR novo obrigatório** (consome o core pinado; reusa ADR-0016/0017/0026/0027).
  Se surgir decisão de forma do export (nome/local do sidecar, campo vs fn), pode
  registrar como **ADR-0028** (próximo livre). Diante de conflito com qualquer regra:
  **HALT**, não improviso.
