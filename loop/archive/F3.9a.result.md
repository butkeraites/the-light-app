## Decisões / ADRs

- Implementa **ADR-0028 (Gate F3.9 / D1)**: pesquisa web assistida = **Wikipedia
  keyless, OPT-IN** no `deep_study` via `ai::research`. O param `research_backend` é
  **opt-in, padrão DESLIGADO** (`None` → `web_sources = vec![]`, offline; F3.3 preservada).
- **Delegação ao core, sem drift:** a fronteira só chama `build_research_provider`
  (chave **sempre `None`** — Wikipedia é keyless) + `provider.search`; a montagem de
  prompt/busca/citação (`from_web_results`, `[W:n]`, `cited_web_indices`) permanece no
  `study()` do core. Nada reimplementado na fronteira.
- **Anti-alucinação:** as citações `Web` vêm das **URLs realmente buscadas** (mock
  canônico → `example.org`), **nunca do modelo**; `passage_text` verbatim do store;
  `interpretation` invariante à web.
- **Offline-first / BYOK preservados:** prova por **MockResearchProvider (sem rede/
  chave)**; `"wikipedia"` (rede opt-in do usuário) fica para a **F3.10**. Nenhuma chamada
  de rede no teste; nenhuma chave em git/log. `research` é embedded-only → grafo wasm
  segue **puro** (sem `rusqlite`/`reqwest`); estudo web = F3.12.
- `the-light` (`c8ecb2f`) e `core/Cargo.toml` **intocados**; **sem PR** (superfície
  `research` já `pub` sob `embedded`).
