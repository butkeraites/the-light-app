## Objetivo

Expor na fronteira UniFFI (`core/src/lib.rs`) a **recuperação de léxico Strong
verificado** de uma passagem, **delegando** à função embedded do core
`the_light_core::ai::lexicon::verified_lexicon(...)` — molde F1.5/F1.8 (fronteira
Rust + teste de host determinístico; export em **todos** os alvos, corpo DB sob
`cfg(not(wasm32))` + stub web; grafo wasm permanece puro). É a **primeira** peça de
NÚCLEO da Fase 3: entrega os dados léxicos (lema/transliteração/glosa/Strong +
atribuição das fontes) que a UI da F3.5 exibirá inline e que o estudo profundo da
F3.3 anexa ao prompt. **Anti-alucinação:** as glosas/lemas/Strong vêm do **léxico
local verificado** (STEP CC-BY, tabelas do `bible.sqlite` populadas na F3.1),
**nunca** do LLM; a fronteira só **entrega** o dado verificado.

Escopo = **só o núcleo** (fronteira + teste de host). A UI de léxico é a **F3.5**;
a paridade web do léxico é a **F3.12** (per gate F3.9). Sem UI, sem MOCK de LLM
(esta função **não** chama modelo — é lookup puro de banco).
