## Achados adversariais (8 checks)

1. **Reuso genuíno (OK).** `driveSync.ts` importa APENAS `exportSnapshot`/`importSnapshotIntoStore`/
   `serializeSnapshot`(+tipos) de `./userdataSnapshot`. Push = `exportSnapshot`→`serializeSnapshot`→upload;
   pull = `importSnapshotIntoStore`. Token vem só de `getToken()` injetado. Nenhum merge/parse/OAuth
   reescrito. Nenhum `import` de rede/glue/node.
2. **Pull valida antes de escrever (OK).** Pull roteia EXCLUSIVAMENTE por `importSnapshotIntoStore`,
   que faz `parseSnapshot` + `assertValidReference` (core `parseReference`) em TODAS as refs ANTES de
   qualquer escrita (base é leitura, merge é puro, writes só no fim). Referência irreal → lança sem
   corromper local. A rejeição de import irreal é provada na regressão F5.23 `test:web:snapshot` (caso D).
   Observação menor: a prova do drivesync não injeta um remoto corrompido próprio; coberto pela
   delegação total + regressão F5.23 (cláusula "ou" do critério satisfeita).
3. **Merge não destrói local (OK, real).** Caso (6): pull de remoto MENOR mantém a nota só-local
   (`afterBig.notes.length===2`, `Genesis 1:1` preservada). Caso (4): 2 stores separados
   (`makeMemoryDir(new Map())` distintos) + 1 "nuvem" compartilhada (`makeCloud()`): A.syncNow→B.syncNow
   →A.syncNow ⇒ `finalA deepEqual finalB`, união de 2 notas, `planProgress.completed===5` (max). Não é fake.
4. **Idempotência (OK).** Caso (5): `syncNow` 2ª vez seguida ⇒ `applied {notes:0,highlights:0,planProgress:false}`
   e `exportSnapshot(storeA) deepEqual finalA` (estado inalterado); `cloud.files.size===1`. fileId estável
   provado no caso (1) (`p2.fileId===p1.fileId`). Bytes idênticos garantidos por `exportSnapshot`
   determinístico (ordenado, sem `exportedAt`) — não asserido byte-a-byte no bloco idempotente, mas
   estruturalmente garantido; claim central verde.
5. **Push só o escopo do snapshot (OK).** Caso (2): parse do conteúdo enviado; assert `app==='the-light-app'`,
   `version===1`, chaves ⊆ {app,version,exportedAt,notes,highlights,planProgress}, ausência de
   `For God so loved`, regex negando `access_token|Bearer|apiKey|sk-|refresh_token|sessions|messages`,
   e `assertNoSecretIn`. Sem texto bíblico/IA/banco/chave.
6. **Token não vaza (OK, real).** Independentemente confirmei: 0 `console.*` em `driveSync.ts`, 0
   `client_secret`/`apiKey`/`sk-`/`refresh_token`. Token só em `Authorization: Bearer` (o mock exige em
   TODA request) e ausente de todo corpo capturado. `notoken=ok` é lastreado por asserções REAIS
   (espião de console + grep do fonte por `console.*` e `client_secret`, fora do try/finally) —
   `SECRETS=[MOCK_ACCESS_TOKEN]` não-vazio, não é no-op; marcador só imprime se tudo passa.
7. **Mock/offline-first/pureza (OK).** `fetch` injetado (nenhum `globalThis`/`window`/node/XHR no CÓDIGO
   — só em comentários); nenhuma conta/rede real. `driveSync` é importado APENAS pelo entry de teste,
   por NENHUMA tela do app → fora do entry eager (moduleCount 839 inalterado confirma). Nada essencial
   passou a exigir Google.
8. **Disciplina de escopo + não-negociáveis (OK).** Sem UI opt-in e sem wiring do store nativo (F5.26 —
   `driveSync` não é referenciado por telas). `the-light` intocado em `225b8c9`. `.claude/settings.json`
   modificado no working tree mas NÃO commitado. Commit `7f4fac2` toca só 5 arquivos
   (DECISIONS.md, lib/driveSync.ts, package.json, driveSync-headless-entry.ts, driveSync.web.test.mjs)
   — nada em `loop/` ou `core/`. ADR-0053 presente, coerente, reconcilia ADR-0036/0051/0052.
