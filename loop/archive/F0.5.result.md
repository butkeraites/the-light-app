## Decisões / ADRs

- **Node v25 compatível com Expo SDK 56.** A toolchain do SDK 56 (Metro/web)
  bundlou e exportou sem erro sob o Node do ambiente; sem incompatibilidade — o
  risco levantado na task (Node v25 × SDK 56) não se materializou.
- **Verificação não-interativa via web export.** A prova de compilação foi
  `expo export --platform web` (sai 0, gera `app/dist/`), não `expo start`
  (servidor interativo que não encerra) — correto para o loop headless.
- **Runs nativos adiados para F0.7/F0.8.** `expo run:ios`/`run:android` não foram
  exigidos nem usados (Xcode completo e Android SDK/NDK ausentes por F0.0/DECISIONS);
  pertencem às tarefas de ligação nativa.
- **`app/` é projeto npm independente da raiz.** Lockfile próprio
  (`app/package-lock.json`), sem mistura com o `package.json` da raiz (ubrn, ADR-0004).
