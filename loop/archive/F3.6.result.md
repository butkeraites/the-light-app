## Decisões / ADRs

- **Sem ADR novo.** A UI de conversa apenas CONSOME `ask_session_anchored` (F3.4)
  via callback `onChat`; nenhuma função/tipo novo no core. A superfície `pub`
  (`ask_session_anchored`/`ChatTurn`/`ChatRole`, `ai-pure`, em `c8ecb2f`) já existia.
- **Ordem REAL dos argumentos** confirmada no binding gerado e no glue (delegação 1:1):
  `lang` ANTES de `turns`; `studyMode`/`studyLens` DEPOIS de `turns` e ANTES de
  `providerName`; passagem NUMÉRICA (book u8 / chapter u16 / verse Option<u16>), não
  string "John 3:16".
- **Anti-alucinação** = `cited_text` (âncora, verbatim do store, montada pelo core)
  exibida separada/rotulada de cada `interpretation` (LLM). O texto bíblico nunca vem
  do modelo; o histórico reenviado carrega só as `interpretation` como turnos do
  assistente.
- **Mock sem chave/rede**: única entrega desta tarefa (`provider="mock"`, `key`/`model`
  `undefined`); BYOK + rede real são a F3.10 (gate). Conversa web = stub F3.12.
