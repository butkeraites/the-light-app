## Objetivo

A ADR-0043 (F5.14) entregou persistência do modo de tema e o `ThemeProvider` já expõe
`setMode(mode|null)` (null → `removePref` → segue `useColorScheme`) mais um flag `isSystem`
(`app/lib/theme.ts:51-113`), mas o único controle de UI é o `ThemeToggleButton` binário
(chama `toggle`) — então a capacidade "seguir sistema" é inalcançável pelo usuário. Adicionar
um controle tri-estado (claro / escuro / sistema) no header/ajustes que chame
`setMode('light'|'dark'|null)` e reflita `isSystem`. 100% offline, sem tocar o `the-light`,
sem segredo.

## Aceite

- [ ] Um controle no header/ajustes deixa o usuário escolher claro, escuro ou sistema; escolher
      "sistema" chama `setMode(null)` (`removePref` → segue `useColorScheme`) e cada escolha
      sobrevive a um reinício do app.
- [ ] O controle reflete o estado atual via `isSystem`/`mode` do `ThemeProvider`; nativo + web
      ambos o renderizam; todo cromo via `t()` com paridade pt/en.
- [ ] `npm run test:web:theme` verde (ainda assere que "sistema" limpa `tla.pref.theme.mode`);
      `npm run test:i18n` + `npm run test:i18n-coverage` verdes; `npx tsc --noEmit` 0;
      `npx expo export --platform web` sucede.
- [ ] a11y: o controle tem role/label adequados (`test:a11y-scan` verde); the-light intocado (`225b8c9`); sem segredo.

## Verificação

```sh
cd app && npx tsc --noEmit && npm run test:web:theme && npm run test:i18n && npm run test:i18n-coverage && npm run test:a11y-scan && npx expo export --platform web >/dev/null && echo EXPO_OK
```
