## Resumo

Caminho **iOS NATIVO** da ponte Rust→Expo provado de forma headless e
reprodutível. O `ubrn build ios --sim-only --and-generate` gerou o
`TheLightAppCoreFramework.xcframework` (staticlib `aarch64-apple-ios-sim` com a
UniFFI scaffolding) + a glue do Turbo Module (JSI); a integração Expo (raiz como
RN turbo-module library, autolink via `app/react-native.config.js`, prebuild +
pod install + New Arch codegen) linka o módulo; e o app rodando no **iPhone 17
(iOS 26.5)** resolve `"Jo 3.16"` (PT) e `"John 3:16"` (EN) **pelo Rust nativo**
(`the-light-core::reference` via UniFFI → Turbo Module JSI), ambos para
`book=43, chapter=3, verses=Single{verse:16}`.

Re-verificação **independente** e **incremental** (reaproveitando xcframework,
prebuild `app/ios/`, Pods e o `.app` em DerivedData fora do file-provider):
`scripts/run-ios-selftest.sh` (com `SKIP_BUILD=1`) booto o simulador, subiu o
Metro, instalou+lançou o app e capturou do log unificado do simulador os DOIS
marcadores — **saiu 0 em duas execuções consecutivas**. Como o build é Debug+Metro,
o JS revisado (`reference.ts` → barrel gerado → JSI; `selftest.ts`) é re-empacotado
e executado **fresco** contra o Turbo Module nativo (Rust) embarcado no `.app`,
tornando a prova de runtime genuína.

**Anti-fake confirmado lendo o código** (não confiei no relato): o glue nativo
`app/web/reference.ts` **delega** ao barrel gerado (`./native-generated/src/index`),
sem parser TS nem JSON hardcoded; o barrel chama `installRustCrate()` e reexporta os
bindings; o `parseReference` gerado é um shim UniFFI/JSI real — invoca
`nativeModule().ubrn_uniffi_the_light_app_core_fn_func_parse_reference(...)`,
lowering a string e lifting o `Reference` do rustbuffer (nenhuma lógica de parsing
em TS, nenhum eco). O `selftest.ts` compõe o marcador a partir do **retorno** real
(`ref.book` / `ref.chapter` / `v.inner.verse`); os literais `43/3/16` **não**
aparecem no glue/selftest — só na **asserção** do `run-ios-selftest.sh` (valor
esperado) e num comentário de doc. `reference.web.ts` (wasm, F0.6b) intacto; core e
fronteira intactos.
