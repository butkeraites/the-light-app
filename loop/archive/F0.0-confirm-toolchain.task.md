## Objetivo
Levantar e fixar as versões das ferramentas necessárias para a ponte Rust→Expo,
sem escrever código de produto. Esta tarefa só descobre e registra o ambiente —
é a base para todas as decisões seguintes.

## Contexto / arquivos
- Ainda não há estrutura de projeto (ela nasce na F0.1). Aqui só inspecionamos o
  ambiente e produzimos um relatório.
- Fonte de verdade de design: `VISION_AND_ARCHITECTURE.md` (seções 4 e 9).
- A arquitetura escolhida usa `uniffi-bindgen-react-native` (Mozilla/Filament),
  que gera bindings TS para Turbo Modules nativos (iOS/Android) e para WASM (web)
  a partir do mesmo Rust. O projeto pode ter sido renomeado para
  `uniffi-bindgen-javascript` — verificar o nome/versão atuais.

## Passos sugeridos
1. Rust: `rustc --version`, `cargo --version`, `rustup target list --installed`.
   Verificar/instalar `wasm32-unknown-unknown` e os targets de iOS/Android.
2. Node/JS: versão do Node, gerenciador de pacotes, e versão atual do Expo SDK
   e do `expo-router`.
3. UniFFI: localizar `uniffi-bindgen-react-native`/`uniffi-bindgen-javascript`,
   anotar versão e o fluxo de build/CLI (ex.: `ubrn`).
4. Mobile: confirmar Xcode (iOS) e Android SDK/NDK (`cargo-ndk`).
5. Anotar tudo numa tabela ferramenta → versão → status (ok / faltando).

## Critério de aceite
- [ ] Tabela com cada ferramenta e versão detectada.
- [ ] `wasm32-unknown-unknown` instalado (ou marcado como pendência a instalar).
- [ ] Nome e versão do gerador de bindings UniFFI confirmados.
- [ ] Lacunas (Xcode/NDK/targets ausentes) listadas como bloqueios explícitos.

## Comando(s) de verificação
```sh
rustc --version && cargo --version && rustup target list --installed | grep -E "wasm32|ios|android"
node --version && npx expo --version
```
