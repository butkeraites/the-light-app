## Objetivo

Fazer o **transporte web de IA** streamar a resposta do provedor **token-a-token**
(SSE/`ReadableStream`) e chamar um **callback de token**, para a UI web
(`ReaderAskPanel`) exibir a interpretação **incremental** (como o nativo já faz). Isto
realiza o follow-up **explicitamente adiado** em ADR-0025:

> "`askAnchoredStream` web é **não-streaming** (emite a interpretação 1× via `onToken`;
> SSE/`ReadableStream` = follow-up)."

Hoje (`app/web/reading.web.ts:331` `askAnchoredStream`) o web obtém a resposta
**completa** por `askAnchored` e emite a `interpretation` **inteira 1×** via `onToken`.
Esta tarefa troca isso por **streaming real**: o transporte lê o `ReadableStream` do
`fetch` (endpoint `:streamGenerateContent?alt=sse`), extrai cada **delta** de texto e
chama `onToken(delta)` incrementalmente, acumulando o texto completo para a
`ai_web_finalize`.

**App-side APENAS.** A fronteira Rust `ai_web_prepare`/`ai_web_finalize` (`ai-pure`,
cfg-free) **NÃO muda**: `cited_text` (texto bíblico numerado, VERBATIM do store) sai do
`prepare` e a `interpretation` final (concatenada) passa pela MESMA `finalize`
(`rewrite_anchors` — mesma impl Rust do nativo). **Anti-alucinação ZERO-DRIFT:** o
streaming só fatia a **interpretação do modelo** em incrementos; nenhum texto bíblico
é streamado nem reconstruído em TS; o `cited_text` viaja SEPARADO.

**NÃO-BLOQUEANTE** (prova por MOCK de stream, sem chave/rede real). **NÃO toca
`the-light`** (`04b9b24` intacto).

  `JSON.parse` do payload após `data: `, extrair o texto do delta, `onToken(delta)`,
  acumular. A extração do delta REUSA a lógica de `geminiExtract` (mesmo shape).
- **Fronteira Rust (INALTERADA):** `ai_web_prepare`/`ai_web_finalize` em
  `core/src/lib.rs` (bindings web já gerados; F2.7b/ADR-0025). NÃO editar `core/`.

## Passos sugeridos

1. Em `ai-anchored.web.ts`, refatorar a extração do delta a partir de `geminiExtract`
   (uma função que dado um `GenerateContentResponse` parcial devolve o texto daquele
   evento) e adicionar um `geminiCompleteStream(fetchImpl, key, request, onToken?)` que:
   - usa o endpoint `:streamGenerateContent?alt=sse` (quando `onToken` presente);
   - lê `res.body.getReader()` + `TextDecoder`, parseia eventos `data: {...}`, chama
     `onToken(delta)` por evento e acumula o texto completo;
   - devolve o texto completo (para a `finalize`). Sem `onToken`, mantém o caminho
     não-streaming (`:generateContent` + `res.json()` + `geminiExtract`) — sem regressão.
2. `webLlmTransport(fetchImpl, provider, key, parts, onToken?)`: quando `provider ===
   'mock'` e `onToken` presente, emitir o `MOCK_INTERPRETATION` em **≥1 incremento**
   (ex.: fatiado por sentença/palavra) para exercitar o caminho de stream offline;
   `'gemini'` → `geminiCompleteStream(..., onToken)`.
3. `askAnchoredOnHandle(..., onToken?)`: repassar `onToken` ao `webLlmTransport`; a
   `interpretation` que vai à `aiWebFinalize` é o texto **acumulado** (idêntico ao
   não-streaming).
4. `reading.web.ts::askAnchoredStream`: destubar para chamar `askAnchoredOnHandle(...,
   onToken)` (store web aberto/fechado), passando o `onToken` real. Manter a assinatura
   pública e o comportamento do `AiAnswer` final.
5. Criar a prova headless: `app/web/__tests__/askAnchoredStream-headless-entry.ts`
   (reexporta `init`/`mod`/`askAnchoredOnHandle`) + `askAnchoredStream.web.test.mjs`
   (molde do `askAnchored.web.test.mjs`, mas `fetch` MOCK devolve `ReadableStream` SSE
   com N chunks e o teste coleta `onToken`). Adicionar o script
   `test:web:ai-stream` em `app/package.json`.
6. `tsc --noEmit`, `eslint`, `expo export --platform web` verdes; sem regressão do
   `test:web:ai` (não-streaming) nem dos demais testes web.

## Restrições

- **Anti-alucinação ZERO-DRIFT:** o streaming muda **só o transporte**. NÃO
  reimplementar prompt/RAG/citação/aparato em TS (PROIBIDO, ADR-0029). `cited_text`
  sai do `ai_web_prepare` (store, VERBATIM) e a `interpretation` completa passa pela
  MESMA `ai_web_finalize` (`rewrite_anchors`, Rust). Os tokens são só da
  **interpretação do modelo** — **nunca** texto bíblico.
- **BYOK:** a chave vai **só no header** do `fetch` (nunca na URL/log). `fetch`
  INJETÁVEL; a prova usa MOCK — **nenhuma** rede/chave real no loop.
- **Offline-first:** a IA web é opt-in; sem chave, o app segue offline. O caminho
  `"mock"` roda sem rede.
- **NÃO tocar `the-light`** nem `core/src/lib.rs`/`core/Cargo.toml` (app-side apenas;
  `04b9b24` intacto). Se o streaming exigir mudar a fronteira Rust ou o core → **HALT**
  (não é o caso: `ai_web_prepare`/`finalize` são suficientes).
- **Qualidade por tarefa:** `tsc`/`eslint`/`expo export web` verdes; prova headless
  determinística (MOCK) verde; sem regressão dos testes web existentes.
- Se registrar a forma do streaming web (SSE `alt=sse` + parsing de `ReadableStream`),
  usar **ADR-0033** (próximo ADR livre).
