## Objetivo

Entregar a **UI nativa (iOS/Android)** de **estudo assistido ancorado**: a partir do
Reader (painel por-versículo da F1.9/F1.11), o usuário abre um painel de "Perguntar"
sobre a passagem selecionada, escolhe **provedor/modelo**, digita uma pergunta e recebe
a resposta com a **interpretação em STREAMING** (token-a-token), mostrando **claramente
separados** o **texto citado** (verbatim do store) e a **interpretação** (do LLM). A
capacidade é provada **ponta a ponta no device** com o provedor **`"mock"`** (sem
chave/rede), delegando à fronteira já entregue (`ask_anchored`/`ask_anchored_stream` +
`AiTokenCallback`, F2.1/F2.3a) via o glue nativo estendido → JSI → Rust. Só nativo; a
paridade web de IA é a **F2.7** (stub aqui). A validação real com a chave do usuário
(Claude/GPT/Gemini) é a **F2.6** (bloqueante, gate) — esta tarefa é **não-bloqueante**.
