## Resumo

Paridade web de **BUSCA (FTS5)** sobre o subset `reading-sample.sqlite` (~4,4 MB),
**verificada de forma independente**. O risco crítico da tarefa — `wa-sqlite@1.0.0`
do npm **não compila FTS5** — foi resolvido de verdade: o executor rebuildou o
`wa-sqlite` SYNC com `-DSQLITE_ENABLE_FTS5` (docker/emscripten, pinado no gitHead
`5147454` do release npm) e **vendorizou** o par `.mjs`+`.wasm` como asset local
offline-first em `app/web/vendor/wa-sqlite-fts5/`. Confirmei com **probe binário**
(25 ocorrências de `fts5` no `.wasm` vendored vs **0** no `dist/` do npm) e com um
**probe funcional** em node (criar `fts5` vtable + `highlight()`/`bm25()`/`MATCH` →
`FTS5_OK`; o `dist/` do npm → `FTS5_FAIL no such module: fts5`). A busca web é REAL
(`search` destubado em `reading.web.ts` → `searchOnHandle` em `sqlite-search.web.ts`),
espelhando o SELECT do core (`MATCH`+`bm25`+`highlight`+filtro de livro+limit+
translation, `has_translation` antes), **sem LIKE** e **sem ranqueamento reimplementado
em TS**. Prova headless verde, leitura sem regressão, `tsc`/`expo export` verdes,
`the-light` intacto (`8f66004`), caminho nativo não tocado.

**Decisão: PASSED.**

## Decisões / ADRs

- **ADR-0020** — Busca WEB (A1, FTS5). **FTS5 habilitado no `wa-sqlite` SYNC via build
  da FONTE** com `-DSQLITE_ENABLE_FTS5` (Emscripten/docker, `emscripten/emsdk:3.1.61`),
  **pinado no COMMIT do release npm `1.0.0`** (gitHead `5147454`, NÃO o tag `v1.0.0` —
  preserva a API JS `registerVFS`/`vfs_register` do `src/sqlite-api.js` do npm).
  Artefato **vendorizado como asset local offline-first** em
  `app/web/vendor/wa-sqlite-fts5/{wa-sqlite.mjs,wa-sqlite.wasm}` (versionado;
  reprodutível pelo `scripts/build-wa-sqlite-fts5.sh`). **UM ÚNICO wasm p/ leitura E
  busca** (leitura não regride). Espelho TS do SELECT de busca do core
  (`MATCH`/`bm25`/`highlight`); ranqueamento/destaque vivem no FTS5 do SQLite, não em
  TS. **Reusa ADR-0018** (Opção A/A1) e **ADR-0019** (store web de leitura,
  `openReadingDbWeb`); herda **ADR-0011/0012** (wa-sqlite+OPFS, build SYNC sem
  SharedArrayBuffer/COOP-COEP, MemoryVFS hidratado) e **ADR-0014** (subset bundled).
  Anti-alucinação preservada (texto sempre do store local; U+0002/U+0003 só em
  `highlighted`).
- **Alternativas rejeitadas (confirmadas no ADR):** cair para `LIKE` (drift vs
  FTS5/BM25 do nativo) — não foi necessário; `@sqlite.org/sqlite-wasm` (exigiria
  Worker+COOP-COEP/SharedArrayBuffer → fere offline-first estático); tag `v1.0.0`
  (API JS incompatível); build assíncrono (Asyncify/JSPI desnecessário).
