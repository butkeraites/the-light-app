## Resumo

A fronteira UniFFI passou a expor `parse_reference(input: String) -> Result<Reference, CoreError>`
**delegando** o parsing ao `the-light-core` (git dep pinada por commit, ADR-0002),
sem reimplementar lógica de domínio. O Executor adicionou a git dep em
`core/Cargo.toml` pinada pelo rev exato `0888ac0...`, e o `cargo build` resolveu o
core real (**`the-light-core` v1.2.0** do workspace `butkeraites/the-light`), fixando-o
em `core/Cargo.lock`. Em `core/src/lib.rs`, os tipos de fronteira `Reference { book,
chapter, verses }` e `VerseRange { Single | Range | WholeChapter }` espelham
`the_light_core::model` via `impl From`, e o erro do core (`ReferenceError`) é mapeado
para o `CoreError::Generic` já existente da F0.2. Rodei eu mesmo o bloco de verificação
1–5 completo: **fmt/clippy/build/test verdes, pinos corretos, delegação real, working
tree limpo** — terminando em "F0.3 verificada.". Inspecionei a fonte clonada (consumo
permitido) e confirmei que a API delegada é real e que **nenhuma fonte do core foi
copiada/vendorizada** para dentro de `core/`. Decisão: **passed**.

## Decisões / ADRs

- **ADR-0002 exercida de verdade:** a resolução real do `the-light-core` que a F0.2
  transferiu para esta tarefa foi concretizada. O `cargo build` clonou e resolveu o
  workspace público `butkeraites/the-light` no rev `0888ac0...`, e a resolução ficou
  **fixada em `core/Cargo.lock`** como `the-light-core 1.2.0` (`source = git+...?rev=0888ac0...`).
- **API real do core descoberta na fonte clonada** (mecanismo da tarefa — leitura de
  `~/.cargo/git/checkouts/`, consumo permitido):
  - Crate alvo: `crates/the-light-core` (package `the-light-core`, v1.2.0) dentro de um
    workspace; cargo resolve por **nome de package** (o `package = "the-light-core"` bate).
  - `the_light_core::reference::parse_reference(input: &str) -> Result<Reference, ReferenceError>`.
  - `the_light_core::model::Reference { book: u8, chapter: u16, verses: VerseRange }` e
    `VerseRange { Single(u16) | Range { start, end } | WholeChapter }`.
- **Tipos de fronteira desacoplados:** o Executor não exportou os tipos do core direto;
  definiu `Reference` (uniffi::Record) e `VerseRange` (uniffi::Enum) na fronteira com
  `impl From<the_light_core::model::...>` — mantém a API UniFFI estável e ainda assim com
  fonte única do parsing. Aprovado: é adaptação de tipo, não reimplementação de lógica.
- **Erro:** `ReferenceError` do core → `CoreError::Generic { message }` via `to_string()`,
  reutilizando o enum da F0.2 (sem variante nova). Aceitável pelo aceite (mapeamento, não
  reimplementação).
