## Objetivo

Dar **paridade web de REFERÊNCIAS CRUZADAS (xref)** sobre o subset
`reading-sample.sqlite` (~4,4 MB; o MESMO que o nativo empacota, ADR-0014),
implementando o ADR-0018 (Opção A / A1) e o ADR-0019/0020 (store web já
estabelecido). Hoje `app/web/reading.web.ts::crossRefs` é um **stub** que lança em
runtime (`WEB_XREF_MSG`). Esta tarefa o **destuba**, espelhando em TS — como pura
INFRAESTRUTURA — o **SELECT real** de `the_light_core::xref::for_verse` (filtro
`from_*` + `votes >= min_votes`, `ORDER BY votes DESC, …`, `LIMIT`, montagem
`VerseRange::Single`/`Range`), lendo do MESMO store `wa-sqlite`/OPFS+MemoryVFS já
provado nas F1.13/F1.14 (REUSAR `openReadingDbWeb`, **sem recarregar o subset**).

As MESMAS telas React compartilhadas (`app/app/read/[book]/[chapter].tsx` +
`ReaderXrefPanel` / `ReaderVersePanel`) passam a funcionar no browser **só por
este glue**: lista de xrefs por votos + clique → Reader + **atribuição CC-BY
visível** (string EXATA, ADR-0016). A xref nativa (F1.8/F1.9) **não** é tocada.

**NÃO reimplementar** ordenação/filtro/semântica de xref em TS: votos DESC e o
filtro `min_votes` vêm do **SQL** que espelha o core; o web só monta o SELECT
idêntico, faz o bind dos params na mesma ordem e compõe os Records `CrossRef`.

## Contexto / arquivos

**SQL REAL a espelhar — confirmado na fonte** (`the-light-core/src/xref.rs`, rev
pinado `8f66004`, função livre `for_verse`, consumo só-leitura):

```sql
SELECT to_book, to_chapter, to_verse_start, to_verse_end, votes
FROM cross_references
WHERE from_book = ?1 AND from_chapter = ?2 AND from_verse = ?3 AND votes >= ?4
ORDER BY votes DESC, to_book, to_chapter, to_verse_start
LIMIT ?5
```

- **Colunas de filtro:** `from_book`, `from_chapter`, `from_verse` (a tríade
  canônica de ORIGEM; xref é **independente de tradução** → NÃO há parâmetro
  `translation` nem checagem `has_translation`, diferente de `getChapter`/`search`).
- **Colunas lidas:** `to_book`, `to_chapter`, `to_verse_start`, `to_verse_end`,
  `votes`.
- **Filtro:** `votes >= min_votes` (oculta disputadas/negativas por padrão).
- **Ordenação EXATA (espelhar inteira, anti-drift):**
  `ORDER BY votes DESC, to_book, to_chapter, to_verse_start` — **não** só
  `votes DESC`; os tiebreakers fazem parte do mirror.
- **Limit:** `LIMIT ?5` com **clamp** `limit.clamp(1, i64::MAX)` → mínimo 1 (mesmo
  padrão do `querySearch` da F1.14: `Math.max(1, Math.trunc(limit))`).
- **Bind (ordem do core):** `book` (i64), `chapter` (i64), `verse` (i64),
  `min_votes` (i64), `limit` (i64).
- **Montagem da `Reference` de destino** (espelhar a regra do core):
  `if to_verse_start >= to_verse_end { VerseRange::Single(to_verse_start) } else
  { VerseRange::Range { start: to_verse_start, end: to_verse_end } }`.
- **Defaults (mesmos do core/F1.8):** `min_votes = None → DEFAULT_MIN_VOTES = 1`;
  `limit = None → DEFAULT_LIMIT = 20`.

**Fronteira nativa que o web ESPELHA** (NÃO tocar):
`core/src/lib.rs::cross_refs(db_path, book:u8, chapter:u16, verse:u16,
min_votes:Option<i64>, limit:Option<u32>) -> Vec<CrossRef>` delega a
`xref::for_verse(store.conn(), …)`; Record `CrossRef { reference: Reference,
votes: i64 }`. **Tipo `votes`: i64 → `bigint` no binding TS**; o parâmetro
`minVotes` é `bigint` (ver assinatura web abaixo). Use `column_int64` (BigInt) para
`votes` e bind de `min_votes`/`limit` como int64 — fiel ao contrato i64.

**Assinatura web a destubar** (já existe, hoje stub) em `app/web/reading.web.ts`:
```ts
crossRefs(_dbPath, _book, _chapter, _verse, _minVotes?: bigint, _limit?: number): Promise<CrossRef[]>
```
A tela chama `crossRefs(dbPath, bookNumber, chapterNumber, selectedVerse)` (4 args
→ minVotes/limit `undefined` → defaults).

**Store web já estabelecido (REUSAR, não duplicar):**
- `app/web/sqlite-reading-opfs.web.ts` → `openReadingDbWeb()` (MemoryVFS+OPFS,
  build SYNC, wasm FTS5 vendorizado — **um único wasm** p/ leitura/busca/xref).
- `app/web/sqlite-reading.web.ts` → tipo `ReadingDb`/`PassageDb` (handle de infra).
- `app/web/sqlite-search.web.ts` → **molde do PADRÃO de glue de query** (constante
  SQL espelhada + `querySearch` + `composeSearchHit` + `searchOnHandle`).
- `app/web/reading.web.ts` → onde `crossRefs` é destubado (remover `WEB_XREF_MSG`).

**UI (F1.9 / F1.11) compartilhada — passa a funcionar no web só pelo glue:**
- `app/components/ReaderXrefPanel.tsx` (lista + votos via `String(cr.votes)` +
  clique + **atribuição CC-BY** `XREF_ATTRIBUTION`).
- `app/components/ReaderVersePanel.tsx` (painel combinado F1.11; reusa
  `XREF_ATTRIBUTION`).
- `app/app/read/[book]/[chapter].tsx` (chama `crossRefs`, faz `router.push`).

**Dado do subset (confirmado por SQL — para as asserções, NÃO hardcodar no
produto):** `reading-sample.sqlite` tem **22.413** xrefs; João 3:16 (`from_book=43
AND from_chapter=3 AND from_verse=16`) tem **9** xrefs; por `votes DESC` o 1º é
**João 3:15** (`to_book=43, to_chapter=3, to_verse_start=15, to_verse_end=15`,
`votes=439`) → `Single`; o 2º é **João 11:25-26** (`start=25, end=26`, `votes=400`)
→ `Range`. **PARIDADE** com o nativo: `TLA_XREF first_ref="John 3:15"
first_votes=439` (F1.9).

## Restrições (regras rígidas relevantes)

- **NÃO tocar `the-light`/fronteira Rust.** A xref já existe na F1.8
  (`cross_refs` → `xref::for_verse`); o web **só ESPELHA** o SELECT no glue. Se algo
  exigir mudar o core → `blocked` + **HALT** com o erro EXATO (núcleo só via PR + ADR).
- **NÃO reimplementar** ordenação/filtro/semântica de xref em TS: votos DESC (com os
  tiebreakers) e o filtro `votes >= min_votes` vêm do **SQL espelhando o core**; o
  `clamp` do limit também (≥1). Nada de `.sort`/`.filter` de domínio em TS.
- **Subset A1 só** (`reading-sample.sqlite` ~4,4 MB, o MESMO do nativo, ADR-0014). O
  corpus completo (~59 MB) é backlog transversal — fora desta tarefa.
- **REUSAR o store das F1.13/F1.14** (`openReadingDbWeb`/MemoryVFS+OPFS, wasm
  vendorizado) — **sem recarregar** o subset nem criar novo backend OPFS.
- **Anti-alucinação:** refs e votos vêm **sempre do store local**; nada hardcoded no
  produto (constantes só nas asserções do teste); **zero rede** em runtime.
- **Atribuição CC-BY OBRIGATÓRIA e visível no web** (ADR-0016): string EXATA
  `Cross references courtesy of OpenBible.info (CC-BY)`, não omitir/alterar.
- **Offline-first:** assets locais (subset + wasm vendorizado já no bundle); **sem
  SharedArrayBuffer/COOP-COEP**.
- **Paridade real com o nativo:** João 3:15 como 1º por votos (439), igual ao
  `TLA_XREF` da F1.9.
- **Tipo `votes`:** i64 → `bigint` no binding; `minVotes` é `bigint`. Ler `votes` via
  `column_int64` e bind de `min_votes`/`limit` como int64 (fiel ao i64 do core).
- Gerados/bundled seguem **ignorados** (bindings web, `app/dist/`). Versionar só:
  glue web novo/destubado, entry+teste headless, `package.json` (script),
  `DECISIONS.md` (ADR novo).
