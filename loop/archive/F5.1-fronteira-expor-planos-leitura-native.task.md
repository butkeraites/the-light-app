## Objetivo

Expor a geração de planos de leitura JÁ EXISTENTE no core
(`the_light_core::userdata::plans`) através da fronteira do app (`core/src/lib.rs`),
com uma **fronteira NATIVA** (impl sob `#[cfg(not(target_arch = "wasm32"))]` + **stub
web**) e um **host test Rust determinístico**. Menor fatia provável: listar os 3 planos,
obter as referências (capítulos inteiros) de um dia, e computar o índice do dia de hoje —
**sem** persistência, **sem** UI ainda.

> **RE-ESCOPO (attempt 1 → blocked → native-first).** A versão anterior exigia funções
> **cfg-free compiláveis em wasm**. Isso é **impossível** no core pinado `2fc2dab`:
> `pub mod userdata` é `#[cfg(feature = "embedded")]` (verificado: `core/src/lib.rs:38-39`),
> logo `userdata::plans` **não existe** sob `ai-pure`/wasm (E0433 comprovado por probe
> isolado). Tornar isso cfg-free exigiria um **PR ao `the-light`** (expor a superfície PURA
> de geração de planos sob `ai-pure`) — decisão de gate/PR+ADR. Portanto seguimos o **padrão
> documentado** do projeto (nasce no núcleo → **fronteira NATIVA + teste host** → UI nativa →
> **paridade web após gate estratégico**): F5.1 é NATIVA; a **paridade web (F5.10)** será o
> gate estratégico que exige a PR `ai-pure` ao core (futuro ADR). Anti-drift: NÃO reimplementar
> a geração/chunking em Rust/TS.
