## Objetivo

Quarta e **última etapa construível** da trilha de sync (ADR-0036) antes do gate humano
F5.27. Entregar a **UI opt-in** que costura tudo — export/import manual (F5.23, todos os
alvos) + link/unlink do Google Drive (F5.24, web) + "Sincronizar agora" (F5.25, web) — e
**ligar o `SnapshotStore`** (a interface injetável das F5.23/F5.25) ao **store real** do app,
para que export/import funcione de ponta a ponta contra os dados reais do usuário.

**Offline-first é a base, e a UI deve DEIXAR ISSO EXPLÍCITO:** sync é **OFF por padrão**,
100% opt-in, e a tela diz claramente "o app funciona totalmente offline sem isto".

## Contexto / peças a costurar

- **F5.23** `app/lib/userdataSnapshot.ts` — `exportSnapshot`/`parseSnapshot`/`importSnapshotIntoStore`
  sobre `SnapshotStore` (a interface a ligar ao store real).
- **F5.24** `app/lib/driveAuth.ts` — `createDriveAuth`/`link`/`unlink`/`isLinked`/`currentToken`.
- **F5.25** `app/lib/driveSync.ts` — `createDriveSync`/`pushSnapshot`/`pullSnapshot`/`syncNow`.
- **F5.2** prefs KV (`app/lib/prefs.ts`) — persistir o flag opt-in (OFF por padrão) + i18n `t()`.
- **i18n (F5.2/5/…)** — TODO texto de cromo desta tela via `t()` (PT/EN); nada de hex/PT hardcoded.
- **Store real do usuário:** as fns de fronteira de notas/marcações/progresso já existentes
  (`putNote`/`listNotes`/`addHighlight`/`listHighlights`/`readingPlanProgress`/
  `startReadingPlan`/`setReadingPlanCompleted`). Um adaptador `SnapshotStore` sobre elas.

## Escopo desta tarefa

1. **Adaptador `SnapshotStore` sobre o store real** (`app/lib/snapshotStore.ts` ou similar):
   liga a interface da F5.23 às fns de fronteira reais do userdata. **NATIVO é o alvo
   garantido** (as fns fs-backed existem lá). Para o **web**, ligue onde a superfície já
   existir (OPFS/wasm); se alguma peça de notas/marcações NÃO estiver disponível no web sem
   tocar o `the-light`, **NÃO expanda escopo e NÃO toque o `the-light`** — entregue o
   adaptador onde é possível, deixe o resto graciosamente indisponível na UI com aviso, e
   **registre a limitação** no ADR (candidato a follow-up). Se o wiring exigir mudança no
   `the-light` p/ funcionar, **HALT** com o motivo (não improvise).
2. **Tela/So seção de Sync** (rota tipo `/settings/sync` ou seção em ajustes):
   - **Toggle opt-in de sync — OFF por padrão** (persistido no KV da F5.2).
   - **Aviso de privacidade** explícito: o QUE sincroniza (notas + marcações + progresso de
     plano) e o que **NUNCA** sai do device (sessões de IA, banco bíblico, chaves/segredos,
     texto de versículo além da referência). Sem telemetria.
   - Mensagem proeminente: **"funciona 100% offline sem isto"**.
   - **Export/import manual** (todos os alvos): botão exportar (gera o JSON da F5.23) + importar
     (arquivo → `importSnapshotIntoStore` com merge). Usar o file picker/share do alvo.
   - **Google Drive (web, opt-in):** link/unlink da conta (F5.24) + "Sincronizar agora"
     (F5.25). Só aparece/habilita no web e quando o opt-in está ON.
   - **a11y:** roles/labels nos interativos (padrão das F5.20/21), contraste (F5.18), i18n (F5.5/11/16).
3. **Segurança/privacidade (reforço, não-negociável):** token do Drive só em session/secure
   storage (F5.24), **nunca** em git/log; nada de segredo no repo; a UI não loga token nem
   conteúdo do snapshot; o opt-in OFF por padrão significa **zero rede** sem ação explícita.
4. **Prova** adequada ao alvo (a UI não roda 100% headless):
   - **Headless (obrigatório):** teste do **adaptador `SnapshotStore`** contra um store real
     in-memory/OPFS-mock — export→import round-trip via o adaptador (reusa a infra da F5.23);
     e um teste de que o **flag opt-in default é OFF** (lê do KV vazio → `false`) e que ligar
     grava/relê. Marcador `SYNC_UI store=ok optin_default_off=ok optin_persist=ok`. Script
     `test:web:syncui`.
   - **On-device/manual (se aplicável):** se houver como provar o export/import manual no
     simulador (como a F5.7 fez com `TLA_PLANS`), capture o marcador; senão, documente no
     result por que a prova é headless + inspeção.
5. **ADR-0054** — a decisão da UI de sync (opt-in OFF-por-padrão, superfície por-alvo
   —manual em todos, Drive só web—, o adaptador `SnapshotStore`→store real, e qualquer
   limitação de wiring web registrada).

## Fora de escopo

- **Validação com conta Google REAL** → **F5.27 (⛔ gate humano)**. Aqui NADA de rede real:
  o "Sincronizar agora" no web usa o motor da F5.25, cuja rede real só é exercida com uma
  conta linkada de verdade — isso é a F5.27. Não faça chamada real ao Google nesta tarefa.

## Critério de aceite

- [ ] Adaptador `SnapshotStore`→store real (nativo garantido; web onde possível sem tocar o-core);
      round-trip export/import provado via o adaptador.
- [ ] Tela de Sync: opt-in **OFF por padrão** (KV F5.2), aviso de privacidade (o que sincroniza /
      o que nunca sai), "funciona offline sem isto", export/import manual (todos alvos),
      link/unlink + "Sincronizar agora" (web), a11y + i18n + contraste.
- [ ] Só notas+marcações+progresso no snapshot; token/segredo nunca em git/log; sem telemetria;
      opt-in OFF ⇒ zero rede.
- [ ] Prova headless `SYNC_UI ...` verde; tsc 0; expo web 0; perf-budget LOCKED OK (ou re-baseline
      justificado se a tela adicionar módulos eager — rode a métrica!).
- [ ] i18n-coverage/contrast/a11y guardas verdes p/ a nova tela; the-light intocado (`225b8c9`); ADR-0054.
- [ ] Se algo exigir tocar o `the-light` → **HALT** com motivo (não improvisar).
