## Decisões / ADRs

- **ADR-0031** (F3.12a) — paridade WEB do estudo: fronteira web nova em
  `core/src/lib.rs` (`study_web_prepare`/`study_web_finalize` + Records), **cfg-free
  só `ai-pure`**, padrão prepare → (fetch TS) → finalize (ADR-0025).
- **ai-pure zero-drift provado por teste de paridade:** prompt
  (`system_prompt_in`+`user_prompt`), `verify`, citações (`CitationCollector`) e
  `to_academic_markdown` vêm do MESMO Rust do nativo — `assert_eq!` nativo↔web em
  `passage_text`/`user`/`system`/`sections`/`warnings`/`citations`/`academic_markdown`.
- **Léxico web = SELECT do subset (ADR-0011) + tipos puros:** `sqlite-lexicon.web.ts`
  espelha o shape de `ai::lexicon::verified_lexicon` (SELECT + agregação por Strong
  base + atribuições STEP CC-BY); glosas/lemas/Strong VERBATIM do store; nenhuma
  lógica de anti-alucinação/aparato em TS.
- **`web_sources` vazio nesta fatia** — pesquisa web Wikipedia = **F3.12b**
  (app-side apenas, sem re-tocar Rust); `fetched_at` do `WebSource` usa `Default`
  (época) porque `chrono` é dep **transitiva** do core (não nomeável sem tocar
  `Cargo.toml`) — inerte com a lista vazia.
- BYOK / chave web **session-only** (ADR-0025) — só no header do `fetch`, nunca em
  git/log/URL. Offline-first: IA web é opt-in; a única rede em runtime é o `fetch`.
